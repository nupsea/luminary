## Goal

Add a deck health report service that identifies orphaned, mastered, stale, and uncovered cards for a document, with three new API endpoints and a sibling Health Report card in the Study tab alongside the existing Bloom's Audit panel.

## Layers Affected

- Types (new TypedDicts for DeckHealthReport)
- Service (new DeckHealthService)
- API (three new endpoints in the flashcards router)
- Frontend (new HealthReportPanel component in Study.tsx)

## Files to Create or Modify

- `backend/app/types.py` -- add DeckHealthReport TypedDict and supporting types
- `backend/app/services/deck_health.py` -- create DeckHealthService (new file)
- `backend/app/routers/flashcards.py` -- add 3 new endpoints and Pydantic schemas; register health routes BEFORE /{document_id}
- `frontend/src/pages/Study.tsx` -- add HealthReportPanel component and wire it in as a sibling card below DeckHealthPanel
- `backend/tests/test_deck_health.py` -- unit tests for all three service methods (new file)
- `scripts/smoke/S160.sh` -- smoke test calling the health endpoint (new file)

### 2. [Service] Create `backend/app/services/deck_health.py`

Create the file from scratch. No reverse imports -- the service imports only from `app.models`, `app.services.flashcard` (for `get_flashcard_service`), `app.services.llm` (for generating cards), and standard library.

Key implementation notes:

**analyze(document_id, db):**
- Query all FlashcardModel rows for document_id where fsrs_state != 'archived'.
  Use `select(FlashcardModel).where(FlashcardModel.document_id == document_id)`.
- Query all SectionModel rows for document_id to build a valid section_id set.
  Use `select(SectionModel.id, SectionModel.heading).where(SectionModel.document_id == document_id)`.
- Compute orphaned: cards where section_id is not None (resolved via chunk join) AND the resolved section_id is not in the valid section_id set. To get section_id from cards: join FlashcardModel.chunk_id -> ChunkModel.id -> ChunkModel.section_id.
  - Use a single JOIN query: `select(FlashcardModel.id, ChunkModel.section_id).join(ChunkModel, FlashcardModel.chunk_id == ChunkModel.id).where(FlashcardModel.document_id == document_id)`.
  - A card is orphaned when `chunk.section_id is not None` AND `chunk.section_id not in valid_section_ids`.
  - Cards with `chunk_id = None` (source='note', 'gap', 'feynman') are NOT orphaned.
- Compute mastered: cards where `fsrs_stability > 180`.
- Compute stale: cards where `last_review is not None` AND `(datetime.now(UTC) - last_review).days > 90` AND `fsrs_stability < 7`.
- Compute uncovered_sections: section_ids that have zero matching cards (no FlashcardModel with chunk_id pointing to a ChunkModel with that section_id).
  - Build covered_section_ids from the JOIN query above (section_ids that appear in at least one card's chunk).
  - uncovered_section_ids = [sid for sid in valid_section_ids if sid not in covered_section_ids]
- Compute hotspot_sections: top 5 sections by card count from covered_section_ids. Count from the JOIN result; sort descending; take 5.
- Return DeckHealthReport TypedDict.

**archive_mastered(document_id, db):**
- Fetch mastered cards (fsrs_stability > 180) for document_id.
- Set `card.fsrs_state = 'archived'` for each. This is not a delete -- it hides them from study sessions without destroying them.
- `await db.commit()`.
- Return count of archived cards.

**generate_for_uncovered(document_id, section_ids, db):**
- For each section_id, fetch its SectionModel (heading, preview) and first-N chunks (limit 5) to build a text sample.
- Call `get_flashcard_service().generate(document_id, scope='section', section_heading=heading, count=3, session=db)` for each section.
  - Uses existing FlashcardService.generate which already handles LLM call, parsing, and DB write.
  - Note: FlashcardService.generate uses section heading (str), not section_id, to scope chunks.
  - This is correct: it reuses the existing `_fetch_chunks` path with scope='section'.
- Returns total count of cards created across all sections.
- Raises `litellm.ServiceUnavailableError` if Ollama is unreachable (propagated from FlashcardService).
- AsyncSession constraint: FlashcardService.generate calls `session.commit()` internally per section. Do NOT call sections concurrently -- call them sequentially with `for section_id in section_ids:` to avoid AsyncSession concurrent access violations (per MEMORY.md pattern).

**POST /flashcards/health/{document_id}/fill-uncovered (fire-and-forget):**
```python
class FillUncoveredRequest(BaseModel):
    section_ids: list[str] = Field(min_length=1)

@router.post("/health/{document_id}/fill-uncovered", response_model=FillUncoveredResponse, status_code=202)
async def fill_uncovered(
    document_id: str,
    req: FillUncoveredRequest,
    session: AsyncSession = Depends(get_db),
    health_service: DeckHealthService = Depends(get_deck_health_service),
) -> FillUncoveredResponse:
    task = asyncio.create_task(
        health_service.generate_for_uncovered(document_id, req.section_ids, session)
    )
    _background_tasks.add(task)
    task.add_done_callback(_background_tasks.discard)
    return FillUncoveredResponse(queued=len(req.section_ids))
```

Note on fire-and-forget: The story specifies this endpoint is fire-and-forget. The 202 response is returned immediately; card generation runs in the background. The AsyncSession from Depends(get_db) will be shared with the background task -- this is acceptable because the background task runs sequentially (no concurrent DB access) and the session is not closed until the task completes (FastAPI's session lifecycle). However, the more correct pattern is to open a NEW session inside the background task rather than sharing the request-scope session. Use a lazy import of `get_db_session` from `app.database` to open a new session inside the task. This avoids the session being closed by FastAPI before the task finishes.

Revised approach (safer):
```python
@router.post("/health/{document_id}/fill-uncovered", response_model=FillUncoveredResponse, status_code=202)
async def fill_uncovered(
    document_id: str,
    req: FillUncoveredRequest,
    health_service: DeckHealthService = Depends(get_deck_health_service),
) -> FillUncoveredResponse:
    async def _run():
        from app.database import get_db as _get_db  # noqa: PLC0415
        async for db in _get_db():
            await health_service.generate_for_uncovered(document_id, req.section_ids, db)
            break

    task = asyncio.create_task(_run())
    _background_tasks.add(task)
    task.add_done_callback(_background_tasks.discard)
    return FillUncoveredResponse(queued=len(req.section_ids))
```

```typescript
async function fetchDeckHealth(documentId: string): Promise<DeckHealthReport> {
  const res = await fetch(`${API_BASE}/flashcards/health/${documentId}`)
  if (!res.ok) throw new Error("Failed to load deck health report")
  return res.json() as Promise<DeckHealthReport>
}

async function archiveMastered(documentId: string): Promise<{ archived: number }> {
  const res = await fetch(`${API_BASE}/flashcards/health/${documentId}/archive-mastered`, {
    method: "POST",
  })
  if (!res.ok) throw new Error("Failed to archive mastered cards")
  return res.json() as Promise<{ archived: number }>
}

async function fillUncovered(documentId: string, sectionIds: string[]): Promise<{ queued: number }> {
  const res = await fetch(`${API_BASE}/flashcards/health/${documentId}/fill-uncovered`, {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ section_ids: sectionIds }),
  })
  if (!res.ok) throw new Error("Failed to queue uncovered section fill")
  return res.json() as Promise<{ queued: number }>
}
```

## Risk Flags

- Risk: The `/health/{document_id}` route segment could be matched as a document_id wildcard by FastAPI if registered after `/{document_id}`. Mitigation: place all three health routes in flashcards.py BEFORE the existing `/{document_id}` export/csv and list routes (same pattern as audit and cloze routes).

- Risk: The fire-and-forget fill-uncovered endpoint shares an AsyncSession with the background task if the request-scope session is passed directly. Mitigation: open a fresh session inside the task using `async for db in get_db(): ... break` (same pattern as other fire-and-forget tasks in the codebase).

- Risk: `generate_for_uncovered` iterates sections sequentially and calls `FlashcardService.generate` which internally calls `session.commit()`. If one section fails (LLM offline), cards for prior sections are already committed; subsequent sections are skipped. Mitigation: wrap in try/except per section; log failures; continue. Raise `litellm.ServiceUnavailableError` only if ALL sections fail (or propagate first Ollama error).

- Risk: Mastered cards with `fsrs_state = 'archived'` will still appear in `analyze()` counts (orphaned, stale) if the query does not exclude archived cards. Mitigation: in `analyze()`, filter cards with `FlashcardModel.fsrs_state != 'archived'` before applying all other criteria.

- Risk: Orphaned detection relies on a chunk JOIN. Cards with `chunk_id = None` (source='note', 'gap', 'feynman') have no chunk and should NOT be counted as orphaned. Mitigation: the JOIN naturally excludes them (NULL chunk_id produces no JOIN rows); verified by test_analyze_orphaned_cards which also adds a gap card and asserts it is NOT in orphaned_ids.

- Risk: `hotspot_sections` in `DeckHealthReport` is `list[HealthSection]` (a TypedDict), but when serialized through Pydantic `DeckHealthReportResponse` it must be `list[dict]`. Mitigation: define `hotspot_sections: list[dict]` in the Pydantic response model (same pattern as `gaps: list[dict]` in `CoverageReportResponse`).
