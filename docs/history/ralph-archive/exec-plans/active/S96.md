1. Extract `doc_ids[0]` as `document_id`; if absent, return error card

3. Call `get_gap_detector().detect_gaps(note_ids, document_id)` inside try/except

5. On success, return card with `gaps`, `covered`, `query_used`, `document_id` fields

raw_answer = result.get("answer") or ""

## __card__ Protocol Specification

This is the reference for S97, S98, S99 implementors.

### Producer contract (any graph node)

```python
import json

card_payload = {
    "type": "gap_result",   # or other type strings for S97-S99
    # node-specific fields
}
return {"answer": "__card__" + json.dumps(card_payload), "chunks": []}
```

Node must be wired directly to END in `build_chat_graph`. Never route through `synthesize_node` or `confidence_gate_node` for card answers.

### Consumer contract (stream_answer in qa.py)

```python
raw_answer = result.get("answer") or ""
if raw_answer.startswith("__card__"):
    card_payload = json.loads(raw_answer[8:])  # "__card__" is exactly 8 chars
    yield f"data: {json.dumps({'card': card_payload})}\n\n"
    yield f"data: {json.dumps({'done': True})}\n\n"
    return
```

Exactly two SSE events emitted: `card` then `done`. No `token` events.

### Frontend SSE consumer contract (Chat.tsx)

```typescript
if (payload["card"] !== undefined) {
  const cardData = payload["card"] as AnyCardData
  setMessages((m) =>
    m.map((msg) =>
      msg.id === assistantId
        ? { ...msg, type: "card" as const, cardData, isStreaming: false, text: "" }
        : msg,
    ),
  )
}
```

The `done` event that follows is a no-op for card messages (isStreaming already false, finalAnswer not present in card done payloads).

### Card payload schema for gap_result

```json
{
  "type": "gap_result",
  "gaps": ["string", "..."],
  "covered": ["string", "..."],
  "query_used": "string",
  "document_id": "string",
  "error": "string (optional, present on error cases only)"
}
```

## Risk Flags

**Risk: notes_gap with scope=all (no doc_ids) returns error card, not a failure.**
The `notes_gap_node` gracefully handles `doc_ids=[]` by returning an error card telling the user to switch to single-document scope. This is correct per the story description. The Chat suggestion pill "Find gaps in my notes" sends scope=all by default, which will always produce the error card. Mitigation: the error card message is explicit about what the user must do.
(mitigation: implemented correctly per AC)

**Risk: `__card__` prefix is exactly 8 characters.**
`raw_answer[8:]` strips the prefix. This offset is hardcoded. The comment in `qa.py` documents this. Any rename of the prefix requires updating both producer (all `notes_gap_node`, `socratic_node`, `teach_back_node`) and consumer (SSE loop). Do not change the prefix in S97-S99.
(mitigation: comment in code, documented in this plan)

**Risk: `synthesize_node` pass-through for card answers would trigger confidence retry.**
`synthesize_node` passes through when `existing_answer` is set. But the default `confidence="low"` and `retry_attempted=False` would then route to `augment_node`. Therefore `notes_gap_node` MUST be wired directly to END, not through `synthesize_node`. The graph is correctly wired.
(mitigation: implemented with direct edge to END)

**Risk: `GapDetectDialog` uses `Promise.all` for concurrent fetches.**
This is a pre-existing pattern not introduced by S96. Do not change the dialog in this story. Tracked as tech debt.
(mitigation: out of scope for S96)

**Risk: `GapCardData` must be exported from `GapResultCard.tsx` and imported in `Chat.tsx`.**
`GapResultCard.tsx` exports `GapCardData` interface. `Chat.tsx` imports it as `import type { GapCardData } from "@/components/GapResultCard"`. This avoids duplicate type definitions.
(mitigation: implemented correctly)
