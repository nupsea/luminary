# S142 -- Tech Chat Web Grounding

## Goal

Add optional per-conversation web augmentation to the chat graph so the LLM can surface version mismatches between local (potentially stale) content and current web sources, with a rate-limited toggle in the Chat UI.

---

## Layers Affected

- Types (ChatState extension, new WebSnippet TypedDict)
- Service (new WebSearcher service)
- Runtime (new web_augment_node, synthesize_node prompt extension, version_mismatch support)
- API (QARequest model extension to pass web_enabled)
- Frontend Chat (web toggle, counter, source labeling, version mismatch badge)

---

## Pre-Implementation Checks

1. `WEB_SEARCH_PROVIDER` already exists in `backend/app/config.py` (added in S138). No config change needed.
2. No `WebSnippet` or `web_enabled` fields exist anywhere in the codebase yet -- all clean slate.
3. The `ChatState` TypedDict in `backend/app/types.py` does NOT have `web_enabled` or `web_calls_used` fields -- both must be added.
4. `QARequest` in `backend/app/routers/qa.py` does NOT have a `web_enabled` field -- must be added.
5. `stream_answer()` in `backend/app/services/qa.py` builds `initial_state` -- must pass `web_enabled` from the request.
6. The chat graph `build_chat_graph()` in `backend/app/runtime/chat_graph.py` must register `web_augment_node` and wire it into the confidence gate routing.
7. `Chat.tsx` in `frontend/src/pages/Chat.tsx` fetches `GET /settings/llm` already -- the web search provider status must be surfaced through a new dedicated endpoint `GET /settings/web-search` so the Chat UI can know whether to enable the toggle. Alternatively, add `web_search_provider` field to the existing `GET /settings/llm` response.
8. Privacy invariant: web snippets must NOT be written to QAHistoryModel, DocumentModel, or any other DB table.

**WebSearcher class:**

```python
class WebSearcher:
    """Dispatch web search to the configured provider.

    provider='none'     -- always returns [] silently
    provider='brave'    -- calls Brave Search API (requires BRAVE_API_KEY env)
    provider='tavily'   -- calls Tavily API (requires TAVILY_API_KEY env)
    provider='duckduckgo' -- uses duckduckgo_search library (no key required)

    All provider failures are caught and logged; returns [] on any error.
    Content is NOT stored in the database per privacy invariant.
    """

    async def search(self, query: str, k: int = 3) -> list[WebSnippet]:
        """Return up to k WebSnippet results for the query.

        Returns [] if provider='none' or on any provider error.
        """
```

Provider dispatch:
- `none`: return [] immediately (no network calls)
- `duckduckgo`: use `duckduckgo_search` package (already available via litellm deps or install via `uv add duckduckgo-search`). Call `DDGS().text(query, max_results=k)`. Map each result to `WebSnippet`.
- `brave`: call `https://api.search.brave.com/res/v1/web/search?q={query}&count={k}` with header `X-Subscription-Token: {BRAVE_API_KEY}`. Map results.
- `tavily`: call `https://api.tavily.com/search` POST with `{"query": query, "max_results": k}` and `Authorization: Bearer {TAVILY_API_KEY}`. Map results.

Note: For brave and tavily, the API keys must come from environment variables `BRAVE_API_KEY` and `TAVILY_API_KEY`. Add these to `Settings` in `config.py`:

```python
BRAVE_API_KEY: str = ""
TAVILY_API_KEY: str = ""
```

For V1, implement `none` and `duckduckgo` fully. Brave and Tavily stubs that return [] with a logged warning are acceptable if the keys are absent.

### Step 4. [Runtime] Add web_augment_node to chat_graph.py

Add to `backend/app/runtime/chat_graph.py`:

```python
async def web_augment_node(state: ChatState) -> dict:
    """Fetch web snippets to supplement low-confidence local answers.

    Fires only when:
      - state['web_enabled'] is True
      - state['confidence'] is 'low' (confidence < 0.8 threshold, expressed as 'low')
      - state['web_calls_used'] < 3 (rate limiter)

    Returns {} (no-op) when any condition is False.
    Appends snippets to state['web_snippets'] and increments web_calls_used.
    Web snippets are formatted as [Web: domain.com] labeled section_context entries.
    Web snippets are NOT written to DB (privacy invariant).
    """
    web_enabled = state.get("web_enabled", False)
    confidence = state.get("confidence", "high")
    web_calls_used = state.get("web_calls_used", 0)

    if not web_enabled:
        logger.info("web_augment_node: web_enabled=False -- skipping")
        return {}

    if confidence != "low":
        logger.info("web_augment_node: confidence=%s -- skipping (only fires on low)", confidence)
        return {}

    if web_calls_used >= 3:
        logger.info("web_augment_node: rate limit reached (%d/3) -- local-only fallback", web_calls_used)
        return {}

    question = state.get("rewritten_question") or state["question"]
    logger.info("web_augment_node: fetching web snippets for query=%r", question[:60])

    from app.services.web_searcher import get_web_searcher  # noqa: PLC0415

    snippets: list[dict] = []
    try:
        results = await get_web_searcher().search(question, k=3)
        snippets = [dict(s) for s in results]
    except Exception:
        logger.warning("web_augment_node: web search failed", exc_info=True)
        snippets = []

    if not snippets:
        logger.info("web_augment_node: no web results returned")
        return {"web_calls_used": web_calls_used + 1}

    # Format snippets as labeled section_context entries
    existing_section_context = state.get("section_context") or ""
    web_lines = [
        f"[Web: {s['domain']}]\nTitle: {s['title']}\n{s['content']}"
        for s in snippets
    ]
    web_context = "\n\n".join(web_lines)

    if existing_section_context:
        combined = existing_section_context + "\n\n---\n\n" + web_context
    else:
        combined = web_context

    existing_snippets = list(state.get("web_snippets") or [])
    logger.info(
        "web_augment_node: added %d web snippets (total web_calls_used=%d)",
        len(snippets), web_calls_used + 1,
    )
    return {
        "section_context": combined,
        "web_snippets": existing_snippets + snippets,
        "web_calls_used": web_calls_used + 1,
        "retry_attempted": True,  # prevents a second confidence-gate retry
    }
```

### Step 5. [Runtime] Modify _route_after_confidence_gate to include web_augment branch

Modify the existing `_route_after_confidence_gate` function:

```python
def _route_after_confidence_gate(state: ChatState) -> str:
    confidence = state.get("confidence", "low")
    retry_attempted = state.get("retry_attempted", False)
    web_enabled = state.get("web_enabled", False)
    web_calls_used = state.get("web_calls_used", 0)

    if confidence == "low" and not retry_attempted:
        # Check web augmentation first if enabled and within rate limit
        if web_enabled and web_calls_used < 3:
            logger.info("confidence_gate_node: low confidence + web_enabled -- routing to web_augment_node")
            return "web_augment_node"
        # Fall back to local augment_node
        logger.info("confidence_gate_node: low confidence -- routing to augment_node")
        return "augment_node"
    logger.info("confidence_gate_node: confidence=%s -- routing to END", confidence)
    return END
```

Note: After `web_augment_node` runs, it sets `retry_attempted=True` so the next pass through `confidence_gate_node` routes to END regardless of confidence. This guarantees at most one web augmentation per conversation turn.

The graph topology after this change:

```
confidence_gate_node
  → (low + web_enabled + calls < 3) → web_augment_node → synthesize_node → confidence_gate_node → END
  → (low + no web) → augment_node → synthesize_node → confidence_gate_node → END

```

if [ "$HTTP" != "200" ]; then
  echo "FAIL: GET /settings/web-search returned ${HTTP}"
  cat "${RESULT}"
  rm -f "${RESULT}"
  exit 1
fi
HAS_PROVIDER=$(python3 -c "import json,sys; print('yes' if 'provider' in json.load(sys.stdin) else 'no')" < "${RESULT}")
rm -f "${RESULT}"
if [ "${HAS_PROVIDER}" != "yes" ]; then
  echo "FAIL: /settings/web-search missing 'provider' field"
  exit 1
fi

# 3. POST /qa with web_enabled=false must return SSE stream with done event
TMPFILE=$(mktemp)
HTTP=$(curl -s -o "${TMPFILE}" -w "%{http_code}" \
  -X POST "${BASE}/qa" \
  -H "Content-Type: application/json" \
  -d '{"question": "What is asyncio?", "scope": "all", "web_enabled": false}')

if [ "$HTTP" != "200" ]; then
  echo "FAIL: POST /qa returned ${HTTP}"
  cat "${TMPFILE}"
  rm -f "${TMPFILE}"
  exit 1
fi

HAS_DONE=$(grep -c '"done":true' "${TMPFILE}" || true)
rm -f "${TMPFILE}"

if [ "${HAS_DONE}" -lt 1 ]; then
  echo "FAIL: SSE stream missing done event"
  exit 1
fi

echo "PASS: S142 smoke test -- GET /settings/web-search and POST /qa with web_enabled=false both pass"

## Risk Flags

**Risk 1: duckduckgo-search package availability.**
The `duckduckgo_search` package may not be in the backend venv. Check before implementing: run `uv run python -c "import duckduckgo_search"`. If absent, add via `cd backend && uv add duckduckgo-search`. For provider=none (the default and the only provider tested in CI), no external package is needed -- none path is always safe.

**Risk 2: TypedDict extension of ChatState breaks LangGraph.**
LangGraph's StateGraph requires that all TypedDict keys have known types at graph compile time. Adding `web_enabled`, `web_calls_used`, and `web_snippets` to `ChatState` requires updating EVERY place that constructs the initial state (currently `stream_answer()` in `qa.py`) to include these new keys. Missing keys in initial_state cause LangGraph to raise a KeyError at runtime. Mitigation: grep for all `initial_state` dicts and add the three new fields to each.
Affected file: `backend/app/services/qa.py` (one occurrence).

**Risk 3: _route_after_confidence_gate conditional edge map mismatch.**
`build_chat_graph()` uses an explicit `{node_name: node_name}` dict for conditional edges. Adding `web_augment_node` to the routing must also add it to this dict. Missing the dict entry causes LangGraph to raise a ValueError at compile time. Mitigation: update both the function and the edge map together in the same edit.

**Risk 4: Chat.tsx TypeScript strict mode.**
The `Citation` interface extension with `version_mismatch?: boolean` must be handled carefully: existing code renders citations without checking for `version_mismatch`. The optional field (`?`) ensures backward compatibility. Run `npx tsc --noEmit` before committing.

**Risk 5: Web search privacy -- LLM may echo web content into citations.**
The LLM is prompted to label web sources as `[Web: domain.com]`. The `_split_response` parser in `qa.py` extracts citations from the LLM JSON block. A citation with `document_title = "[Web: docs.python.org]"` would be passed to `_store_qa()` and written to `QAHistoryModel`. This is acceptable: the citation metadata (URL label, section heading, excerpt) is not the same as the full web snippet content (500 chars). The raw `web_snippets` list is never written to DB. Document this distinction explicitly.

**Risk 6: Confidence threshold is `'low'` string, not a float.**
The story description says "confidence < 0.8" but the codebase uses string values `'high'`, `'medium'`, `'low'`. The `web_augment_node` must check `confidence == 'low'`, not a float comparison. The current codebase (confidence_gate_node, augment_node) uses the string form -- this plan follows the same pattern.

**Risk 7: web_calls_used counter reset per conversation.**
The counter resets to 0 when `clearConversation()` is called in `Chat.tsx`. It is also reset to 0 in `initial_state` for every new `sendMessage()` call (each message is a new graph invocation with fresh state). The frontend counter is cumulative for the UI display only -- it reflects the `web_calls_used` value returned in the done SSE event. Reset `webCallsUsed` state to 0 inside `clearConversation()`.

**Risk 8: Globe icon import.**
`lucide-react` version already imported in `Chat.tsx` may or may not include `Globe`. Check with `grep -r "Globe" frontend/src/` -- if absent, use `Globe2` or `Search` as a substitute. Both are available in lucide-react v0.3+.
