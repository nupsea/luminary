# S148 — Citation Deep-Links: Chunk-to-Location Metadata in Chat Answers

## Goal

Add `SourceCitation` TypedDict to `app/types.py`, collect deduplicated citations from retrieved chunks in `synthesize_node`, emit them as a `sources` SSE event after streaming, and wire the frontend to display clickable citation chips in Chat that navigate to the DocumentReader at the exact section and PDF page.

## Layers Affected

- **Types** — new `SourceCitation` TypedDict in `backend/app/types.py`
- **Runtime** — `synthesize_node` in `backend/app/runtime/chat_graph.py` gains citation collection and DB lookup
- **Service** — `stream_answer` in `backend/app/services/qa.py` gains `sources` SSE event emission
- **API** — no new endpoints; the existing `POST /qa` SSE stream gains a new event type
- **Frontend** — `Chat.tsx` receives and renders citation chips; `DocumentReader.tsx` and `PDFViewer.tsx` accept and act on initial page param; `Learning.tsx` reads `doc` and `page` URL params

```python
async def _fetch_section_ids_and_pages_for_chunks(
    chunk_ids: list[str],
) -> dict[str, tuple[str | None, int | None]]:
    """Return {chunk_id: (section_id, pdf_page_number)} for the given chunk_ids.

    Used by synthesize_node to build SourceCitation entries from retrieved chunks.
    Returns {} on any DB error (non-fatal).
    """
    if not chunk_ids:
        return {}
    try:
        from app.models import ChunkModel  # noqa: PLC0415
        async with get_session_factory()() as session:
            rows = await session.execute(
                select(ChunkModel.id, ChunkModel.section_id, ChunkModel.pdf_page_number)
                .where(ChunkModel.id.in_(chunk_ids))
            )
            return {row.id: (row.section_id, row.pdf_page_number) for row in rows}
    except Exception:
        logger.warning("_fetch_section_ids_and_pages_for_chunks: DB lookup failed", exc_info=True)
        return {}
```

**2c. Modify `synthesize_node` to collect and return citations.**

At the end of `synthesize_node`, just before `return {"_llm_prompt": prompt, "_system_prompt": system_prompt}`, add citation collection:

```python
    # Collect SourceCitations from context chunks for post-stream emission.
    # Deduplicate by section_id (first occurrence wins); fall back to chunk_id when
    # section_id is None (each chunk with no section gets its own citation entry).
    citations_out: list[dict] = []
    if chunks_dicts:
        chunk_ids = [c["chunk_id"] for c in chunks_dicts if c.get("chunk_id")]
        chunk_meta = await _fetch_section_ids_and_pages_for_chunks(chunk_ids)
        doc_titles_map = await _fetch_doc_titles_for_chunks(chunks_dicts)

        seen_keys: set[str] = set()
        for c in chunks_dicts:
            cid = c.get("chunk_id", "")
            meta = chunk_meta.get(cid, (None, None))
            section_id, pdf_page = meta
            doc_id = c.get("document_id", "")
            doc_title = doc_titles_map.get(doc_id, "")
            section_heading = c.get("section_heading", "")

            # Dedup key: section_id when present, else chunk_id (unique per chunk)
            dedup_key = section_id if section_id else cid
            if dedup_key in seen_keys:
                continue
            seen_keys.add(dedup_key)

            citations_out.append({
                "chunk_id": cid,
                "document_id": doc_id,
                "document_title": doc_title,
                "section_id": section_id,
                "section_heading": section_heading,
                "pdf_page_number": pdf_page,
            })

    return {
        "_llm_prompt": prompt,
        "_system_prompt": system_prompt,
        "citations": citations_out,
    }
```

Note: `citations` field already exists in `ChatState` (line 72 of types.py). The `initial_state["citations"]` is already initialized to `[]` in `qa.py` (line 323). This step simply populates it from the runtime node.

## Risk Flags

- **Risk**: `synthesize_node` is a pure prompt-preparation node by design (the "true streaming" refactor in S81). Adding two async DB calls (`_fetch_section_ids_and_pages_for_chunks` and `_fetch_doc_titles_for_chunks`) increases its latency. Mitigation: both calls are batched (single query each) and non-fatal — wrapped in try/except, returning empty on failure. This adds ~2 DB round-trips but no LLM calls; acceptable for a sync SQLite DB on localhost.

- **Risk**: The existing `citations` field in `ChatState` (line 72) is already used by the LLM-extracted citation system (`_split_response`, `_enrich_citation_titles`). Using the same field name for SourceCitations would collide. Mitigation: the `done` event payload uses `source_citations` (new key) for SourceCitations from chunks, and retains `citations` for LLM-extracted prose citations. The `synthesize_node` writes SourceCitations to `state["citations"]` ONLY when it sets `_llm_prompt` (i.e., the LLM path). When the LLM runs in `stream_answer`, the LLM-extracted citations overwrite `result.get("citations")`. To avoid collision: rename the synthesize_node output to `state["source_citations"]` and add `source_citations: list[dict]` as a new field in `ChatState`. This requires updating `initial_state` in `qa.py` and the `_make_state` helper in tests.

  **Decision**: Use `source_citations` as the new `ChatState` field name throughout. Do NOT reuse the existing `citations` field. This avoids overwriting LLM-extracted citations with chunk-derived ones.

- **Risk**: `Learning.tsx` currently uses Zustand `activeDocumentId` to know which document to show. Navigating to `/learning?doc=<id>` sets the param but Learning doesn't read it. Mitigation: add `useEffect` that reads `doc` param and calls `setActiveDocument`. Must clear the param after reading to avoid it persisting through the Back button flow.

- **Risk**: `PDFViewer` is lazy-mounted in DocumentReader (only rendered when `pdfViewVisited` is true). If the user arrives via a citation with `page` param but has never opened the PDF view, the tab must be auto-switched. Mitigation: the `useEffect` in DocumentReader that fires on `initialPage` sets both `leftTab("pdfview")` and `setPdfViewVisited(true)`.

- **Risk**: `synthesize_node` is called twice in the retry loop (via `augment_node → synthesize_node`). On the second call, `chunks_dicts` may contain merged chunks from the augment. The citation list will be re-collected on the retry. This is correct — the final citations in state will reflect all context chunks used, which is the desired behavior.

- **Risk**: `SectionModel` is not currently imported in `chat_graph.py`. It is already in `models.py`. Add to the existing import line.

- **Risk**: `_fetch_section_headings_for_ids` helper defined in Step 2b is actually unnecessary — `section_heading` is already stored in the chunk dict from `search_node` (as `c.get("section_heading", "")`). The heading from the chunk dict is correct and does not require an extra DB lookup. Remove Step 2b entirely; use `c.get("section_heading", "")` directly.

- **Risk**: TypeScript `tsc` will fail if `source_citations` is referenced without being in the `ChatMessage` interface. Ensure the interface update in Step 5 and the `setMessages` call in Step 6 are consistent.
