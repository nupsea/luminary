# S155 — Source-grounded answer panel on flashcard review errors

## Goal

When a learner rates a flashcard "Again" (1) or "Hard" (2) during a study session, lazily fetch and display a collapsible SourceContextPanel showing the original document passage, attribution line, and a deep-link back to the reader.

---

## Layers Affected

- **API** — new GET /flashcards/{id}/source-context endpoint in `backend/app/routers/flashcards.py`
- **Service** — no new service file needed; DB join logic is simple enough to live in the router (following the pattern of other thin flashcard endpoints that query directly via SQLAlchemy)
- **Types (Pydantic response model)** — new `SourceContextResponse` Pydantic model added in the router file
- **Frontend (StudySession.tsx)** — new `SourceContextPanel` component, lazy fetch after rating=1 or rating=2, session-level dismissed-set to prevent reappearance

## Key Implementation Details

### Backend: Join path

`FlashcardModel` does NOT have a direct `section_id` column. The join chain is:

```
FlashcardModel.chunk_id -> ChunkModel.section_id -> SectionModel.id -> DocumentModel.id
```

For the 404 cases:
- 404 when `flashcard.chunk_id` is null (no chunk link)
- 404 when `ChunkModel.section_id` is null (chunk not assigned to a section)
- 404 when SectionModel row not found for that section_id

`pdf_page_number` comes from `ChunkModel.pdf_page_number` (set during PDF ingestion per S146 — nullable for non-PDF documents).

`section_preview` is `SectionModel.preview[:400]` (the preview field already exists on SectionModel, truncated client-side if needed, but the story says max 400 chars so we truncate server-side to keep response small).

`section_heading` comes from `SectionModel.heading`.

`document_title` comes from `DocumentModel.title`.

`section_id` in the response is `SectionModel.id` (for the frontend deep-link).

### Backend: Route ordering

The new endpoint `GET /flashcards/{card_id}/source-context` is a parameterized route on card_id. It must NOT conflict with existing routes. Currently the router has:
- `GET /flashcards/{document_id}/export/csv`
- `GET /flashcards/{document_id}` (list)
- `PUT /flashcards/{card_id}`
- `DELETE /flashcards/{card_id}`
- `POST /flashcards/{card_id}/review`

The new route `GET /flashcards/{card_id}/source-context` follows the same pattern as `POST /flashcards/{card_id}/review`. FastAPI matches the literal segment "source-context" as a sub-path after the card_id wildcard, so no ordering conflict. Register it after the `/audit/` and `/cloze/` routes but before `/{document_id}` to be safe (though it has a distinct suffix so it would not collide regardless).

### Frontend: SourceContextPanel

A new component `SourceContextPanel` inside `StudySession.tsx` (not a separate file, following the existing `SourcePanel` pattern in the same file).

State additions to `StudySession`:
- `sourceContext: SourceContext | null` — the fetched context for the current card (null = not fetched or 404)
- `sourceContextDismissed: Set<string>` — card IDs dismissed this session (persisted in a `useRef` to avoid re-renders)

1. `submitReview` is called immediately (fire and forget — do not await before posting)

The existing `SourcePanel` (S138, web references) renders after "again" only. S155 renders `SourceContextPanel` after "again" OR "hard". The two panels are independent.

The Continue button in the "again" flow already exists (`advanceCard()`). For "hard" the card currently advances immediately. S155 must change "hard" to pause (same as "again") when `sourceContext` is available, showing the panel with a "Continue" button. If 404 (no context), "hard" still advances immediately.

### Step 2 — [API] Add GET /flashcards/{card_id}/source-context endpoint

In `backend/app/routers/flashcards.py`, add the new endpoint. The route must be registered AFTER the explicit-prefix routes (`/generate`, `/from-gaps`, `/cloze/{section_id}`, `/audit/{document_id}`, `/entity-pairs`, `/generate-from-graph`, `/generate-technical`, `/create-trace`, `/document/{document_id}`) and BEFORE `/{document_id}` (or immediately before the `/{card_id}/review` endpoint for clarity).

The implementation:

```python
@router.get("/{card_id}/source-context", response_model=SourceContextResponse)
async def get_source_context(
    card_id: str,
    session: AsyncSession = Depends(get_db),
) -> SourceContextResponse:
    """Return the source passage for a flashcard (for SourceContextPanel on Again/Hard).

    Join chain: FlashcardModel.chunk_id -> ChunkModel.section_id -> SectionModel -> DocumentModel.
    Returns 404 when:
      - flashcard not found
      - flashcard.chunk_id is null
      - ChunkModel.section_id is null (chunk not section-assigned)
      - SectionModel row not found for that section_id
    """
    # Fetch flashcard
    fc_result = await session.execute(
        select(FlashcardModel).where(FlashcardModel.id == card_id)
    )
    card = fc_result.scalar_one_or_none()
    if card is None:
        raise HTTPException(status_code=404, detail="Flashcard not found")
    if card.chunk_id is None:
        raise HTTPException(status_code=404, detail="Flashcard has no source chunk")

    # Fetch chunk to get section_id
    chunk_result = await session.execute(
        select(ChunkModel).where(ChunkModel.id == card.chunk_id)
    )
    chunk = chunk_result.scalar_one_or_none()
    if chunk is None or chunk.section_id is None:
        raise HTTPException(status_code=404, detail="No section found for this flashcard")

    # Fetch section
    sec_result = await session.execute(
        select(SectionModel).where(SectionModel.id == chunk.section_id)
    )
    section = sec_result.scalar_one_or_none()
    if section is None:
        raise HTTPException(status_code=404, detail="Section not found")

    # Fetch document
    doc_result = await session.execute(
        select(DocumentModel).where(DocumentModel.id == section.document_id)
    )
    doc = doc_result.scalar_one_or_none()
    document_title = doc.title if doc else ""

    preview = (section.preview or "")[:400]

    logger.info(
        "source-context: card=%s section=%s", card_id, chunk.section_id
    )
    return SourceContextResponse(
        section_heading=section.heading,
        section_preview=preview,
        document_title=document_title,
        pdf_page_number=chunk.pdf_page_number,
        section_id=chunk.section_id,
        document_id=section.document_id,
    )
```

Note: `pdf_page_number` comes from `ChunkModel.pdf_page_number` (nullable, added in S146). Use the chunk's page number (first chunk of the section approximation is acceptable since we use the flashcard's specific chunk).

### Step 8 — [Frontend/Logic] Modify handleRate to lazy-fetch source context

Current `handleRate` logic (summarized):
- Posts review immediately
- For "again": sets `lastRating = "again"`, does NOT advance card
- For "hard"/"good"/"easy": advances card immediately

New logic:

```ts
async function handleRate(rating: Rating) {
  if (!sessionId || isRating) return
  const card = queue[currentIndex]
  if (!card) return

  setIsRating(true)

  try {
    await submitReview(card.id, rating, sessionId)  // immediate POST (unchanged)
    const isCorrect = rating !== "again"
    // ... update reviewed/correct/nextReviewDate (unchanged) ...

    setLastRating(rating)

    if (rating === "again" || rating === "hard") {
      // Lazy-fetch source context if not dismissed this session
      if (!dismissedSourceContextIds.current.has(card.id)) {
        setSourceContextLoading(true)
        const ctx = await fetchSourceContext(card.id)
        setSourceContextLoading(false)
        if (ctx !== null) {
          setSourceContext(ctx)
          // Panel shown; do NOT advance card — user must click Continue
          return
        }
      }
      // 404 or dismissed: fall through to advance
    }

    // Advance card immediately for: good/easy, or again/hard with no context (404/dismissed)
    const nextIndex = currentIndex + 1
    if (rating === "again") {
      // For "again" with no source context (404): advance
      if (nextIndex >= queue.length) {
        await endSession(sessionId)
        setSessionState("complete")
      } else {
        setCurrentIndex(nextIndex)
        setShowAnswer(false)
        setLastRating(null)
      }
    } else {
      // hard/good/easy: advance immediately
      if (nextIndex >= queue.length) {
        await endSession(sessionId)
        setSessionState("complete")
      } else {
        setCurrentIndex(nextIndex)
        setShowAnswer(false)
        setLastRating(null)
      }
    }
  } finally {
    setIsRating(false)
  }
}
```

Key invariant: `submitReview` is always called BEFORE the source context fetch. The await on `submitReview` ensures the rating is recorded before the panel appears (AC#6).

## Risk Flags

- **Risk: Route ordering conflict** — `GET /flashcards/{card_id}/source-context` has the same pattern as `POST /flashcards/{card_id}/review`. FastAPI differentiates on HTTP method (GET vs POST), so no conflict. The route suffix "source-context" prevents it from matching against `/{document_id}` patterns. However, the NOTE in flashcards.py says to register `/audit/` and `/cloze/` BEFORE `/{document_id}`. The new route must similarly be placed before the `GET /{document_id}` route. (Mitigation: register it near the existing `POST /{card_id}/review` endpoint, which is already after the document-level routes.)

- **Risk: SectionModel.preview may be empty string** — Many sections have a non-empty `preview` field (set during ingestion), but for some edge cases (e.g. very short sections or code-only sections) it may be "". The SourceContextPanel must handle empty preview gracefully with a fallback message ("No preview available"). This is covered in Step 6 above.

- **Risk: S138 SourcePanel interaction** — The existing S138 `SourcePanel` (web references) is shown after "again" ratings. S155 introduces a second panel for "again" AND "hard". Showing two stacked panels may overwhelm the UI. The plan prioritizes `SourceContextPanel` (new, S155) over `SourcePanel` (S138 web refs) when source context is available; S138 SourcePanel only shows as fallback when 404. This is a UX trade-off that avoids UI clutter while keeping S138 behavior for cards with no source passage.

- **Risk: "hard" rating behavior change** — Currently "hard" advances immediately. S155 changes it to pause if source context is available. This is intentional but changes the flow that prior stories assumed. AC#7 explicitly requests this. The card advance timer mentioned in the story description ("Card advance happens on timer regardless") refers to non-blocking behavior — the spec says the panel is shown but the user can dismiss or continue. Our implementation uses an explicit Continue button, not a timer. This is consistent with the existing "again" flow.

- **Risk: document_id missing from SourceContextResponse** — The story spec lists {section_heading, section_preview, document_title, pdf_page_number, section_id} but does NOT list document_id. However, the frontend needs document_id to build the /learning?doc=... URL. We add document_id to the response to avoid a second lookup. The story says "Open in reader link using S148 URL routing (/learning?doc=...&section=...&page=...)" — this requires doc id. We add it.

- **Risk: No FlashcardModel.section_id column** — The story description says "Join: FlashcardModel.section_id -> SectionModel" but FlashcardModel has no `section_id` column (confirmed in models.py). The actual join is via `chunk_id -> ChunkModel.section_id`. The 404 conditions cover both chunk_id null and chunk.section_id null.

- **Risk: Test isolation** — Tests must use the `test_db` fixture from `test_flashcards.py` pattern (in-memory SQLite + monkeypatched engine). Import stubs from `stubs.py`. No real DB or LLM calls.

## Notes on Invariant Compliance

1. **uv only** — no new pip dependencies (uses existing SQLAlchemy, FastAPI, pydantic)
2. **Layer order** — API router imports Types (Pydantic models) and SQLAlchemy models directly; no reverse imports
3. **LiteLLM** — this endpoint makes no LLM calls; invariant not applicable
4. **Pydantic at every boundary** — `SourceContextResponse(BaseModel)` is the typed response; no raw dict return
5. **Tests required** — 6 pytest tests covering the endpoint
6. **Three frontend states** — Loading: spinner skeleton; Error/404: silent omission (per spec AC11); Empty: not applicable (panel only appears when content available). For the SourceContextPanel itself: loading skeleton shown while fetching (Step 11), 404 is silently omitted (no panel), content panel is the success state.
