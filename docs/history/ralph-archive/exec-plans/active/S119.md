# S119 — Audio ingestion via faster-whisper

## Goal

Add MP3/M4A/WAV transcription to the ingestion pipeline using faster-whisper so audio
files flow through the existing embed-NER-store pipeline as timestamped chunks, and appear
in the library with an audio icon and duration badge.

### Step 4 — [Service] Create AudioTranscriber

Create `backend/app/services/audio_transcriber.py`:

```python
"""AudioTranscriber — wraps faster_whisper.WhisperModel.

Returns (segments, duration_seconds) where segments is a list of
{"start": float, "end": float, "text": str} dicts.
"""
import logging
from functools import lru_cache
from pathlib import Path

logger = logging.getLogger(__name__)


class AudioTranscriber:
    def __init__(self, model_size: str = "base") -> None:
        from faster_whisper import WhisperModel  # noqa: PLC0415
        self._model = WhisperModel(model_size, device="cpu", compute_type="int8")
        logger.info("AudioTranscriber: loaded model_size=%s", model_size)

    def transcribe(self, file_path: Path) -> tuple[list[dict], float]:
        """Return (segments, duration_seconds).

        segments: list of {"start": float, "end": float, "text": str}
        """
        segments_iter, info = self._model.transcribe(str(file_path), beam_size=1)
        result = [
            {"start": seg.start, "end": seg.end, "text": seg.text.strip()}
            for seg in segments_iter
            if seg.text.strip()
        ]
        logger.info(
            "AudioTranscriber: transcribed %d segments, duration=%.1fs",
            len(result),
            info.duration,
        )
        return result, info.duration


@lru_cache(maxsize=1)
def get_audio_transcriber() -> AudioTranscriber:
    from app.config import get_settings  # noqa: PLC0415
    return AudioTranscriber(model_size=get_settings().WHISPER_MODEL_SIZE)
```

Design decisions:
- `device="cpu"`, `compute_type="int8"` for laptop-friendly inference without GPU
- `faster_whisper` is imported lazily inside `__init__` so tests can stub the class
  without loading the C extension
- Returns a tuple `(list[dict], float)` so `transcribe_node` can write `audio_duration_seconds`
- `@lru_cache(maxsize=1)` singleton pattern is consistent with all other services in this codebase

**5f. Add _chunk_audio() pure helper:**

This is a pure function (no I/O per Belief #19), testable with plain `assert`:

```python
def _chunk_audio(
    segments: list[dict],
    doc_id: str,
    window_seconds: float = 60.0,
) -> list[dict]:
    """Group Whisper segments into ~60-second windows.

    Each returned dict has:
      id, document_id, text, index, start_time (float), end_time (float)
    """
    import uuid as _uuid  # noqa: PLC0415
    chunks: list[dict] = []
    bucket_texts: list[str] = []
    bucket_start: float = 0.0
    bucket_end: float = 0.0
    chunk_idx: int = 0

    def _flush(start: float, end: float, texts: list[str]) -> None:
        nonlocal chunk_idx
        if not texts:
            return
        chunks.append({
            "id": str(_uuid.uuid4()),
            "document_id": doc_id,
            "text": " ".join(texts),
            "index": chunk_idx,
            "start_time": start,
            "end_time": end,
        })
        chunk_idx += 1

    for seg in segments:
        if not bucket_texts:
            bucket_start = seg["start"]
        if seg["end"] - bucket_start >= window_seconds and bucket_texts:
            _flush(bucket_start, bucket_end, bucket_texts)
            bucket_texts = []
            bucket_start = seg["start"]
        bucket_texts.append(seg["text"])
        bucket_end = seg["end"]

    _flush(bucket_start, bucket_end, bucket_texts)
    return chunks
```

**5g. Add transcribe_node:**

```python
async def transcribe_node(state: IngestionState) -> IngestionState:
    """Transcribe audio files using faster-whisper.

    For non-audio content types this is a pass-through.
    For audio files: calls AudioTranscriber, builds parsed_document from segments,
    writes audio_duration_seconds to DocumentModel, stores pre-built _audio_chunks.
    """
    content_type = state.get("content_type")
    if content_type != "audio":
        return state

    doc_id = state["document_id"]
    await _update_stage(doc_id, "transcribing")
    logger.info("transcribe_node: start", extra={"doc_id": doc_id})

    try:
        from sqlalchemy import update as _update  # noqa: PLC0415

        from app.models import DocumentModel  # noqa: PLC0415
        from app.services.audio_transcriber import get_audio_transcriber  # noqa: PLC0415

        fp = Path(state["file_path"])
        transcriber = get_audio_transcriber()
        loop = asyncio.get_event_loop()
        # CPU-bound — run in thread pool to keep event loop free for status polls
        segments, duration = await loop.run_in_executor(
            None, transcriber.transcribe, fp
        )

        raw_text = " ".join(s["text"] for s in segments)
        audio_chunks = _chunk_audio(segments, doc_id)

        # Sections for section_summarize_node: one section per audio window
        sections = [
            {
                "heading": f"Segment {i + 1} ({c['start_time']:.0f}s-{c['end_time']:.0f}s)",
                "level": 1,
                "text": c["text"],
                "page_start": 0,
                "page_end": 0,
            }
            for i, c in enumerate(audio_chunks)
        ]

        parsed_document = {
            "title": fp.stem,
            "format": fp.suffix.lstrip("."),
            "pages": 0,
            "word_count": len(raw_text.split()),
            "sections": sections,
            "raw_text": raw_text,
        }

        # Persist duration to DocumentModel
        async with get_session_factory()() as session:
            await session.execute(
                _update(DocumentModel)
                .where(DocumentModel.id == doc_id)
                .values(
                    audio_duration_seconds=duration,
                    word_count=len(raw_text.split()),
                )
            )
            await session.commit()

        logger.info(
            "transcribe_node: done",
            extra={"doc_id": doc_id, "segments": len(segments), "duration": duration},
        )
        return {
            **state,
            "parsed_document": parsed_document,
            "audio_duration_seconds": duration,
            "_audio_chunks": audio_chunks,
            "status": "chunking",
        }
    except Exception as exc:
        logger.error("transcribe_node failed", exc_info=exc)
        return {**state, "status": "error", "error": str(exc)}
```

    builder.add_edge(START, "parse")
    builder.add_conditional_edges("parse", _route_on_status("classify"))

    builder.add_conditional_edges("transcribe", _route_on_status("chunk"))      # new

    builder.add_edge("keyword_index", "entity_extract")

export function formatDuration(seconds: number): string {
  const h = Math.floor(seconds / 3600)
  const m = Math.floor((seconds % 3600) / 60)
  if (h > 0) return `${h}h ${m}m`
  if (m > 0) return `${m}m`
  return `${Math.round(seconds)}s`
}

## Risk Flags

- **Risk: faster-whisper model download on first run** — The `base` model is ~150MB
  and downloads automatically on first use. CI tests never instantiate the real
  `AudioTranscriber` (the stub replaces it), so this is only a concern for manual
  end-to-end testing. Set `WHISPER_MODEL_SIZE=tiny` for quicker first-run (~75MB).
  Mitigation: the lazy `faster_whisper` import inside `__init__` ensures tests that
  import the module without instantiating `AudioTranscriber` pay no download cost.

- **Risk: CPU inference blocking the event loop** — `transcribe_node` runs
  `transcriber.transcribe()` in `asyncio.get_event_loop().run_in_executor(None, ...)`.
  Without this, the synchronous Whisper call blocks the event loop for minutes on
  a 1-hour audio file, preventing status poll responses from returning. Pattern
  matches `embed_node` which uses the same executor pattern.

- **Risk: _audio_chunks absent from IngestionState TypedDict** — If `_audio_chunks`
  is not added to `IngestionState`, accessing `state.get("_audio_chunks")` will raise
  a type error in strict TypedDict usage. It is included in the TypedDict definition
  above and initialized to `None` in `run_ingestion()` initial state.

- **Risk: parse_node raises ValueError before transcribe_node can run** —
  `DocumentParser.parse()` does not accept audio formats and raises `ValueError`.
  The audio bypass at the top of `parse_node` (Step 5e) is required. Without it,
  `parse_node` returns `status="error"` and `error_finalize_node` runs instead of
  the audio path.

- **Risk: TypeScript Record completeness for ContentType** — `CONTENT_TYPE_ICONS`,
  `CONTENT_TYPE_BADGE`, `LABELS`, and `STATUS_VARIANTS` are typed as
  `Record<ContentType, ...>`. After adding `"audio"` to the union, TypeScript will
  flag missing keys. All four records in the four frontend files must be updated
  atomically or `tsc --noEmit` will fail.

- **Risk: format field for audio documents** — The `format` field on `DocumentModel`
  is set to `ext` (e.g. `"mp3"`) by the API router. `DocumentParser.parse(fp, fmt)`
  is called inside `parse_node` for non-audio documents; the audio bypass ensures
  this never happens for audio files. The `format` field is stored as `"mp3"`/`"m4a"`/
  `"wav"` and is cosmetic only.

- **Risk: section_summarize_node runs on audio documents** — This is intentional and
  beneficial. The sections created by `transcribe_node` (one per 60s window) are
  valid inputs to the section summarizer, enabling Chat to answer questions with
  section-level context. No changes needed here.
