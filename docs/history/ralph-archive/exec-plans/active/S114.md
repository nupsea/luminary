# S114 -- Adaptive Difficulty Signals: Surface Struggling Cards with Source Section Links

## Goal

Expose flashcards rated "again" three or more times in the last 14 days via a dedicated
`GET /study/struggling` endpoint and a Struggling panel in the Study tab that links each card
back to its source section in DocumentReader.

## Implementation Steps

### Implemented in partial commit (no action needed)

Steps 1-5 are complete. Read the existing implementations before verifying:

1. [Service] `FSRSService.get_struggling_cards(session, document_id, again_threshold, days)`
   in `fsrs_service.py`. Correct query: count `ReviewEventModel` where `rating='again'`
   and `reviewed_at >= (now - timedelta(days=days))`, group by `flashcard_id`, filter where
   count >= `again_threshold`. Resolve `source_section_id` via `OUTER JOIN chunks`.

2. [API] `GET /study/struggling` in `study.py`. Uses `Query` for typed `again_threshold`
   and `days` params (ge=1 and le=365 constraints). Returns `list[StrugglingCardItem]`.
   `StrugglingCardItem` has `flashcard_id`, `document_id`, `question`, `again_count`,
   `source_section_id`.

3. [Frontend] `StrugglingPanel` in `Study.tsx`. Uses `useQuery(["struggling", documentId])`.
   Loading state: 2 animated `bg-muted` skeleton rows. Error state: red border div with
   `AlertCircle` icon. Empty state: "No struggling cards in the last 14 days." paragraph.
   Card row: truncated question, `{again_count}x Again` red badge, "Re-read source" button
   (only rendered when `source_section_id` is not null).

4. [Frontend] Navigation. `handleReread` sets `activeDocumentId` via Zustand, then navigates
   to `/?section_id=<id>` using `useNavigate`. `Learning.tsx` reads `section_id` from search
   params and passes it to `DocumentReader` as `initialSectionId`. `DocumentReader` has a
   `useEffect` that calls `scrollIntoView` on the element with `data-section-id` matching
   `initialSectionId`.

5. [Test] `test_struggling_cards.py`: 6 tests -- three agains returns card, two agains does
   not, agains outside 14-day window excluded, good ratings not counted, endpoint returns
   correct item, endpoint returns empty when no struggles.

## Risk Flags

- **Risk**: `ruff --fix` auto-apply changes UP045 (`Optional[X]` to `X | None`) throughout
  `book_parser.py`. This changes 10+ type annotations. The change is mechanical and safe for
  Python 3.10+, and the project requires Python 3.13. Mitigation: run the full test suite
  after applying the ruff fixes to confirm book_parser functionality is unchanged.
  (`uv run pytest tests/test_book_parser.py -v` if that test file exists, or
  `uv run pytest -x` to run all tests.)

- **Risk**: E741 renames (`l` to `line` / `ln`) in book_parser.py and test_book_parser.py
  must be consistent -- the variable is used in the same comprehension it is defined in, so
  there is no cross-reference risk. Mitigation: apply rename in-place with the Edit tool;
  verify the renamed variable appears only in its enclosing comprehension expression.

- **Risk**: The `StrugglingPanel` empty state currently renders `null` (nothing) when
  `cards.length === 0`. This satisfies the "no struggling cards" AC (AC-10 in the plan) but
  means the section heading is also hidden. The AC in the PRD says "Struggling panel separate
  from Weak Areas" -- verified: it IS rendered separately when cards exist. No change needed.

- **Risk**: `ReviewEventModel` rows in tests are created without a matching `StudySessionModel`
  row. SQLite does not enforce FK constraints without PRAGMA, so the existing tests use a
  synthetic UUID for `session_id` and this is correct. No change needed.
