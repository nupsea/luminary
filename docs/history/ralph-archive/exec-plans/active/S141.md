## Kuzu Schema Changes

### New edge table: `SAME_CONCEPT`

```
CREATE REL TABLE IF NOT EXISTS SAME_CONCEPT(
  FROM Entity TO Entity,
  source_doc_id STRING,
  target_doc_id STRING,
  confidence FLOAT,
  contradiction BOOLEAN,
  contradiction_note STRING,
  prefer_source STRING
)
```

Properties:
- `source_doc_id` / `target_doc_id` -- the two documents where these entity nodes live
- `confidence` -- float 0.0..1.0 from the matching rule (exact=1.0, substring=0.8, token=0.6)
- `contradiction` -- boolean; default false; set to true when LLM says contradictory
- `contradiction_note` -- one-sentence description of the contradiction (or empty string)
- `prefer_source` -- `"a"` (source doc) or `"b"` (target doc) or empty string; set when
  `contradiction=True` and LLM indicates which source is likely more current

IMPORTANT: Kuzu does not support parameterized relation type names in Cypher. Like PREREQUISITE_OF,
all SAME_CONCEPT create/match statements must be written explicitly (no string interpolation for
the relation type).

Idempotency: before creating a SAME_CONCEPT edge between two entity IDs, check whether one
already exists in either direction (Entity A -> B or B -> A). If found, skip creation.

**4b.** Add `add_same_concept_edge()` method:

```python
def add_same_concept_edge(
    self,
    entity_id_a: str,
    entity_id_b: str,
    source_doc_id: str,
    target_doc_id: str,
    confidence: float,
    contradiction: bool = False,
    contradiction_note: str = "",
    prefer_source: str = "",
) -> None:
    """Create a SAME_CONCEPT edge between two Entity nodes from different documents.

    Idempotent: checks both directions (a->b and b->a) before creating.
    If an existing edge is found, updates its contradiction fields if contradiction=True.
    """
    # Check a->b direction
    result_ab = self._conn.execute(
        "MATCH (a:Entity {id: $aid})-[r:SAME_CONCEPT]->(b:Entity {id: $bid}) RETURN r.confidence",
        {"aid": entity_id_a, "bid": entity_id_b},
    )
    # Check b->a direction
    result_ba = self._conn.execute(
        "MATCH (a:Entity {id: $bid})-[r:SAME_CONCEPT]->(b:Entity {id: $aid}) RETURN r.confidence",
        {"aid": entity_id_a, "bid": entity_id_b},
    )
    if result_ab.has_next() or result_ba.has_next():
        # Edge exists -- update contradiction fields if this call brings new info
        if contradiction:
            try:
                self._conn.execute(
                    "MATCH (a:Entity {id: $aid})-[r:SAME_CONCEPT]->(b:Entity {id: $bid})"
                    " SET r.contradiction = $c, r.contradiction_note = $cn,"
                    " r.prefer_source = $ps",
                    {
                        "aid": entity_id_a, "bid": entity_id_b,
                        "c": contradiction, "cn": contradiction_note, "ps": prefer_source,
                    },
                )
            except Exception:
                logger.debug("add_same_concept_edge: SET contradiction failed", exc_info=True)
        return
    # Create new edge
    try:
        self._conn.execute(
            "MATCH (a:Entity {id: $aid}), (b:Entity {id: $bid})"
            " CREATE (a)-[:SAME_CONCEPT {"
            "source_doc_id: $sdid, target_doc_id: $tdid,"
            " confidence: $conf, contradiction: $c,"
            " contradiction_note: $cn, prefer_source: $ps"
            "}]->(b)",
            {
                "aid": entity_id_a, "bid": entity_id_b,
                "sdid": source_doc_id, "tdid": target_doc_id,
                "conf": confidence, "c": contradiction,
                "cn": contradiction_note, "ps": prefer_source,
            },
        )
    except Exception:
        logger.debug("add_same_concept_edge: CREATE failed", exc_info=True)
```

**4c.** Add `get_same_concept_edges()` method:

```python
def get_same_concept_edges(self) -> list[dict]:
    """Return all SAME_CONCEPT edges across the entire graph.

    Returns list of dicts:
        {entity_id_a, entity_id_b, name_a, name_b,
         source_doc_id, target_doc_id,
         confidence, contradiction, contradiction_note, prefer_source}
    Returns [] on any Kuzu error.
    """
    try:
        result = self._conn.execute(
            "MATCH (a:Entity)-[r:SAME_CONCEPT]->(b:Entity)"
            " RETURN a.id, b.id, a.name, b.name,"
            " r.source_doc_id, r.target_doc_id,"
            " r.confidence, r.contradiction,"
            " r.contradiction_note, r.prefer_source"
        )
        edges: list[dict] = []
        while result.has_next():
            row = result.get_next()
            edges.append({
                "entity_id_a": row[0],
                "entity_id_b": row[1],
                "name_a": row[2],
                "name_b": row[3],
                "source_doc_id": row[4],
                "target_doc_id": row[5],
                "confidence": float(row[6] or 0.0),
                "contradiction": bool(row[7]),
                "contradiction_note": row[8] or "",
                "prefer_source": row[9] or "",
            })
        return edges
    except Exception:
        logger.debug("get_same_concept_edges failed", exc_info=True)
        return []
```

**4d.** Add `get_concept_clusters()` method:

```python
def get_concept_clusters(self) -> list[dict]:
    """Return concept clusters: groups of Entity nodes linked by SAME_CONCEPT edges.

    Uses union-find on SAME_CONCEPT edge pairs to group entities into clusters.
    Returns list of cluster dicts:
        {concept_name, entity_ids, document_ids, has_contradiction, contradiction_note}
    """
    edges = self.get_same_concept_edges()
    if not edges:
        return []

    # Union-find
    parent: dict[str, str] = {}

    def find(x: str) -> str:
        if parent.setdefault(x, x) != x:
            parent[x] = find(parent[x])
        return parent[x]

    def union(x: str, y: str) -> None:
        parent[find(x)] = find(y)

    # Collect node metadata
    node_names: dict[str, str] = {}
    node_docs: dict[str, str] = {}  # entity_id -> source_doc_id

    for edge in edges:
        aid, bid = edge["entity_id_a"], edge["entity_id_b"]
        node_names[aid] = edge["name_a"]
        node_names[bid] = edge["name_b"]
        node_docs[aid] = edge["source_doc_id"]
        node_docs[bid] = edge["target_doc_id"]
        union(aid, bid)

    # Group by root
    groups: dict[str, list[str]] = {}
    for eid in node_names:
        root = find(eid)
        groups.setdefault(root, []).append(eid)

    # For each group, gather contradiction info
    edge_index: dict[tuple[str, str], dict] = {
        (e["entity_id_a"], e["entity_id_b"]): e for e in edges
    }

    clusters: list[dict] = []
    for root, eids in groups.items():
        has_contradiction = False
        contradiction_note = ""
        doc_ids: set[str] = set()
        for eid in eids:
            if eid in node_docs:
                doc_ids.add(node_docs[eid])
        for eid_a in eids:
            for eid_b in eids:
                e = edge_index.get((eid_a, eid_b))
                if e and e["contradiction"]:
                    has_contradiction = True
                    if not contradiction_note:
                        contradiction_note = e["contradiction_note"]
        # Use longest name as cluster concept_name
        concept_name = max((node_names.get(eid, "") for eid in eids), key=len)
        clusters.append({
            "concept_name": concept_name,
            "entity_ids": eids,
            "document_ids": list(doc_ids),
            "has_contradiction": has_contradiction,
            "contradiction_note": contradiction_note,
        })
    return clusters
```

### Step 5 -- [Service] Create `backend/app/services/concept_linker.py`

This is the core service. Follow the structure of `prereq_extractor.py` and `entity_disambiguator.py`.

**Module docstring and imports:**

```python
"""ConceptLinkerService: cross-document SAME_CONCEPT edge creation with contradiction detection.

Registered as enrichment job_type='concept_link'.
Compares CONCEPT and ALGORITHM entities in the newly ingested document against all existing
CONCEPT and ALGORITHM entities across all other documents.

Matching rules (same as EntityDisambiguator -- reuses the same functions):
  A. Exact stripped match (confidence=1.0)
  B. Substring containment -- longer wins (confidence=0.8)
  C. Token overlap >= 2 (confidence=0.6)

For each matched pair, runs a second LLM call to detect contradictions between the two
section summaries that most prominently mention each concept.

Contradiction prompt output JSON:
  {"has_contradiction": bool, "note": str, "prefer_source": "a" | "b"}

If has_contradiction=True, sets contradiction=True, contradiction_note, prefer_source on the edge.
"""
```

**Key methods:**

**`_parse_year(text: str) -> int | None`** -- pure function:
- Searches for patterns: `Copyright YYYY`, `Published YYYY`, `(c) YYYY`, `(C) YYYY`,
  `published in YYYY`, or a standalone 4-digit year 1900-2099 near the start of the text.
- Returns the first matched year as int, or None.
- Uses `re.search` with pattern `r'(?:Copyright|Published|published in|\(c\)|\(C\))\s+(\d{4})'`.
- Fallback: scan first 500 chars of text for any `\b(19|20)\d{2}\b` and return first match.

**`_find_summary_for_concept(concept_name: str, document_id: str, session) -> str | None`**:
- Queries `SectionSummaryModel` rows for the given document.
- Returns the content of the section summary whose content contains the most occurrences
  of `concept_name` (case-insensitive). Uses Python `str.lower().count()` after fetching all rows.
- Returns None if no section summaries exist for the document.

**`_detect_contradiction(concept_name: str, summary_a: str, summary_b: str) -> dict`** (async):
- Builds prompt: compares two passage excerpts about the same concept.
- Calls `litellm.acompletion` with system prompt exactly as specified in story description:
  `"Do these two passages make contradictory claims about the same concept? If yes, describe
  the contradiction in one sentence and which is likely more current. Output JSON:
  {\"has_contradiction\": bool, \"note\": str, \"prefer_source\": \"a\" or \"b\"}"`
- Parses JSON from response; if parse fails returns `{"has_contradiction": False, "note": "", "prefer_source": ""}`.
- Raises `litellm.ServiceUnavailableError` (propagates to caller).

**`_get_all_concepts_by_type(session) -> dict[str, list[dict]]`**:
- Queries `KuzuService` to get all Entity nodes of type CONCEPT or ALGORITHM across ALL documents.
- For each entity: returns `{id, name, document_id}`. Uses existing `get_entities_by_type_for_document`
  iterated over all document IDs from SQLite `DocumentModel`.
- Returns dict: `{"CONCEPT": [...], "ALGORITHM": [...]}` where each item is `{id, name, document_id}`.

**`link_for_document(document_id: str, session) -> int`** (async):
- Main logic:
  1. Get all CONCEPT and ALGORITHM entities for `document_id` from Kuzu.
  2. Get all CONCEPT and ALGORITHM entities for all OTHER documents from Kuzu.
  3. For each entity in the new doc, call `find_canonical(name, etype, [e["name"] for e in existing])`
     from `entity_disambiguator`. If the returned canonical differs from the input name, a match
     was found. Also detect confidence level:
     - Exact stripped match -> confidence 1.0
     - Substring match -> confidence 0.8
     - Token overlap >= 2 -> confidence 0.6
     Use `_compute_match_confidence(name_a, name_b) -> float | None` pure function.
  4. For each match pair `(entity_in_new_doc, entity_in_existing_doc)`:
     a. Check if SAME_CONCEPT edge already exists (call `graph_svc.add_same_concept_edge` which
        is idempotent -- but here we need to check to decide whether to do contradiction analysis).
     b. If no existing edge:
        - Find section summary snippets for each concept in its respective document.
        - If both summaries found, call `_detect_contradiction()`.
        - Call `graph_svc.add_same_concept_edge(...)` with contradiction data if applicable.
  5. Also look up and update `DocumentModel.publication_year` for this document if not yet set:
     - Scan first 2000 chars of the document's first chunk text for year patterns.
     - If found, update `DocumentModel.publication_year`.
  6. Return total edges created count.

**`_compute_match_confidence(name_a: str, name_b: str) -> float | None`** -- pure function:
- Uses `_strip_honorifics` from `entity_disambiguator`.
- Returns 1.0 for exact stripped match.
- Returns 0.8 if one stripped form is a substring of the other.
- Returns 0.6 if token overlap >= 2.
- Returns None if no match.

**`async def concept_link_handler(document_id: str, job_id: str) -> None`**:
- Top-level enrichment handler function.
- Creates a SQLAlchemy session, calls `link_for_document(document_id, session)`.
- Logs start/done with edge count.

```python
async def _fetch_contradiction_context(doc_ids: list[str]) -> str:
    """Fetch SAME_CONCEPT edges with contradiction=True for the given document set.

    Returns a formatted context block like:
      '[Contradictions] "dependency injection": Source A (2019) and Source B (2024)
       disagree: B prefers IoC containers; B preferred as more current.'
    Returns empty string if no contradictions exist or on any error.
    """
    from app.services.graph import get_graph_service  # noqa: PLC0415

    try:
        svc = get_graph_service()
        all_edges = svc.get_same_concept_edges()
        relevant = [
            e for e in all_edges
            if e["contradiction"]
            and (e["source_doc_id"] in doc_ids or e["target_doc_id"] in doc_ids)
        ]
        if not relevant:
            return ""
        lines: list[str] = ["[Cross-source contradictions detected:]"]
        for e in relevant[:3]:  # cap at 3 to avoid prompt bloat
            prefer = ""
            if e["prefer_source"] == "b":
                prefer = " (newer source preferred)"
            elif e["prefer_source"] == "a":
                prefer = " (older source preferred)"
            lines.append(
                f'- Concept "{e["name_a"]}": {e["contradiction_note"]}{prefer}'
            )
        return "\n".join(lines)
    except Exception:
        logger.debug("_fetch_contradiction_context failed", exc_info=True)
        return ""
```

## Enrichment Job Flow

```
Document ingested (run_ingestion completes)
  -> enrichment_enqueue_node runs
       -> Always enqueues concept_link job (status=pending)
       -> Also enqueues image_extract if PDF/EPUB

EnrichmentQueueWorker (polling loop, started in main.py lifespan)
  -> Picks up concept_link job
  -> Calls concept_link_handler(document_id, job_id)
       -> ConceptLinkerService.link_for_document(document_id, session)
            -> Reads CONCEPT/ALGORITHM entities for this doc from Kuzu
            -> Reads CONCEPT/ALGORITHM entities for ALL other docs from Kuzu
            -> For each match (rules A/B/C):
                 -> Fetch section summary snippets for both concepts
                 -> Call _detect_contradiction() via LiteLLM
                 -> Call graph_svc.add_same_concept_edge() (idempotent)
            -> Updates DocumentModel.publication_year if not yet set
  -> Job status -> done

Re-run trigger: when a new document is added, a new concept_link job is enqueued.
The handler compares the new doc's concepts against ALL existing docs.
This means older documents may gain SAME_CONCEPT edges pointing TO the new doc even though
their own concept_link jobs already ran. This is acceptable: the edge is created from the
new doc's job run (it is the "source" of discovery).
```

## Risk Flags

**Risk 1: Kuzu SAME_CONCEPT schema with boolean type (HIGH)**

Kuzu's Python binding may not support BOOLEAN column type in all versions. The existing codebase
uses `FLOAT`, `STRING`, `INT64` in Kuzu schemas. If `BOOLEAN` is not supported, use `INT64` (0/1)
instead.

Mitigation: In `_create_schema`, use `INT64` for `contradiction` and convert `bool -> int` in
`add_same_concept_edge()`. In `get_same_concept_edges()`, convert int back to bool with `bool(row[7])`.
This is already shown in the code above.

**Risk 2: concept_link job runs before entity extraction is complete (HIGH)**

The `enrichment_enqueue_node` in the ingestion graph runs AFTER `entity_extract_node` and
`section_summarize_node`. So the Kuzu entity nodes are already written when the concept_link
job is enqueued. However, the `EnrichmentQueueWorker._dispatch_pending()` is called as a
background task immediately. If the worker starts before the Kuzu write transaction completes,
it may not find any entities.

Mitigation: In `concept_link_handler`, if no CONCEPT/ALGORITHM entities are found for the
document, log a warning and return early (count=0) without failing. The job will still be
marked `done`. A manual re-run would be needed if this race condition occurs.

**Risk 3: O(N^2) entity comparison as library grows (MEDIUM)**

For each new document, `link_for_document` compares each of its CONCEPT/ALGORITHM entities
against all entities of those types across all other documents. With N documents, each having
K entities, this is O(N * K^2) comparisons.

Mitigation: the story scope is a personal knowledge assistant with tens to low hundreds of
documents. Performance is acceptable. For larger libraries, a future story could cache the
entity-name pool. For now: add a warning log if entity count exceeds 500.

**Risk 4: LLM contradiction call doubles cost per matched pair (MEDIUM)**

Each SAME_CONCEPT match triggers one additional LLM call. With many matched pairs, this adds
significant processing time.

Mitigation: cap contradiction detection at 10 calls per `link_for_document` run. Use a counter
in the loop:
```python
contradiction_calls = 0
MAX_CONTRADICTION_CALLS = 10
...
if summary_a and summary_b and contradiction_calls < MAX_CONTRADICTION_CALLS:
    contradiction_calls += 1
    contradiction_data = await _detect_contradiction(...)
```
This prevents runaway cost while still covering the most important matches.

**Risk 5: enrichment handler registration was missing for ALL job types (HIGH)**

As discovered during planning, no handler registration or `worker.start()` call exists in the
current `main.py` lifespan. Step 7 adds ALL missing registrations including the existing
image_extract, image_analyze, diagram_extract, web_refs, and prerequisites handlers.
This is a comprehensive fix, not just adding concept_link.

Tests that previously passed for these features may have been testing the service methods
directly (not through the worker). The worker integration will now actually function.

**Risk 6: Viz `fetchGraphData` signature change with `showCrossBook` (MEDIUM)**

The `fetchGraphData` function is called in a `useQuery` and its signature currently does
not take `showCrossBook`. Adding it requires updating both the function and the `queryKey`
array. Failing to update the queryKey means the cache will not invalidate when the toggle
changes.

Mitigation: Step 10g explicitly adds `showCrossBook` to the queryKey.

**Risk 7: `GET /graph/concepts/linked` path may conflict with existing routes (LOW)**

The existing routes in `routers/graph.py` are:
- `GET /graph/learning-path`
- `GET /graph/entities/{document_id}`
- `GET /graph/{document_id}`
- `GET /graph`

The new `GET /graph/concepts/linked` could be matched as `GET /graph/{document_id}` with
`document_id="concepts"` if not ordered correctly. FastAPI matches routes in declaration order.
The new route must be declared BEFORE `GET /graph/{document_id}`.

Mitigation: Step 6 explicitly requires placing the new route before the parameterized routes.
