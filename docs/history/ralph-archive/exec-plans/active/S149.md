# S149 — EPUB Chapter Rendering Viewer with Styled HTML, Spine Navigation, and SelectionActionBar

## Goal

Add a "Book View" tab to DocumentReader that renders sanitized EPUB chapter HTML with a two-column layout (chapter TOC on the left, rendered HTML on the right), backed by two new API endpoints, with SelectionActionBar (S147) integrated for text selection actions.

## Layers Affected

- **Service** — new `epub_service.py` in `backend/app/services/`
- **API** — new endpoints added to `backend/app/routers/documents.py` (extends existing router)
- **Frontend Types** — no new TS types file needed; types are inline in `EPUBViewer.tsx`
- **Frontend Component** — new `EPUBViewer.tsx` in `frontend/src/components/reader/`
- **Frontend Integration** — `DocumentReader.tsx` gains a new left-tab option and wires the viewer

## Files to Create or Modify

- `backend/app/services/epub_service.py` — new service: TOC parsing, chapter HTML extraction, sanitization
- `backend/app/routers/documents.py` — add two new endpoints (`epub/toc`, `epub/chapter/{n}`) and Pydantic response models
- `backend/app/main.py` — no change needed (epub endpoints live on the existing documents router)
- `frontend/src/components/reader/EPUBViewer.tsx` — new component: TOC panel + rendered chapter panel
- `frontend/src/components/reader/DocumentReader.tsx` — add "Book View" tab for epub format; mount EPUBViewer
- `backend/tests/test_s149_epub_viewer.py` — unit tests for sanitizer and TOC/chapter endpoints
- `scripts/smoke/S149.sh` — smoke test confirming endpoints respond

## Backend API Design

### Response Models (in `documents.py`)

```python
class EpubChapterTocItem(BaseModel):
    chapter_index: int
    title: str
    word_count: int

class EpubTocResponse(BaseModel):
    chapters: list[EpubChapterTocItem]

class EpubChapterResponse(BaseModel):
    html: str              # sanitized HTML string
    chapter_title: str
    word_count: int
    section_ids: list[str] # SectionModel.id values whose section_order falls in this chapter
```

### Endpoints

```
GET /documents/{document_id}/epub/toc
  -> 404 if document not found
  -> 400 if document.format != 'epub'
  -> 200 EpubTocResponse

GET /documents/{document_id}/epub/chapter/{chapter_index}
  -> 404 if document not found
  -> 400 if document.format != 'epub'
  -> 404 if chapter_index out of range
  -> 200 EpubChapterResponse
```

### Service: `epub_service.py`

The service is stateless; methods are synchronous (ebooklib reads files synchronously).

**bleach.clean() configuration:**

```python
_ALLOWED_TAGS = [
    "p", "h1", "h2", "h3", "h4", "h5", "h6",
    "table", "thead", "tbody", "tr", "td", "th",
    "ul", "ol", "li", "em", "strong", "code", "pre",
    "blockquote", "figure", "a", "span", "div", "br", "hr",
]
_ALLOWED_ATTRS: dict[str, list[str]] = {
    "a": ["href", "title"],
    "*": ["class", "id"],
}
# img is intentionally excluded to avoid broken relative EPUB image paths
_STRIP_TAGS_BEFORE_BLEACH = ["script", "style", "link", "iframe", "meta", "head"]
```

**Section ID mapping logic (in the router, not the service):**

EPUB spine items are 0-indexed. SectionModel rows have `section_order` (0-indexed integer). The chapter_index maps to the Nth spine item. To identify which section_ids belong to chapter N:
- total_chapters = number of chapters returned by `get_toc()`
- total_sections = count of SectionModel rows for this document
- sections_per_chapter = ceil(total_sections / total_chapters) if total_chapters > 0 else 0
- Chapter N owns sections with section_order in [N * spc, (N+1) * spc)
- This is a best-effort proportional partition; EPUB chapters do not have a guaranteed 1:1 mapping to SQLite sections.

## Frontend Component Architecture

### EPUBViewer.tsx

```
frontend/src/components/reader/EPUBViewer.tsx

Props:
  documentId: string

Internal state:
  currentChapterIndex: number (0-based, default 0)

TanStack Query keys:
  ["epub-toc", documentId]      staleTime: 300_000
  ["epub-chapter", documentId, currentChapterIndex]   staleTime: 60_000

Layout: flex-row, full height
  Left: w-56 shrink-0 overflow-auto border-r border-border (TOC list)
  Right: flex-1 overflow-auto (chapter content)

TOC panel (left, 224px fixed):
  - Loading: 3 Skeleton rows (h-6 w-full each)
  - Error: inline <p className="text-destructive"> + Retry button
  - Empty: <p>No chapters found.</p>
  - Populated: scrollable list; each item is a button
    - Active chapter: bg-accent text-foreground
    - Inactive: text-muted-foreground hover:bg-accent/50
    - Click: setCurrentChapterIndex(item.chapter_index)

Chapter panel (right):
  - Loading: 5 Skeleton rows (h-4 w-full each with varying widths)
  - Error: inline error message + retry button
    (retry = refetch via useQuery refetch())
  - Empty: "Select a chapter from the list." when toc is empty
  - Content:
    <div className="prose prose-sm max-w-none dark:prose-invert px-6 py-4"
         dangerouslySetInnerHTML={{ __html: chapterData.html }} />
  - Prev/Next bar at bottom of right panel (sticky bottom-0 bg-background border-t):
    <button disabled={currentChapterIndex === 0} onClick={() => prev}>
      Previous Chapter
    </button>
    <span>{chapter_title}</span>
    <button disabled={currentChapterIndex === tocData.chapters.length - 1} onClick={() => next}>
      Next Chapter
    </button>
```

Note: EPUBViewer does NOT own a SelectionActionBar. The parent DocumentReader's existing SelectionActionBar is bound to `containerRef` (the whole left panel). EPUBViewer renders inside that container, so mouseup events bubble naturally. The `resolveSourceRef` walk-up will yield `sectionId: undefined` (no `data-section-id` on EPUB HTML) -- meaning Highlight is disabled, the other 4 actions (Explain, Note, Flashcard, Ask) remain available.

## Implementation Steps

1. **[Dependency]** Add Python dependencies:
   ```bash
   cd backend && uv add bleach beautifulsoup4
   ```
   Note: `bs4` was verified NOT present in the venv (ModuleNotFoundError confirmed). `beautifulsoup4` must be added explicitly. `bleach` also not present.

2. **[Service]** Create `backend/app/services/epub_service.py`:
   - Module-level logger: `logger = logging.getLogger(__name__)`
   - Define `_ALLOWED_TAGS`, `_ALLOWED_ATTRS`, `_STRIP_TAGS_BEFORE_BLEACH` constants
   - `EpubService.sanitize_html(raw_html: str) -> str` static method:
     - Parse with `BeautifulSoup(raw_html, "html.parser")`
     - Decompose all tags in `_STRIP_TAGS_BEFORE_BLEACH`
     - Get body content: `soup.find("body")` or whole soup if no body tag
     - Call `bleach.clean(str(body_soup), tags=_ALLOWED_TAGS, attributes=_ALLOWED_ATTRS, strip=True)`
     - Return result
   - `EpubService.get_toc(file_path: str) -> list[dict]`:
     - `book = ebooklib.epub.read_epub(file_path, options={"ignore_ncx": True})`
     - Iterate `book.spine`: each entry is `(item_id, linear)` -- use `book.get_item_with_id(item_id)`
     - For items with `ITEM_DOCUMENT` type: parse content with BeautifulSoup, extract title, compute word_count
     - Return `[{"chapter_index": i, "title": title, "word_count": wc} for i, ...]`
   - `EpubService.get_chapter(file_path: str, chapter_index: int) -> dict`:
     - Validate index against spine length; raise `IndexError` if out of range
     - Get spine item, decode content, call `sanitize_html`
     - Return `{"html": sanitized, "chapter_title": title, "word_count": wc}`
   - `get_epub_service() -> EpubService`: returns `EpubService()`

3. **[API]** In `backend/app/routers/documents.py`:
   - Add `EpubChapterTocItem`, `EpubTocResponse`, `EpubChapterResponse` Pydantic models after the existing `PDFMetaResponse` model (around line 196)
   - Add `GET /{document_id}/epub/toc` endpoint after the `get_pdf_meta` endpoint:
     - Fetch DocumentModel; 404 if not found; 400 if `doc.format.lower() != "epub"`
     - Lazy import: `from app.services.epub_service import get_epub_service  # noqa: PLC0415`
     - Call `get_epub_service().get_toc(doc.file_path)` in `asyncio.get_event_loop().run_in_executor(None, ...)` or directly (ebooklib is sync I/O; wrap in `run_in_executor` to avoid blocking the event loop)
     - Return `EpubTocResponse(chapters=[EpubChapterTocItem(**c) for c in chapters])`
   - Add `GET /{document_id}/epub/chapter/{chapter_index}` endpoint:
     - Fetch DocumentModel; 404 / 400 guards as above
     - Fetch all SectionModel rows for document_id ordered by section_order
     - Call `get_epub_service().get_toc(doc.file_path)` to get total chapter count for index validation and section_ids partition
     - Validate chapter_index: 404 if >= total chapters or < 0
     - Compute section_ids using proportional partition formula
     - Call `get_epub_service().get_chapter(doc.file_path, chapter_index)`
     - Return `EpubChapterResponse(section_ids=section_ids, **chapter_dict)`

## Risk Flags

- **Risk: bs4 not available as transitive dep (CONFIRMED)** -- `import bs4` raises `ModuleNotFoundError` in the current venv. Must run `uv add beautifulsoup4` explicitly. Do not assume it is available. (Mitigation: Step 1 includes `uv add bleach beautifulsoup4` together.)

- **Risk: ebooklib epub.read_epub warnings** -- ebooklib emits `UserWarning` for certain EPUB 3 features. These warnings do not cause failures but may appear in test output. (Mitigation: add `warnings.filterwarnings("ignore", category=UserWarning, module="ebooklib")` at the top of `epub_service.py` to suppress noise.)

- **Risk: bleach deprecation** -- bleach is in maintenance mode as of 2023; its `clean()` function still works for this use case but the library is no longer actively developed. The story explicitly requires bleach; use it as specified. Do not substitute with nh3 or other alternatives without story approval.

- **Risk: EPUB file not on disk at test time** -- unit tests for the TOC/chapter endpoints must not depend on a real EPUB file. Mock `get_epub_service` at the router module using `patch("app.routers.documents.get_epub_service", return_value=mock_service)`. For sanitizer unit tests, `sanitize_html` is a pure static method and needs no mocks.

- **Risk: SelectionActionBar coordinates in EPUBViewer** -- the existing SelectionActionBar in DocumentReader is attached to `containerRef` (the entire left panel). EPUBViewer renders inside that DOM subtree, so mouseup events bubble naturally. The `resolveSourceRef` walk-up will find no `data-section-id` attribute in EPUB HTML, yielding `sectionId: undefined` -- meaning the Highlight action is disabled but the other 4 actions remain available. This matches the PDF viewer behavior.

- **Risk: @tailwindcss/typography confirmed available** -- `frontend/tailwind.config.cjs` already includes `require('@tailwindcss/typography')` in the plugins array. Using `prose prose-sm max-w-none dark:prose-invert` in EPUBViewer is safe.

- **Risk: EPUB relative image src paths** -- EPUB images use internal relative paths (e.g., `../images/cover.jpg`) that will 404 when rendered. `img` is intentionally excluded from `_ALLOWED_TAGS` to avoid broken image display at MVP. This means images in EPUB chapters are stripped. Accept this limitation for S149; a later story can add image serving support.

- **Risk: Blocking event loop for ebooklib I/O** -- `ebooklib.epub.read_epub()` is synchronous file I/O. In the FastAPI async router, wrap the call in `await asyncio.get_event_loop().run_in_executor(None, epub_service.get_toc, file_path)` to avoid blocking the event loop. Do the same for `get_chapter`. This is critical for the server to remain responsive during large EPUB parsing.

- **Risk: Large chapter HTML** -- some EPUB chapters (e.g., SICP Chapter 1) may have 50k+ words. The entire chapter HTML is returned in one response and rendered in one `dangerouslySetInnerHTML`. No pagination in this story. Add a tech debt note if chapter HTML exceeds 200KB.
