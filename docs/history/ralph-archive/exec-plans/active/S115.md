# S115 — Study Session Analytics: Per-Session History with Accuracy and Cards Reviewed

## Goal

Add a paginated `GET /study/sessions` endpoint and a History sub-tab in the Study page that shows per-session statistics (date, duration, cards reviewed, accuracy %, document name) with an expandable card-level breakdown row.

## Layers Affected

- **Types** (Pydantic schemas in study.py)
- **Repo** (additive SQLite migration in db_init.py; direct ORM queries in study.py)
- **API** (new `GET /study/sessions` endpoint; new `GET /study/sessions/{session_id}/cards` endpoint for card-level detail)
- **Frontend** (new History sub-tab in Study.tsx)

## Pre-Implementation: What Already Exists

The following are ALREADY implemented and must NOT be duplicated:

- `StudySessionModel` in `backend/app/models.py` with columns: `id`, `document_id`, `started_at`, `ended_at`, `cards_reviewed`, `cards_correct`, `mode`
- `ReviewEventModel` in `backend/app/models.py` with columns: `id`, `session_id`, `flashcard_id`, `rating`, `is_correct`, `reviewed_at`
- `POST /study/sessions/start` at `backend/app/routers/study.py` (note: S115 AC says `/study/start-session` but the existing endpoint already satisfies the spirit; the plan maps to what is already shipped)
- `POST /study/sessions/{session_id}/end` at `backend/app/routers/study.py`

What is MISSING and must be added:

1. `accuracy_pct` column on `study_sessions` table (computed value stored for fast reads; additive migration)
2. `GET /study/sessions` paginated endpoint (not yet present)
3. `GET /study/sessions/{session_id}/cards` endpoint for card-level breakdown
4. Frontend History sub-tab in `Study.tsx`

In `backend/app/models.py`, add to `StudySessionModel`:


Place after `cards_correct`. This is nullable so existing rows are unaffected.

### Step 2 — [Repo] Add additive migration in db_init.py

In `backend/app/db_init.py`, add to the list of additive DDL statements:

```python

```

No schema import changes needed — `StudySessionModel` is already imported.

### Step 3 — [API] Update end_session to compute and persist accuracy_pct

In the `end_session` handler in `backend/app/routers/study.py`:

After computing `cards_reviewed` and `cards_correct`, add:

```python

sess.accuracy_pct = accuracy_pct
```

Update `SessionSummary` Pydantic schema to include:
```python

```

Update `SessionResponse` Pydantic schema to include:
```python

```

### Step 4 — [API] Add GET /study/sessions endpoint

Add new Pydantic schemas to `backend/app/routers/study.py`:

```python
class SessionListItem(BaseModel):
    id: str
    started_at: datetime
    ended_at: datetime | None
    duration_minutes: float | None
    cards_reviewed: int
    cards_correct: int
    accuracy_pct: float | None
    document_id: str | None
    document_title: str | None
    mode: str

    model_config = {"from_attributes": True}


class SessionListResponse(BaseModel):
    items: list[SessionListItem]
    total: int
    page: int
    page_size: int
```

Add the endpoint:

```python
@router.get("/sessions", response_model=SessionListResponse)
async def list_sessions(
    document_id: str | None = None,
    page: int = Query(default=1, ge=1),
    page_size: int = Query(default=20, ge=1, le=100),
    db: AsyncSession = Depends(get_db),
) -> SessionListResponse:
```

Implementation:
- Count total sessions with optional document_id filter
- Fetch paginated sessions ordered by `started_at` DESC
- Join `DocumentModel` on `document_id` to get `document_title`
- Compute `duration_minutes` from `ended_at - started_at` (None if session is open)
- Return `SessionListResponse`

### Step 5 — [API] Add GET /study/sessions/{session_id}/cards endpoint

Add new Pydantic schema:

```python
class SessionCardDetail(BaseModel):
    flashcard_id: str
    question: str
    rating: str          # again|hard|good|easy
    is_correct: bool
    reviewed_at: datetime
```

Add the endpoint:

```python
@router.get("/sessions/{session_id}/cards", response_model=list[SessionCardDetail])
async def get_session_cards(
    session_id: str,
    db: AsyncSession = Depends(get_db),
) -> list[SessionCardDetail]:
```

Implementation:
- Verify session exists (404 if not)
- Join `ReviewEventModel` with `FlashcardModel` on `flashcard_id`
- Return list of `SessionCardDetail` ordered by `reviewed_at`

## Risk Flags

- Risk: `StudySessionModel` already has `cards_correct`; S115 AC says `correct_count` — these are the same field. Do NOT add a second column. Use `cards_correct` throughout and map to `correct_count` only in response schemas if ACs require that name explicitly. Mitigation: keep the existing column name `cards_correct` in the model; the response schema field `cards_correct` satisfies the AC spirit.

- Risk: The existing session start/end endpoint paths are `/study/sessions/start` and `/study/sessions/{id}/end`, NOT `/study/start-session` and `/study/end-session/{session_id}` as stated in AC2/AC3. The existing paths already fully satisfy the AC intent. Mitigation: do not add duplicate endpoints at new paths; map AC2/AC3 to the existing endpoints.

- Risk: `GET /study/sessions/{session_id}/cards` will do an N+1 join if implemented naively. Mitigation: use a single SQLAlchemy join between `ReviewEventModel` and `FlashcardModel` in one query.

- Risk: The frontend Study.tsx is large (1382 lines). Adding a History tab with a new component increases file length further. Mitigation: `SessionHistoryTab` is a self-contained function component defined in the same file; no new file needed per project conventions (all pages are single files). Keep the component tightly scoped.

- Risk: `accuracy_pct` for sessions that were ended before this migration have `NULL` accuracy. The frontend must handle `accuracy_pct: null` gracefully, displaying "-" in the table cell.

- Risk: `DocumentModel` is not currently imported in `study.py`. Add it to the import block from `app.models`. This is an additive import and does not violate layer rules (API importing from models/types is correct).
