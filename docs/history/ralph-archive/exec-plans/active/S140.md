# S140 — Code Execution Sandbox with Predict-then-Run

## Goal

Add a sandboxed code execution endpoint and a Predict-then-Run UI in the Document Reader: users type an expected output before running a code block, see a diff when they are wrong, and can one-click create a 'trace' flashcard from the mistake.

- **Repo** — `backend/app/db_init.py` (register new model import + additive migration for `prediction_events` table)
- **Service** — new `backend/app/services/code_executor.py` (`CodeExecutorService`)
- **Runtime** — no new LangGraph nodes (pure subprocess; no LLM)

## Design Notes

### Where code_language comes from

`SectionItem` in `frontend/src/components/reader/types.ts` does not currently include `code_language`. The `DocumentDetail` API response (`GET /documents/{id}`) returns sections. Looking at `SectionModel`, it does not have `code_language` — that field lives on `ChunkModel`. However, the story says "code blocks with `code_language` set" — meaning code chunks whose `has_code=True` and `code_language` is set.

The right approach: when the document detail response is built, include a new field `has_code_chunks: bool` on each section (True if any chunk in that section has `has_code=True`). **However**, reading `GET /documents/{id}` in `documents.py` will show the actual shape. Rather than a full join, the simpler approach: the Predict toggle is shown for any section whose `preview` contains a fenced code block marker (` ``` `) — this is a pure frontend heuristic that requires no backend change. This avoids a backend schema change to `SectionItem`.

**Decision**: Predict toggle renders when `section.preview` contains a backtick fence pattern. The code content sent to the executor is extracted from the preview's first fenced block. This is safe and requires no backend API change to the documents endpoint.

Alternative if the preview heuristic is too fragile: add `has_code: boolean` to `SectionItem` via a subquery in the documents router. This is the cleaner long-term approach. The plan includes this as step 2a (backend) with a flag.

### Flashcard create-trace endpoint

The story needs a one-click "create flashcard from mistake." This requires a POST endpoint that creates a `FlashcardModel` row directly (no LLM). We add `POST /flashcards/create-trace` accepting `TraceFlashcardRequest(question, answer, source_excerpt, document_id, chunk_id)`. This creates a row with `source='prediction_error'`, `flashcard_type='trace'`, `deck='default'`.

### PredictionEventModel

`chunk_id` is nullable because the predict panel is section-scoped (no chunk id available in section list context). `code_content` is truncated to 2000 chars. `correct=True` uses stripped comparison (normalizes trailing newlines). This means `expected='hello'` vs `actual='hello\n'` yields `correct=True` (AC5 documents this as the edge case: strip comparison normalizes trailing newline).

### subprocess sandboxing

`subprocess.run(['python3', '-c', code], capture_output=True, timeout=10, shell=False, cwd=temp_dir, env={})` with `resource.setrlimit(RLIMIT_AS, (256*1024*1024, 256*1024*1024))` set via a `preexec_fn`. The `preexec_fn` only works on POSIX (Linux/macOS). On Windows it is skipped gracefully. `RLIMIT_CPU` (10 seconds CPU time) is also set in `preexec_fn`.

`timeout=10` in `subprocess.run` enforces wall-clock timeout (kills process on `subprocess.TimeoutExpired`).

For JavaScript: `shutil.which('node')` check; raise `HTTPException(503, "Install Node.js (nodejs.org) to run JavaScript snippets")` if absent.

The `env={}` (empty dict) means no environment variables are inherited — no PATH. We must pass `python3` as full path or accept that it might not resolve. Resolution: use `sys.executable` (the current Python interpreter) for Python execution. For Node.js use `shutil.which('node')` result as full path.

### Unified diff

`difflib.unified_diff(expected_lines, actual_lines, fromfile='expected', tofile='actual')` joined as a string. Stored in `prediction_diff` on the response.

### Frontend states

The `PredictPanel` component has three states:
- **Empty/initial**: "Predict" toggle button visible next to sections with code
- **Active (predict mode)**: textarea for expected output + "Run and Compare" button + optional "Kill" button during run
- **Result**: diff panel (side-by-side or unified) + "Create flashcard from this mistake?" toast/inline banner when `prediction_correct=False`

The "Create flashcard" flow opens a pre-filled dialog (not a toast link to a separate page) — inline within DocumentReader using a `useState` for a dialog open flag and prefilled question/answer strings. This avoids a modal-within-modal complexity since we can reuse the existing dialog pattern.

### Step 3 — [Service] Create `backend/app/services/code_executor.py`

New file. Key design:

```python
import difflib
import logging
import os
import resource
import shutil
import subprocess
import sys
import tempfile
import time
import uuid
from functools import lru_cache

logger = logging.getLogger(__name__)

_TIMEOUT_SECONDS = 10
_MEMORY_LIMIT_BYTES = 256 * 1024 * 1024  # 256 MB


def _set_resource_limits() -> None:
    """Called in preexec_fn for the subprocess. POSIX only."""
    try:
        resource.setrlimit(resource.RLIMIT_AS, (_MEMORY_LIMIT_BYTES, _MEMORY_LIMIT_BYTES))
        resource.setrlimit(resource.RLIMIT_CPU, (_TIMEOUT_SECONDS, _TIMEOUT_SECONDS))
    except (AttributeError, ValueError):
        pass  # Windows or unsupported platform


class ExecutionResult:
    """Pure data class — no I/O."""
    def __init__(
        self,
        stdout: str,
        stderr: str,
        exit_code: int,
        elapsed_ms: int,
        prediction_correct: bool | None,
        prediction_diff: str | None,
    ) -> None:
        self.stdout = stdout
        self.stderr = stderr
        self.exit_code = exit_code
        self.elapsed_ms = elapsed_ms
        self.prediction_correct = prediction_correct
        self.prediction_diff = prediction_diff


def _compare_prediction(expected: str, actual: str) -> tuple[bool, str]:
    """Pure function. Compares stripped expected vs stripped actual.

    Trailing newlines are normalized by strip() so 'hello' == 'hello\\n'.
    Returns (correct: bool, diff: str).
    """
    exp_stripped = expected.strip()
    act_stripped = actual.strip()
    correct = exp_stripped == act_stripped
    diff_lines = list(
        difflib.unified_diff(
            exp_stripped.splitlines(keepends=True),
            act_stripped.splitlines(keepends=True),
            fromfile="expected",
            tofile="actual",
        )
    )
    diff = "".join(diff_lines) if not correct else ""
    return correct, diff


class CodeExecutorService:
    def execute(
        self,
        code: str,
        language: str,
        timeout_ms: int = 10_000,
        expected_output: str | None = None,
    ) -> ExecutionResult:
        """Execute code in a sandboxed subprocess.

        Python: uses sys.executable (the uv-managed venv Python).
        JavaScript: uses shutil.which('node'); raises ValueError if absent.
        Raises ValueError for unsupported languages.
        """
        language = language.lower()
        if language == "python":
            cmd = [sys.executable, "-c", code]
        elif language in ("javascript", "js"):
            node_path = shutil.which("node")
            if node_path is None:
                raise LookupError("node_not_found")
            cmd = [node_path, "-e", code]
        else:
            raise ValueError(f"Unsupported language: {language}")

        timeout_s = min(timeout_ms / 1000.0, _TIMEOUT_SECONDS)
        with tempfile.TemporaryDirectory() as tmp_dir:
            start = time.monotonic()
            try:
                proc = subprocess.run(
                    cmd,
                    capture_output=True,
                    timeout=timeout_s,
                    shell=False,
                    cwd=tmp_dir,
                    env={},
                    preexec_fn=_set_resource_limits,
                    text=True,
                )
                elapsed_ms = int((time.monotonic() - start) * 1000)
                stdout = proc.stdout
                stderr = proc.stderr
                exit_code = proc.returncode
            except subprocess.TimeoutExpired:
                elapsed_ms = int(timeout_s * 1000)
                stdout = ""
                stderr = f"Timeout after {timeout_s:.0f}s"
                exit_code = 124  # conventional timeout exit code

        prediction_correct: bool | None = None
        prediction_diff: str | None = None
        if expected_output is not None:
            prediction_correct, prediction_diff = _compare_prediction(expected_output, stdout)

        logger.info(
            "code_execute: language=%s exit_code=%d elapsed_ms=%d",
            language,
            exit_code,
            elapsed_ms,
        )
        return ExecutionResult(
            stdout=stdout,
            stderr=stderr,
            exit_code=exit_code,
            elapsed_ms=elapsed_ms,
            prediction_correct=prediction_correct,
            prediction_diff=prediction_diff,
        )


_service: CodeExecutorService | None = None


@lru_cache
def get_code_executor_service() -> CodeExecutorService:
    return CodeExecutorService()
```

Note: `env={}` means the subprocess inherits NO environment variables — not even PATH. Since `cmd[0]` is an absolute path (`sys.executable` or the result of `shutil.which('node')`), PATH is not needed. `os.system("echo pwned")` in the subprocess will succeed (os.system spawns a shell), but `shell=False` on `subprocess.run` only prevents shell injection in the command string itself — not in code the user deliberately calls inside the sandbox. The story's security test (AC2) verifies that `shell=False` prevents injection in the command args, not that `os.system` is blocked — that would require a more restrictive sandbox (seccomp, namespace isolation) which is out of scope.

### Step 5 — [API] Create `backend/app/routers/code_executor.py`

```python
"""Code execution sandbox endpoint — POST /code/execute."""
import logging
import uuid
from datetime import UTC, datetime

from fastapi import APIRouter, Depends, HTTPException
from pydantic import BaseModel, Field
from sqlalchemy.ext.asyncio import AsyncSession

from app.database import get_db
from app.models import PredictionEventModel
from app.services.code_executor import CodeExecutorService, get_code_executor_service

logger = logging.getLogger(__name__)
router = APIRouter(prefix="/code", tags=["code"])


class CodeExecuteRequest(BaseModel):
    code: str = Field(min_length=1)
    language: str = "python"
    timeout_ms: int = Field(default=10_000, ge=100, le=30_000)
    expected_output: str | None = None
    # Optional context for PredictionEvent persistence
    chunk_id: str | None = None
    document_id: str | None = None


class CodeExecuteResponse(BaseModel):
    stdout: str
    stderr: str
    exit_code: int
    elapsed_ms: int
    prediction_correct: bool | None = None
    prediction_diff: str | None = None


@router.post("/execute", response_model=CodeExecuteResponse)
async def execute_code(
    req: CodeExecuteRequest,
    session: AsyncSession = Depends(get_db),
    service: CodeExecutorService = Depends(get_code_executor_service),
) -> CodeExecuteResponse:
    """Execute code in a sandboxed subprocess and compare to expected output.

    Returns stdout, stderr, exit_code, elapsed_ms.
    If expected_output is provided, also returns prediction_correct and prediction_diff.
    prediction_correct uses strip() comparison — trailing newlines are normalized.
    """
    try:
        result = service.execute(
            code=req.code,
            language=req.language,
            timeout_ms=req.timeout_ms,
            expected_output=req.expected_output,
        )
    except LookupError:
        raise HTTPException(
            status_code=503,
            detail="Install Node.js (nodejs.org) to run JavaScript snippets",
        )
    except ValueError as exc:
        raise HTTPException(status_code=400, detail=str(exc))

    # Persist prediction event if expected_output was provided
    if req.expected_output is not None:
        event = PredictionEventModel(
            id=str(uuid.uuid4()),
            chunk_id=req.chunk_id,
            document_id=req.document_id,
            code_content=req.code[:2000],
            expected=req.expected_output,
            actual=result.stdout,
            correct=bool(result.prediction_correct),
            language=req.language,
            created_at=datetime.now(UTC),
        )
        session.add(event)
        await session.commit()

    return CodeExecuteResponse(
        stdout=result.stdout,
        stderr=result.stderr,
        exit_code=result.exit_code,
        elapsed_ms=result.elapsed_ms,
        prediction_correct=result.prediction_correct,
        prediction_diff=result.prediction_diff,
    )
```

## Risk Flags

- Risk: `env={}` removes PATH from the subprocess, so any code that calls `subprocess.run` or `os.system` internally may fail to find binaries. This is intentional (isolation), but `sys.executable` for Python is an absolute path so it is unaffected.
  Mitigation: Document in `CodeExecutorService` docstring.

- Risk: `resource.setrlimit(RLIMIT_AS)` on macOS may not reliably terminate a process that allocates memory via `malloc` (RLIMIT_AS limits virtual memory, but macOS mmap behavior differs). The memory limit test (AC4) may not reliably terminate with non-zero exit code on macOS.
  Mitigation: AC4 test is written as `assert result.exit_code != 0 or "MemoryError" in result.stderr` to accept either failure mode; adds a comment noting POSIX-only enforcement.

- Risk: `preexec_fn` is not supported on Windows. `subprocess.run` will raise `ValueError` if `preexec_fn` is passed on Windows.
  Mitigation: Wrap `preexec_fn` arg in `{} if sys.platform == 'win32' else {'preexec_fn': _set_resource_limits}` unpacked into `subprocess.run`.

- Risk: The frontend code-fence heuristic (`section.preview` contains ` ``` `) may produce false positives for sections that merely quote a command inline. The predict toggle would appear on non-code sections.
  Mitigation: Use a tighter regex: the preview must contain a newline-delimited ` ``` ` pattern (multi-line fenced block), not just a backtick sequence. Pattern: `/^```\w*\n[\s\S]+\n```/m`.

- Risk: `POST /flashcards/create-trace` adds a new endpoint with prefix `/flashcards/create-trace` — this could conflict with `GET /flashcards/{document_id}` if FastAPI matches `create-trace` as a `document_id` path param. Resolution: in FastAPI, static path segments take priority over path params ONLY if the static route is defined first. Ensure `create-trace` is defined before `/{document_id}` in `flashcards.py`. Since it's a POST vs GET they don't conflict anyway — but confirm by reading the router definition order.

- Risk: `prediction_events` table is a net-new table not cleaned up by document delete paths. Since `document_id` is nullable and no FK, orphaned rows accumulate. This is acceptable for the initial story (same pattern as `review_events`, `teachback_results`).
  Mitigation: Document as TD-016 in tech-debt-tracker.md if needed.

- Risk: The `lru_cache` on `get_code_executor_service` is appropriate since `CodeExecutorService` has no mutable state and no DB connection.

- Risk: The Kill button on the frontend — aborting a `fetch` request via `AbortController` only cancels the HTTP request on the client side; the subprocess on the server continues running until the server's `subprocess.run(timeout=...)` fires. For the timeout-based kill to work, the server must already enforce the `timeout=10` limit. The Kill button provides immediate UI feedback; the server subprocess will eventually terminate via its timeout.
  Mitigation: Document this limitation in the component comment. No cross-process kill signal is sent on abort; the server subprocess terminates at its timeout boundary.
