# S132 — Tech Section Detection: Admonitions, Numbered Hierarchy, and Learning Objectives

## Goal

Extend the tech book ingestion pipeline to detect numbered section hierarchy (Part/Chapter/N.M.P), admonition blocks (NOTE/WARNING/TIP/CAUTION/IMPORTANT), and chapter learning objectives, and surface these in the Document Reader and Learning tab.

## Layers Affected

- **Types** — `Section` dataclass gains `admonition_type` and `parent_heading` fields for propagation through parse pipeline
- **Repo/Models** — `SectionModel` gains `admonition_type` and `parent_section_id` columns; new `LearningObjectiveModel` table
- **Config** — `db_init.py` gains additive ALTER TABLE migrations for both new columns; creates `learning_objectives` table
- **Service** — New `LearningObjectiveExtractor` service calls LiteLLM for objective extraction; `UniversalParser` gains tech_book signature; admonition regex applied in `_chunk_tech_book`
- **API** — `GET /sections/{document_id}` response model gains `admonition_type` and `parent_section_id`; new `GET /documents/{document_id}/objectives` endpoint added to `documents.py`
- **Runtime** — `_chunk_tech_book` in `ingestion.py` reads `admonition_type` and `parent_section_id` from section dicts and writes them to `SectionModel`; fires objective extraction as background task after section creation
- **Frontend** — `types.ts` gains `admonition_type` and `parent_section_id` on `SectionItem`; `DocumentReader.tsx` renders admonition left-border styling; `Learning.tsx` gains Chapter Goals panel

```python
# Admonition detection
_ADMONITION_PATTERNS = [
    (re.compile(r"^(?:NOTE|note)\s*:", re.M), "note"),
    (re.compile(r"^(?:WARNING|warning)\s*:", re.M), "warning"),
    (re.compile(r"^(?:TIP|tip)\s*:", re.M), "tip"),
    (re.compile(r"^(?:CAUTION|caution)\s*:", re.M), "caution"),
    (re.compile(r"^(?:IMPORTANT|important)\s*:", re.M), "important"),
    # Markdown blockquote forms: > **Note**, > **Warning**, etc.
    (re.compile(r"^>\s+\*\*(?:Note|NOTE)\*\*", re.M), "note"),
    (re.compile(r"^>\s+\*\*(?:Warning|WARNING)\*\*", re.M), "warning"),
    (re.compile(r"^>\s+\*\*(?:Tip|TIP)\*\*", re.M), "tip"),
    (re.compile(r"^>\s+\*\*(?:Caution|CAUTION)\*\*", re.M), "caution"),
    (re.compile(r"^>\s+\*\*(?:Important|IMPORTANT)\*\*", re.M), "important"),
]

def detect_admonition(text: str) -> str | None:
    """Return the admonition type if the text starts with an admonition marker, else None."""
    for pattern, admonition_type in _ADMONITION_PATTERNS:
        if pattern.search(text[:500]):  # Check first 500 chars only
            return admonition_type
    return None
```

```python
# Numbered hierarchy: assign level and parent_heading
_RE_PART = re.compile(r"^Part\s+[IVXLCDM\d]+\b", re.I)
_RE_CHAPTER = re.compile(r"^Chapter\s+\d+\b", re.I)
_RE_SECTION_N = re.compile(r"^(\d+)\.\s")          # "1. Introduction"
_RE_SECTION_NM = re.compile(r"^(\d+)\.(\d+)\s")    # "1.1 Overview"
_RE_SECTION_NMP = re.compile(r"^(\d+)\.(\d+)\.(\d+)\s")  # "1.1.1 Detail"

def classify_section_level(heading: str) -> int:
    """Return 1=Part, 2=Chapter, 3=Section, 4=Subsection based on heading pattern."""
    if _RE_PART.match(heading):
        return 1
    if _RE_CHAPTER.match(heading):
        return 2
    if _RE_SECTION_NMP.match(heading):
        return 4
    if _RE_SECTION_NM.match(heading):
        return 3
    if _RE_SECTION_N.match(heading):
        return 2
    return 2  # default: treat as chapter-level

def assign_parent_headings(sections: list[Section]) -> list[Section]:
    """Return new Section list with parent_heading set based on level hierarchy.

    Rules:
    - level=1 (Part): no parent
    - level=2 (Chapter): parent is the most recent level=1
    - level=3 (N.M section): parent is the most recent level=2
    - level=4 (N.M.P subsection): parent is the most recent level=3
    If no ancestor exists at the required level, parent_heading=None.
    """
    ...
```

```python
# Learning objective candidate heuristics
_OBJECTIVE_PHRASES = [
    "you will", "you will be able to", "after this chapter",
    "in this chapter we cover", "by the end of", "goals", "objectives",
    "learning objectives", "chapter goals", "what you will learn",
]

def is_objective_candidate(text: str) -> bool:
    """Return True if the first 300 chars of text match objective patterns."""
    snippet = text[:300].lower()
    return any(phrase in snippet for phrase in _OBJECTIVE_PHRASES)
```

@router.get("/{document_id}/objectives", response_model=LearningObjectivesResponse)
async def get_objectives(document_id: str) -> LearningObjectivesResponse:
    """Return extracted learning objectives for a document.

    Returns [] for fiction books (no rows). Returns 404 if document does not exist.
    """
    from app.models import LearningObjectiveModel  # noqa: PLC0415

    async with get_session_factory()() as session:
        doc_check = await session.execute(
            select(DocumentModel.id).where(DocumentModel.id == document_id)
        )
        if doc_check.scalar_one_or_none() is None:
            raise HTTPException(status_code=404, detail="Document not found")

        result = await session.execute(
            select(LearningObjectiveModel)
            .where(LearningObjectiveModel.document_id == document_id)
            .order_by(LearningObjectiveModel.created_at)
        )
        objectives = result.scalars().all()

    return LearningObjectivesResponse(
        document_id=document_id,
        objectives=[
            LearningObjectiveItem(
                id=obj.id,
                section_id=obj.section_id,
                text=obj.text,
                covered=obj.covered,
            )
            for obj in objectives
        ],
    )

interface LearningObjective {

  section_id: string

interface ChapterGoalsPanelProps {
  documentId: string
}

function ChapterGoalsPanel({ documentId }: ChapterGoalsPanelProps) {
  const { data, isLoading, isError } = useQuery({
    queryKey: ["objectives", documentId],
    queryFn: async () => {
      const res = await fetch(`${API_BASE}/documents/${documentId}/objectives`)
      if (!res.ok) throw new Error("Failed to fetch objectives")
      return res.json() as Promise<{ objectives: LearningObjective[] }>
    },
    staleTime: 300_000,
  })

  // Loading state
  if (isLoading) {
    return (
      <div className="flex flex-col gap-1.5 py-2">
        {[1, 2, 3].map((i) => <Skeleton key={i} className="h-4 w-full" />)}
      </div>
    )
  }

  // Error state
  if (isError) {
    return <p className="text-xs text-destructive">Could not load chapter goals.</p>
  }

  // Empty state (fiction books — 0 objectives)
  if (!data || data.objectives.length === 0) {
    return null  // graceful absence — no panel for fiction
  }

  return (
    <div className="rounded-lg border border-border bg-card p-4">
      <h3 className="mb-2 text-sm font-semibold text-foreground">Chapter Goals</h3>
      <ul className="space-y-1.5">
        {data.objectives.map((obj) => (
          <li key={obj.id} className="flex items-start gap-2 text-xs text-foreground/80">
            <span className="mt-0.5 shrink-0 text-muted-foreground">-</span>
            {obj.text}
          </li>
        ))}
      </ul>
    </div>
  )
}
```

set -euo pipefail
DOC_ID="${1:-}"
if [ -z "$DOC_ID" ]; then
  echo "Usage: $0 <document_id>"
  exit 1
fi
STATUS=$(curl -s -o /dev/null -w "%{http_code}" "http://localhost:8000/documents/${DOC_ID}/objectives")
if [ "$STATUS" -ne 200 ]; then
  echo "FAIL: GET /documents/${DOC_ID}/objectives returned $STATUS"
  exit 1
fi
echo "PASS: GET /documents/${DOC_ID}/objectives returned 200"
```

## Risk Flags

- **Risk: `_chunk_tech_book` only handles tech_book content type** — the admonition detection and hierarchy classification only run for `content_type in ("tech_book", "tech_article")`. Fiction books parsed by `_chunk_book` will never have `admonition_type` set. This is correct behavior per the story, but implementors should not accidentally wire the enrichment into `_chunk_book`. Mitigation: unit tests for `_chunk_book` must still pass unchanged.

- **Risk: `parent_section_id` two-flush pattern** — The current `_chunk_tech_book` does `session.flush()` once, then `session.add_all(chunk_models)`, then `session.commit()`. Inserting a second flush to resolve `parent_section_id` is safe (SQLite supports it), but the order matters: flush sections first, resolve parents, update section models, flush again, then add chunks. Test this explicitly with an in-memory DB test.

- **Risk: LLM extraction fires for every qualifying section on re-ingestion** — `_run_objective_extraction` should be idempotent: delete existing objectives for the document before inserting new ones. Otherwise re-ingestion duplicates rows. The `store()` method on `LearningObjectiveExtractorService` must DELETE then INSERT per document (not per section).

- **Risk: frontend `admonition_type` field breaks TypeScript strict mode** — The existing `SectionItem` interface does not have `admonition_type`. Adding it as `string | null` with a default value in the response model means old cached API responses (from a running server before migration) will not have the field. Use optional chaining (`section.admonition_type ?? null`) defensively in the renderer. The TypeScript type should be `string | null` (not `string | undefined`) to match the Pydantic `Optional[str]` serialization.

- **Risk: admonition styling conflicts with fragility heatmap** — Both use `border-l-4`. The plan resolves this by preferring admonition over fragility. If a section is both an admonition AND has high fragility, the admonition style wins. This is the right UX tradeoff (admonition type is more semantically relevant than fragility for display).

- **Risk: `classify_section_level` regex conflicts with fiction chapter headings** — Fiction books parsed by `BookParser` or `UniversalParser` can produce headings like "Chapter 1 — The Storm" that match `_RE_CHAPTER`. Since `classify_section_level` is only called inside `_chunk_tech_book`, this is not a concern for fiction (they go through `_chunk_book`). No risk.

- **Risk: `detect_admonition` false positives in non-admonition text** — e.g., a passage beginning with "NOTE: The author acknowledges..." in a fiction book. Again, only called inside `_chunk_tech_book`. Low risk. The 500-char cap prevents deep scanning.

- **Risk: LiteLLM offline during objective extraction** — extraction is a background fire-and-forget task. Failure to extract means 0 objectives, which renders as empty panel (not an error). Mitigation: catch `ServiceUnavailableError` in `LearningObjectiveExtractorService.extract()` and return `[]` with a log warning. Do NOT propagate the exception to the ingestion pipeline.

- **Risk: Tech debt TD-010 (no FK CASCADE)** — `LearningObjectiveModel` rows must be explicitly deleted in both `delete_document` and `bulk_delete_documents`. The plan includes this (Step 8). Failure to add would orphan rows when documents are deleted.
