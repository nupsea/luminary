# S101 — Study Session Planner

## Goal

Add a `GET /study/session-plan` endpoint backed by a pure `_build_session_plan()` function, and modify Chat.tsx so the 4th suggestion pill opens a slide-up session plan panel instead of submitting to chat.

No new Service or Repo files. No new SQLAlchemy models. No changes to `models.py`.

### Step 2: [API] Add `_build_session_plan()` pure function to `backend/app/routers/study.py`

Place this function in the module-level helpers section (alongside `_compute_gaps` and `_parse_teachback_response`). Signature and rules:

```python
def _build_session_plan(
    due_count: int,
    gap_areas: list[str],
    recent_doc_titles: list[tuple[str, str]],  # [(id, title), ...]
    budget_minutes: int,
) -> list[SessionPlanItem]:
```

Rules encoded as pure logic (no I/O):

1. If `due_count > 0`: append `SessionPlanItem(type='review', title=f'{due_count} flashcards due for review', minutes=min(10, max(5, due_count // 2)), action_label='Start Review', action_target='/study')`
2. For each gap in `gap_areas[:2]`: append `SessionPlanItem(type='gap', title=f'Weak area: {gap}', minutes=5, action_label='Study Gaps', action_target='/study')`
3. If `recent_doc_titles`: append `SessionPlanItem(type='read', title=f'Continue: {recent_doc_titles[0][1]}', minutes=5, action_label='Open Document', action_target=f'/learning?document_id={recent_doc_titles[0][0]}')`
4. Cap the list at 5 items: `return items[:5]`

The `budget_minutes` parameter is passed through to the response `total_minutes` field but does not constrain which items are included (the PRD spec does not describe budget-based trimming beyond the 5-item cap).

### Step 3: [API] Add `GET /study/session-plan` endpoint to `backend/app/routers/study.py`

Place this endpoint after the `/history` endpoint. Use `Query` from fastapi (already imported) for the `minutes` parameter.

```python
@router.get("/session-plan", response_model=SessionPlanResponse)
async def get_session_plan(
    minutes: int = Query(default=20, ge=5, le=120),
    session: AsyncSession = Depends(get_db),
) -> SessionPlanResponse:
```

The endpoint performs three DB queries then delegates to `_build_session_plan`:

**Query A — due card count:**
Use the same filter logic as the existing `get_due_cards()` (lines 216-230 of study.py). The existing query pattern is:
```python
now = datetime.utcnow()
stmt = select(FlashcardModel).where(FlashcardModel.due_date <= now)
stmt = stmt.order_by(FlashcardModel.due_date.asc())
result = await session.execute(stmt)
cards = result.scalars().all()
due_count = len(cards)
```
No `document_id` filter — count all due cards across the library.

**Query B — gap areas (max 2):**
The existing `get_gaps()` endpoint is scoped to a single `document_id`. For the session planner we need gaps across all documents. The query pattern is:
```python
weak_stmt = (
    select(FlashcardModel)
    .where(FlashcardModel.fsrs_stability < _GAP_STABILITY_THRESHOLD)
    .where(FlashcardModel.reps > _GAP_MIN_REPS)
)
weak_result = await session.execute(weak_stmt)
weak_cards = list(weak_result.scalars().all())
```
Then join to `SectionModel` via `ChunkModel` to get section headings, exactly as in `get_gaps()`. Collect distinct non-null headings from the first 2 results sorted by `avg_stability` ascending (use the existing `_compute_gaps()` helper). Extract `gap_areas = [g.section_heading for g in _compute_gaps(weak_cards, chunk_to_section) if g.section_heading][:2]`.

**Query C — recent complete documents:**
```python
doc_stmt = (
    select(DocumentModel)
    .where(DocumentModel.stage == "complete")
    .order_by(DocumentModel.last_accessed_at.desc())
    .limit(3)
)
```

**Key observation about `updated_at`:** The PRD description says `ORDER BY updated_at DESC` but `DocumentModel` does NOT have an `updated_at` column — it has `last_accessed_at` (line 35 of models.py). Use `last_accessed_at` for the sort. This matches the intent (most recently accessed documents) and avoids a runtime AttributeError.

After fetching docs: `recent_doc_titles = [(d.id, d.title) for d in docs_result.scalars().all()]`

**Assemble response:**
```python
items = _build_session_plan(due_count, gap_areas, recent_doc_titles, minutes)
return SessionPlanResponse(total_minutes=minutes, items=items)
```

**Import addition:** `DocumentModel` is not currently imported in study.py. Add it to the existing import block at line 25:

## DB Query Details — Precise Field Mapping

| Purpose | Table | Filter | Order | Limit |
|---------|-------|--------|-------|-------|
| Due card count | `flashcards` | `due_date <= now()` | `due_date ASC` | none |
| Weak cards for gaps | `flashcards` | `fsrs_stability < 2.0 AND reps > 1` | — | none |
| Chunk-to-section map | `chunks` OUTER JOIN `sections` | `chunk.id IN (weak_card chunk_ids)` | — | — |
| Recent docs | `documents` | `stage = 'complete'` | `last_accessed_at DESC` | 3 |

Note: the PRD description says `ORDER BY updated_at DESC` for documents but `DocumentModel` has `last_accessed_at` (not `updated_at`). Use `last_accessed_at` — this is the correct column and avoids an `AttributeError` at runtime.

## Risk Flags

- Risk: `DocumentModel` has `last_accessed_at` not `updated_at` — the PRD description uses `updated_at` which does not exist on the model. Mitigation: use `last_accessed_at` as the sort column (confirmed in `models.py` line 35). This is documented in Step 3 above and the DB query table.

- Risk: Panel `translate-y-full` hides correctly only if the parent container has `overflow: hidden`. Mitigation: include `overflow-hidden` on the panel's wrapping div and ensure the parent chat layout does not clip the transform. The panel is positioned inside the flex column; if it takes up no height when translated out, it may still shift the layout. Consider wrapping in `absolute bottom-0 left-0 right-0` positioning if layout shift occurs during implementation.

- Risk: Core Belief 23 — suggested pill text must be tested as a routing input. "Plan my session" is intercepted before it reaches the chat router (it calls `openSessionPlan()` instead of `sendMessage()`), so it never reaches the classifier. No routing test is needed for this pill specifically. However, the other 3 pills ("Find gaps in my notes", "Summarize this for me", "Quiz me on the key concepts") do reach the classifier — if they are not already covered by classifier unit tests, they should be. This is pre-existing state, not introduced by S101.

- Risk: `_build_session_plan` imports `Literal` from `typing` — confirm this is not already imported in study.py. Currently study.py does not import from `typing`. Add the import.

- Risk: The gap-fetching in the endpoint runs a join across all documents (no document_id filter), which could be slow for large libraries. For the scope of S101 this is acceptable; a future optimisation could add a `LIMIT` on weak cards before the join.
