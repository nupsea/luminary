# Exec Plan: S187 -- Chat: document-aware contextual recommendations

**Phase:** Phase 4 (Learner Experience Refinement, S184-S190)
**Priority:** dependent on S186 (inline scope)
**Status:** in-progress

## Goal

Replace the Chat page's three static/noisy suggestion components -- `ChatSuggestions` (FSRS
due-card count pills), `ExploreConnectionsChips` (graph-derived entity pairs), and
`ConfusionBanner`/`ConfusionSignals` (confusion-signal nudge banners) -- with a single
`SuggestionPills` component that calls a new backend endpoint `GET /chat/suggestions`. The
endpoint uses pure template logic (no LLM) to derive 4 contextual question strings from the
selected document's `content_type`, top entities from Kuzu, and section headings from SQLite.
When no document is selected (all-scope), it returns cross-document entity overlap suggestions.
When the library is empty it returns onboarding suggestions. The static `EXAMPLE_QUESTIONS`
fallback and the `GET /chat/confusion-signals` endpoint are removed entirely.

- **I-2 (asyncio.to_thread for LanceDB/sync calls)**: All `KuzuService` calls in
  `SuggestionService` are synchronous; each must be wrapped in `asyncio.to_thread`.
- **I-3 (guard Kuzu get_next() with has_next())**: All Kuzu result iteration in
  `SuggestionService` must use `while result.has_next(): row = result.get_next()`.
- **I-10 (loading/error/empty states)**: `SuggestionPills` must have loading (skeletons),
  error (silent null -- pills are non-critical), and empty (null -- valid when onboarding
  strings are not returned yet).
- **I-13 (ruff -> pytest -> tsc order)**: gates must pass in order before setting passes=true.
- **FastAPI route ordering pattern**: `GET /chat/suggestions` is a static path and must be
  registered before any `/{param}` wildcard route in `chat_meta.py` (none currently exist, but
  note for safety).
