# Exec Plan: S214 -- Faithfulness + answer-relevance gates with local Ollama judge default

**Phase:** V3 Phase 8 - Evaluation and Correctness
**Priority:** 57
**Status:** in-progress

## Goal

Enable RAGAS faithfulness and answer_relevance evaluation by default on every `run_eval.py`
run without requiring `--model`. Currently both metrics are gated behind `if args.model` (line
296 of run_eval.py) and have never produced a recorded score in scores_history.jsonl. This story
adds a dedicated `--judge-model` flag (defaulting to `get_settings().LITELLM_DEFAULT_MODEL`,
i.e., an Ollama model per I-16), wraps RAGAS `evaluate()` inside the stub
`GenerationEval.run()` in `evals/lib/runners.py`, adds `--max-questions N` with deterministic
sampling for fast runs, and enforces `--assert-thresholds` gates at faithfulness >= 0.65 and
answer_relevance >= 0.50.

## Cross-Story Risk

- **I-5 (circular imports):** `GenerationEval.run()` lazy-imports ragas, langchain_community,
  datasets inside the method body to avoid import-time failures when these packages are absent
  or when the evals package is imported in a non-eval context. Patch target in tests must be
  `evals.lib.runners.<symbol>` not the ragas module directly.
- **I-6 (module-level get_settings):** `get_settings()` must be imported at module level in
  run_eval.py for the default judge model. Since run_eval.py runs from the `evals/` directory,
  verify that `backend/` is on sys.path before the import (it already adds `_REPO_ROOT` to
  sys.path at lines 30-32, which covers `backend/` via `from app.config import get_settings`
  only if `backend/` is explicitly added -- confirm this).
- **I-16 (local-first):** judge defaults to Ollama model. If someone sets
  `LITELLM_DEFAULT_MODEL=openai/gpt-4o` the default silently becomes non-local -- acceptable
  per I-16 (user-overridable), but worth noting in `--help` text.
- **S213 lib structure:** `GenerationEval` signature change from `run(self, samples)` to
  `run(self, samples, *, judge_model)` breaks the abstract base class contract in `runners.py`.
  Either keep the abstract signature as `run(self, samples, **kwargs)` or add `judge_model`
  to the base `_BaseEval.run()` signature with a default of `""`. Choose the `**kwargs`
  approach to avoid breaking `RetrievalEval.run()`.
- **S215 (citation grounding):** uses judge LLM infrastructure established here. S215's
  `judge_citation()` will need the same `ChatOllama` + `LangchainLLMWrapper` pattern -- factor
  into a `evals/lib/judge.py` helper if the pattern is repeated (out of S214 scope, but flag
  for S215 planner).
- **scores_history.jsonl backward compatibility:** adding `answer_relevance`,
  `context_precision`, `context_recall` fields to new entries is additive. Old entries lack
  these keys; readers must use `.get()`. The existing `append_history` already stores
  `faithfulness`; adding the other three is safe.
