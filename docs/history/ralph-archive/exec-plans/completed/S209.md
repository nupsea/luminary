# S209 Execution Plan -- Global focus timer pill in app header

## Story summary
Mount a global `FocusTimerPill` component in the app's top header so a Pomodoro
focus session is always one click away from any tab. Backed by S208 endpoints.

- `frontend/src/lib/focusUtils.ts` -- surface inference helper (path -> surface),

## State machine

Phases: `idle | focus | paused | break`

Transitions:
- idle -> focus: `start()`. POST /pomodoro/start, set `secondsLeft = focusMinutes * 60`.
- focus -> paused: `pause()`. POST /pomodoro/{id}/pause, halt tick.
- paused -> focus: `resume()`. POST /pomodoro/{id}/resume, resume tick from same second.
- focus -> idle: `stop()`. POST /pomodoro/{id}/abandon.
- focus -> break: when `secondsLeft` hits 0. POST /pomodoro/{id}/complete, play chime
  (unless muted), set `secondsLeft = breakMinutes * 60`.
- break -> idle: when `secondsLeft` hits 0 (no API call -- break is local-only).

Invariants:
- Only `idle` accepts `start()`.
- Only `focus` accepts `pause()` and natural focus completion.
- Only `paused` accepts `resume()`.
- `stop()` is a no-op in `idle`.

## Persistence

LocalStorage key `luminary:focusTimer` holds:
- `sessionId`, `phase`, `secondsLeft`, `focusMinutes`, `breakMinutes`, `surface`,
  `goalId`, `muted`, `lastTickAt` (epoch ms).

On mount, the pill:
1. Reads localStorage. If a session id exists, calls GET /pomodoro/active.
2. If 200 and ids match, hydrates pill state and reconciles `secondsLeft` by
   subtracting (now - lastTickAt) seconds.
3. If 204 or mismatch, clears local storage and returns to idle.

## Chime

A short generated tone via Web Audio API (no binary asset committed). The story
mentions a bundled .mp3 but creating a working .mp3 in this environment is not
practical; a generated tone honours the spirit of the requirement (audible cue,
local-first, mute respected) without adding a binary file or external dependency.

## Surface inference

Pure function `inferSurfaceFromPath(pathname: string): Surface`:
- `/` and `/?...` -> `read`
- `/study` -> `recall`
- `/notes` -> `write`
- `/chat` -> `explore`
- everything else -> `none`

## Acceptance criteria mapping

| AC | Implementation |
|---|---|
| Mounted on every tab | Add inside `<main>` before `<Routes>` so it sits above page content on every route. |
| Idle '25:00 Start' + chevron popover | Pill default render in idle phase. |
| Start posts with surface inferred | `useLocation()` -> `inferSurfaceFromPath()` -> POST /pomodoro/start. |
| Pause/Resume API | Calls /pause and /resume; tick effect halts on paused phase. |
| Stop -> abandon | Calls /abandon; pill returns to idle. |
| Auto-complete + chime + break | Tick decrements; on 0 in focus -> /complete + chime + break phase. |
| Stats popover | TanStack query `["pomodoro-stats"]` 60s staleTime; invalidate on complete. |
| Refresh rehydrates | localStorage + GET /pomodoro/active reconciliation on mount. |
| Minute inputs 1..120 with validation | Inputs in popover; Start disabled if invalid. |
| Loading/error/idle states | Skeleton, inline error message, idle copy 'Ready to focus'. |
| Surface mapping | Per inferSurfaceFromPath. |
| Quality gates | ruff/pytest/tsc all green. |
| Smoke | scripts/smoke/S209.sh exits 0. |
