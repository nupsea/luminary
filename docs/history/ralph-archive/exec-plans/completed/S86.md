# S86 — Entity Disambiguation: Canonical Name Normalisation Before Kuzu Upsert

## Goal

Add a pure-Python `EntityDisambiguator` service that collapses surface-form
variants of the same entity (e.g. "Holmes", "Sherlock Holmes", "Mr. Holmes")
into a single canonical name before writing to Kuzu, storing non-canonical
surface forms as aliases on the Entity node.

- **Service** — new `backend/app/services/entity_disambiguator.py`

  calls canonicalize_batch before Kuzu MERGE)
- **Service (graph)** — `backend/app/services/graph.py` (schema gains `aliases`
  column; upsert_entity signature extended to accept aliases list)

## Implementation Steps

### 1. [Service] Create `entity_disambiguator.py`

File: `backend/app/services/entity_disambiguator.py`

**No imports from any other app layer.** Pure stdlib only (`re`, `logging`).

#### 1a. `_HONORIFICS` constant

```python
_HONORIFICS: frozenset[str] = frozenset({
    "mr", "mrs", "ms", "dr", "sir", "miss", "lady", "lord",
    "prof", "professor", "rev", "captain", "capt", "col", "gen", "sgt", "lt",
})
```

Note: `"sr"` is intentionally NOT included. This is verified by `test_sr_not_in_honorifics`.

#### 1b. `_strip_honorifics(name: str) -> str`

- Lowercase the entire name
- Strip leading/trailing whitespace
- Strip trailing punctuation (period, comma) from each token using `rstrip(".,")`
- Split on whitespace
- Remove tokens from the FRONT of the token list while the current first token
  (after stripping trailing punctuation) is in `_HONORIFICS`
- Re-join remaining tokens
- Return the stripped result (may be empty string if name was honorific-only)

Example: `"Mr. Sherlock Holmes"` -> `["mr.", "sherlock", "holmes"]` -> strip front
"mr" -> `"sherlock holmes"`. `"Sr. Holmes"` -> `["sr.", "holmes"]` -> "sr" not in
`_HONORIFICS` -> returns `"sr. holmes"` (unchanged after lowercase).

Implementation detail: When checking if first token is an honorific, strip the
trailing period from the token before doing the set lookup. So `"mr."` is tested
as `"mr"` against `_HONORIFICS`.

#### 1c. `find_canonical(name: str, entity_type: str, existing_names: list[str]) -> str`

All comparisons are done in lowercase.

Input: a new surface-form name (already lowercase from NER), an entity type
string, and a list of existing canonical names of the SAME entity type that are
already in the Kuzu graph for this document.

Rules applied in order (stop at first match):

- **Rule A — Exact stripped match**: compute `stripped = _strip_honorifics(name)`.
  If `stripped` exactly equals `_strip_honorifics(existing)` for any existing
  name, return that existing name (the one already in the graph wins).

- **Rule B — Substring containment**: check whether `stripped` is a substring of
  `_strip_honorifics(existing)` OR `_strip_honorifics(existing)` is a substring
  of `stripped`. If a containment relationship exists, the longer of the two
  stripped names wins (the longer name is the canonical form). Return whichever
  of (name, existing) has the longer stripped form after applying
  `_strip_honorifics`.

- **Rule C — Token overlap**: tokenize both `stripped` and
  `_strip_honorifics(existing)` by splitting on whitespace. If the number of
  shared tokens >= 2, they are the same entity. The longer stripped name wins
  (same tie-breaking as Rule B).

- **No match**: return `name` unchanged (it becomes its own canonical entry).

`find_canonical` must NOT cross entity_type boundaries. It only receives names
of the same type, so no cross-type check is needed inside the function itself.
The caller (canonicalize_batch) ensures this.

#### 1d. `canonicalize_batch(entities: list[tuple[str, str]], existing_by_type: dict[str, list[str]]) -> list[tuple[str, str, str]]`

Input:
- `entities`: list of `(name, entity_type)` tuples (all lowercase, as produced
  by the NER pipeline)
- `existing_by_type`: dict mapping entity_type -> list of canonical names already
  in the Kuzu graph for this document (used as the lookup pool for disambiguation)

Output: list of `(canonical_name, entity_type, original_name)` triples.
- `canonical_name`: the resolved canonical form
- `entity_type`: unchanged
- `original_name`: the surface form as given (may equal canonical_name if no
  disambiguation occurred)

Algorithm:
1. Make a mutable working copy of `existing_by_type` (deep copy each list) so
   that new canonical names discovered within the batch are added to the pool
   for subsequent entities in the same batch.
2. For each `(name, entity_type)` in `entities`:
   a. Look up the pool for this type: `pool = working[entity_type]` (default `[]`)
   b. `canonical = find_canonical(name, entity_type, pool)`
   c. If `canonical` is not already in `pool`, append it to `pool`
   d. Append `(canonical, entity_type, name)` to results
3. Return results

This ensures that within a single batch, if "Sherlock Holmes" is seen before
"Mr. Holmes", the second entity correctly resolves to "Sherlock Holmes" using
the in-batch pool.

### 2. [Service] Extend `graph.py` — Add aliases to Entity schema

File: `backend/app/services/graph.py`

#### 2a. Update `_create_schema`

Change the Entity node table creation statement to include an `aliases STRING`
column (Kuzu does not have a native array type accessible via the Python client
in all versions; store aliases as a pipe-delimited string `"alias1|alias2"`
which is simple, readable, and avoids type compatibility issues):

```python
"CREATE NODE TABLE IF NOT EXISTS Entity("
"id STRING PRIMARY KEY, name STRING, type STRING, "
"frequency INT64, aliases STRING)"
```

Note: Kuzu `CREATE NODE TABLE IF NOT EXISTS` is idempotent for the table but
does NOT add new columns to an existing table. The `aliases` column will only
appear in newly created databases. For existing databases (e.g. developer
environments), the column is absent. Therefore:
- All reads of `aliases` must use a try/except around the Kuzu query
- `upsert_entity` must catch the case where the column does not exist and
  proceed without aliases (graceful degradation for existing DBs)
- The column is effectively optional at the DB level but always written when
  the schema is fresh

Alternatively, skip the Kuzu schema ALTER (Kuzu does not support ALTER TABLE
ADD COLUMN) and instead encode aliases purely in the working canonical name
lookup in `entity_extract_node` without persisting them to Kuzu at all. This
is simpler and avoids the schema migration problem entirely.

**Decision**: Store aliases in Kuzu only for fresh databases. For the purpose
of this story, the core requirement is that `canonicalize_batch` is called
before Kuzu MERGE and that the canonical name is used for the MERGE key. The
aliases are a secondary concern. Implement as follows:

- The `aliases` column is added to the schema creation statement (affects new
  databases only)
- `upsert_entity` gains an optional `aliases: list[str] | None = None` parameter
- When `aliases` is provided and non-empty, `upsert_entity` attempts to write
  them as a pipe-delimited string; catches `Exception` and logs a debug message
  if the column does not exist (handles existing DB gracefully)

#### 2b. Update `upsert_entity` signature

```python
def upsert_entity(
    self, entity_id: str, name: str, entity_type: str,
    aliases: list[str] | None = None,
) -> None:
```

When persisting: `aliases_str = "|".join(aliases) if aliases else ""`

On CREATE: include `aliases: $aliases_str` in the CREATE statement.
On UPDATE (MATCH + SET): update `e.aliases` by appending new aliases to any
existing ones (split existing on "|", union with new, re-join).

Wrap the aliases SET in a try/except to handle existing databases where the
column does not exist:

```python
try:
    self._conn.execute("MATCH (e:Entity {id: $id}) SET e.aliases = $a", ...)
except Exception:
    logger.debug("aliases column not present, skipping aliases update")
```

### 3. [Runtime] Modify `entity_extract_node` in `ingestion.py`

File: `backend/app/workflows/ingestion.py`

#### 3a. Import EntityDisambiguator (lazy, inside node body)

```python
from app.services.entity_disambiguator import canonicalize_batch  # noqa: PLC0415
```

#### 3b. Build `existing_by_type` from Kuzu

Before the upsert loop, query Kuzu for existing entity names already in the
graph for this document (from prior ingestion runs or other documents). Use
`graph.get_entities_by_type_for_document(doc_id)` — a new method on
`KuzuService` that returns `dict[str, list[str]]` mapping entity_type ->
list of canonical names.

Add `get_entities_by_type_for_document` to `KuzuService` in `graph.py`:

```python
def get_entities_by_type_for_document(self, document_id: str) -> dict[str, list[str]]:
    """Return {entity_type: [canonical_name, ...]} for entities already in graph."""
    result = self._conn.execute(
        "MATCH (e:Entity)-[:MENTIONED_IN]->(d:Document {id: $did}) "
        "RETURN e.name, e.type",
        {"did": document_id},
    )
    by_type: dict[str, list[str]] = {}
    while result.has_next():
        row = result.get_next()
        name, etype = row[0], row[1]
        if name and etype:
            by_type.setdefault(etype, []).append(name)
    return by_type
```

#### 3c. Call canonicalize_batch

```python
entity_tuples = [(ent["name"], ent["type"]) for ent in entities]
existing_by_type = graph.get_entities_by_type_for_document(doc_id)
canonical_triples = canonicalize_batch(entity_tuples, existing_by_type)
```

#### 3d. Rebuild entities list with canonical names

Map each `(canonical_name, entity_type, original_name)` triple back to the
entity dict, updating `name` and `id`. The `entity_id` must be recomputed
using the canonical name so that the same canonical entity always gets the
same UUID regardless of which surface form was seen first:

```python
canonical_entities = []
alias_map: dict[str, list[str]] = {}  # canonical_id -> [non-canonical names]
for (canonical, etype, original), ent in zip(canonical_triples, entities):
    canonical_id = str(uuid.uuid5(uuid.NAMESPACE_DNS, f"{doc_id}:{canonical}"))
    if original != canonical:
        alias_map.setdefault(canonical_id, []).append(original)
    canonical_entities.append({
        **ent,
        "id": canonical_id,
        "name": canonical,
    })
```

#### 3e. Use canonical_entities for Kuzu upsert

Replace the loop over `entities` with a loop over `canonical_entities`:

```python
for ent in canonical_entities:
    aliases = alias_map.get(ent["id"])
    graph.upsert_entity(ent["id"], ent["name"], ent["type"], aliases=aliases)
    graph.add_mention(ent["id"], doc_id)
```

The co-occurrence loop uses `canonical_entities` for its `chunk_id` grouping
(already correct since canonical_entities retains `chunk_id` from the original).

#### Test 3: `test_sr_not_in_honorifics`

```python
from app.services.entity_disambiguator import _HONORIFICS, find_canonical

def test_sr_not_in_honorifics():
    assert "sr" not in _HONORIFICS
    existing = ["sherlock holmes"]
    result = find_canonical("sr. holmes", "PERSON", existing)
    # "sr" is not an honorific, so no stripping occurs
    # "sr. holmes" lowercased stays "sr. holmes"
    # "sr. holmes" is NOT a substring of "sherlock holmes" (the dot and "sr" differ)
    # token overlap: ["sr.", "holmes"] vs ["sherlock", "holmes"] -> 1 shared token ("holmes")
    # Rule C requires >= 2 shared tokens -> no match
    assert result == "sr. holmes"  # does NOT resolve to "sherlock holmes"
```

#### Test 4: `test_canonicalize_batch_deduplicates_variants`

```python
from app.services.entity_disambiguator import canonicalize_batch

def test_canonicalize_batch_deduplicates_variants():
    entities = [
        ("sherlock holmes", "PERSON"),
        ("holmes", "PERSON"),
        ("mr. holmes", "PERSON"),
    ]
    triples = canonicalize_batch(entities, {})
    canonical_names = [t[0] for t in triples]
    # All three should resolve to "sherlock holmes" (longest / first seen)
    assert all(name == "sherlock holmes" for name in canonical_names)
```

#### Test 5: `test_entity_type_boundary`

```python
def test_entity_type_boundary():
    # "london" as a PLACE should NOT merge with "london" as a PERSON
    # (pathological case, but type isolation must hold)
    from app.services.entity_disambiguator import canonicalize_batch
    entities = [
        ("london", "PLACE"),
        ("london", "PERSON"),
    ]
    triples = canonicalize_batch(entities, {})
    # Both resolve independently — PLACE pool and PERSON pool are separate
    place_canonical = triples[0][0]
    person_canonical = triples[1][0]
    # Each becomes its own canonical entry (no cross-type merging)
    assert place_canonical == "london"
    assert person_canonical == "london"
    # Crucially, entity_type on each triple is unchanged
    assert triples[0][1] == "PLACE"
    assert triples[1][1] == "PERSON"
```

```bash
#!/usr/bin/env bash
# Smoke test for S86 — entity disambiguation
# Ingests a small TXT file, waits for completion, then asserts
# GET /graph/{doc_id} returns a JSON object with a non-empty nodes array.
set -euo pipefail

BASE="http://localhost:8000"
FIXTURE="/tmp/s86_smoke_fixture.txt"

# Create a minimal fixture with known entity variants
cat > "$FIXTURE" <<'EOF'
The Adventures of Sherlock Holmes

Chapter 1: The Case of the Missing Clue

Mr. Holmes arrived at Baker Street early in the morning.
Sherlock Holmes examined the evidence carefully.
Holmes concluded that the crime occurred at midnight.
Dr. Watson recorded Holmes's observations in his journal.
Watson and Mr. Holmes then departed for Scotland Yard.
EOF

echo "S86 smoke: uploading fixture document"
UPLOAD=$(curl -sf -X POST "$BASE/ingest" \
  -F "file=@$FIXTURE;type=text/plain" \
  -F "content_type=book")
DOC_ID=$(echo "$UPLOAD" | python3 -c "import sys,json; print(json.load(sys.stdin)['document_id'])")
echo "S86 smoke: document_id=$DOC_ID"

echo "S86 smoke: waiting for ingestion to complete"
for i in $(seq 1 24); do
  STAGE=$(curl -sf "$BASE/documents/$DOC_ID" | python3 -c "import sys,json; print(json.load(sys.stdin).get('stage',''))")
  echo "  stage=$STAGE (attempt $i)"
  if [ "$STAGE" = "complete" ]; then break; fi
  if [ "$STAGE" = "error" ]; then echo "FAIL: ingestion stage=error"; exit 1; fi
  sleep 5
done

if [ "$STAGE" != "complete" ]; then
  echo "FAIL: ingestion did not complete within 120s (stage=$STAGE)"
  exit 1
fi

echo "S86 smoke: GET /graph/$DOC_ID"
GRAPH=$(curl -sf "$BASE/graph/$DOC_ID")
NODES=$(echo "$GRAPH" | python3 -c "import sys,json; d=json.load(sys.stdin); print(len(d.get('nodes',[])))")
echo "S86 smoke: node count=$NODES"
if [ "$NODES" -lt 1 ]; then
  echo "FAIL: expected at least 1 entity node, got $NODES"
  exit 1
fi

echo "S86 smoke: all checks passed (nodes=$NODES)"
rm -f "$FIXTURE"
```

## Risk Flags

- **Risk**: Kuzu does not support `ALTER TABLE ADD COLUMN` — the `aliases` column
  only exists in databases created after this story ships. Existing developer
  databases will fail silently on `upsert_entity` when trying to set `aliases`.
  **Mitigation**: Wrap all `aliases`-related Kuzu writes in `try/except Exception`
  with a `logger.debug` fallback. Do not fail ingestion if aliases cannot be
  written.

- **Risk**: Reassigning entity IDs (from surface-form UUID to canonical UUID)
  mid-ingestion may leave orphaned edges in Kuzu if an entity was already stored
  with the old ID in a prior run.
  **Mitigation**: `upsert_entity` already handles the case where the entity
  exists (it increments frequency). After disambiguation, the canonical ID will
  correctly MATCH and update the existing node. The old surface-form ID node (if
  it exists from a prior run before this story) will linger but become
  disconnected from new MENTIONED_IN edges. This is acceptable as tech debt for
  existing data; fresh ingestions are clean.

- **Risk**: `find_canonical` Rule C (token overlap >= 2) may produce false
  positives for common two-word names like "New York" or "Great Britain" where
  both tokens are shared with unrelated entities.
  **Mitigation**: The entity type boundary is the primary guard. PLACE entities
  with two matching tokens are likely the same place; false positives are rare
  compared to the benefit of merging "Mr. New York" with "New York" (the latter
  would not match Rule C, so this example is actually safe).

- **Risk**: `canonicalize_batch` is called inside `entity_extract_node` which
  already runs in a thread pool executor (CPU-bound). The disambiguator is pure
  Python with no I/O, so running it in the same thread after the NER call
  returns is correct and does not block the event loop.
  **Mitigation**: No additional threading required. `canonicalize_batch` runs
  synchronously after `extractor.extract()` returns and before the Kuzu loop,
  all within the `run_in_executor` callback — but note that the Kuzu loop is
  called directly on the event loop (not in the executor). The executor call
  wraps only `extractor.extract`; the rest of `entity_extract_node` is
  already in the async context. This is the existing pattern and `canonicalize_batch`
  integrates cleanly there.

- **Risk**: The smoke test depends on a running server with Ollama available for
  NER (GLiNER runs locally, no Ollama needed). GLiNER is a local ML model and
  runs without any external service. The smoke test should reliably produce at
  least 1 entity node from the Holmes fixture text.
  **Mitigation**: The fixture contains clear named entities ("Sherlock Holmes",
  "Baker Street", "Dr. Watson", "Scotland Yard") that GLiNER confidently
  extracts. Entity count >= 1 is a very conservative threshold.
