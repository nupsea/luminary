# Exec Plan: S200 -- PDF Viewer: working internal links, external links, and rendering fidelity

**Phase:** V3 Phase 6 - Polish and Bug Fixes
**Priority:** 40
**Status:** in-progress

## Goal

Fix four interrelated bugs in the PDF Viewer's pdfjs-dist v4 integration: (1) internal GoTo/GoToR link annotations fail to scroll the viewer to the target page because `navigateTo()` does not unwrap action dictionaries before resolving destinations; (2) external URL links in some PDFs are not rendered as proper anchors with `target='_blank'`; (3) the `--scale-factor` CSS variable is set on the text-layer div but not synchronized with the canvas element, which can cause the text layer to drift at non-default zoom levels; and (4) highlight `<mark>` elements sit above the annotation layer in z-order, blocking clicks on PDF link annotations. All changes are frontend-only.

## Files to Touch

### Frontend
- `frontend/src/components/reader/PDFViewer.tsx` -- four targeted edits: (a) `navigateTo()` at lines 406-422 to handle GoTo action dicts (`{action, dest}` or `{action, D}`); (b) `addLinkAttributes()` at lines 399-405 to ensure anchors always set `target='_blank'` and `rel='noopener noreferrer'` for http/https/mailto/tel URLs; (c) canvas render path at lines 323-338 to also set `--scale-factor` on the canvas wrapper / annotation layer divs after viewport is computed; (d) CSS z-order: annotation layer must sit above highlight `<mark>` elements.
- `frontend/src/components/reader/pdfLinkService.ts` -- new utility file extracting `createLinkService(pdfDoc, goToPage)` so the logic is unit-testable independently of the React render cycle. Mirrors the `pdfTocUtils.ts` extraction pattern.
- `frontend/src/components/reader/pdfLinkService.test.ts` -- new Vitest test file covering the two required unit tests: GoTo action dict resolution and external-link anchor attributes.

### Other
- `scripts/smoke/S200.sh` -- smoke test verifying the backend `/documents/{id}/file` endpoint remains reachable (the viewer's only backend dependency) and that a running dev server serves the PDF viewer page without a 5xx.

## Cross-Story Risk

- **S198** (highlight reliability): this story modifies `applyPdfHighlights()` behavior indirectly through z-order changes to the text/annotation layer containers. The highlight `<mark>` elements live inside the text layer; increasing the annotation layer's z-index must not make the marks invisible. Mitigation: highlights use `background-color` on `<span>` elements inside the text layer, not absolute-positioned overlays, so they remain visible regardless of annotation layer z-index.
- **S202** (CI and console error sweep): S200 changes must leave tsc clean. The `pdfLinkService.ts` extraction must not introduce `any` type assertions without accompanying `eslint-disable` comments (same pattern already used in PDFViewer.tsx lines 432/437). S202 will catch any regressions.
- Relevant invariants: **I-10** (loading/error/empty states) -- PDFViewer already has all three; no regression expected. **I-13** (quality gates order: ruff -> pytest -> tsc) -- observed in step 7.
