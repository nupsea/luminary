# S161 -- Note collections: schema, API, and group_name migration

## Goal

Introduce a proper collections model for notes (replacing the flat `group_name` field with a
2-level hierarchy), add all CRUD endpoints, and migrate existing `group_name` values into the
new table. The `group_name` column and the GET /notes/groups endpoint are preserved for backwards
compatibility throughout V3 Phase 1.

## Acceptance Criteria (summary)

1. NoteCollectionModel in models.py with self-FK parent_collection_id
2. NoteCollectionMemberModel pivot with composite PK and cascade deletes
3. db_init.py migration: create tables + migrate group_name values
4. POST /collections -- 422 when parent already has a parent (max 2-level nesting)
5. DELETE /collections/{id} -- cascades members, does NOT delete notes
6. GET /collections/tree -- nested {id, name, color, note_count, children}
7. GET /notes?collection_id={id} -- filters via NoteCollectionMemberModel join
8. POST /collections/{id}/notes -- ON CONFLICT DO NOTHING for duplicates
9. group_name preserved; GET /notes/groups still works
10. Unit tests for migration and tree endpoint
11. All quality gates pass; smoke test exits 0

## Design Notes

- NoteCollectionModel uses String PK (uuid4 hex), not Integer, consistent with other models
- NoteCollectionMemberModel uses a UniqueConstraint("note_id", "collection_id") instead of a
  true composite PK (SQLAlchemy mapped_column composite PK is verbose); the unique constraint
  enforces the pivot invariant and the ON CONFLICT DO NOTHING semantics
- group_name migration runs inside the same `try` block as other additive migrations so it is
  idempotent: if the note_collections table already exists the INSERT ... SELECT silently skips
  existing rows via INSERT OR IGNORE
- tree endpoint loads all collections in one query, then assembles the tree in Python to avoid
  recursive CTEs (SQLite supports them but adds complexity)
- DELETE /collections/{id} explicitly deletes from note_collection_members first (no FK pragma
  enforcement in SQLite by default), then deletes the collection row
