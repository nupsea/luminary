# Exec Plan: S177 -- Progress tab: replace Monitoring with learner-centric view, move dev metrics to admin route

**Phase:** V3 Phase 3 - UX Polish and Learner Focus
**Priority:** 17
**Status:** in-progress

## Goal

The Monitoring tab currently serves two incompatible audiences. This story splits it: the "Progress" tab becomes a learner mastery dashboard (activity chart, mastery score, due cards, notes over time, GoalsPanel), and all dev-centric metrics (RAG quality, Phoenix traces, Langfuse, eval runs) move to a hidden /admin route. GoalsPanel moves from Study tab to Progress tab.

## Files to Touch

### Backend
- No backend changes needed. Existing GET /monitoring/* endpoints remain unchanged.

### Frontend
- `frontend/src/pages/Monitoring.tsx` -- gutted and repurposed as Progress.tsx; file renamed
- `frontend/src/pages/Progress.tsx` -- NEW: learner-centric view (mastery, activity, goals, counts)
- `frontend/src/pages/Admin.tsx` -- NEW: all dev panels moved here
- `frontend/src/App.tsx` -- rename nav tab "Monitoring"->"Progress", update lazy import, add /admin route, add footer "Dev Tools" link
- `frontend/src/pages/Study.tsx` -- remove GoalsPanel rendering from Study

### Other
- `scripts/smoke/S177.sh` -- smoke test
