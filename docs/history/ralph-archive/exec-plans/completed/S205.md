# Exec Plan: S205 -- PDF text layer and search highlight rendering: fix font overlap and visual artifacts

**Phase:** V3 Phase 6 - Polish and Bug Fixes
**Priority:** 45
**Status:** in-progress

## Goal

Replace inline `<mark>` element injection into pdfjs text layer spans with absolutely-positioned overlay divs rendered in a dedicated highlight container. The text layer spans are never modified, preserving pdfjs font metrics and eliminating the font-doubling / blurred-text artifact. Both annotation highlights and search highlights are rendered as colored overlay rectangles using the Range API to compute bounding rects from the unmodified spans. This also eliminates the conflict where `applySearchHighlights` and `applyPdfHighlights` could overwrite each other's `<mark>` nodes on shared spans.

2. **Create `pdfHighlightOverlay.ts`** with three exported functions:
   - `computeHighlightRects(textLayerDiv, fullText, parts, matchStart, matchEnd): DOMRect[]` -- uses `Range` API: `range.setStart(span.firstChild, localStart); range.setEnd(span.firstChild, localEnd); range.getBoundingClientRect()` relative to the container. Returns array of rects (one per covered span). Does NOT modify any span.
   - `renderOverlayDivs(container, rects, containerRect, color, dataAttr, isActive?): HTMLDivElement[]` -- for each rect, creates an absolutely-positioned div using `rect.left - containerRect.left` and `rect.top - containerRect.top` for positioning. Sets `pointer-events: none`. Sets background color. Sets `data-*` attribute for identification. Returns created divs so callers can attach click handlers.
   - `clearOverlays(container, dataAttr?)` -- removes all overlay divs matching `[data-*]` selector, or all highlight overlays if no attr given.
