# Exec Plan: S201 -- Fix tag auto-save on note creation and eliminate duplicate notes

**Phase:** V3 Phase 6 - Polish and Bug Fixes
**Priority:** 41
**Status:** in-progress

## Goal

Three bugs in the notes workflow need fixing. First, when a user accepts auto-suggested tags in NoteEditorDialog the tags are shown as dashed-border chips but clicking them only calls `addTagMut` which issues a PATCH -- that is already correct, but the `suggest_tags` endpoint returns raw LLM strings without normalizing through `normalize_tag_slug`, so the stored tag format may be inconsistent. Second, rapid double-clicks on the Save button in NoteCreationDialog can submit two concurrent POST /notes requests, creating duplicate notes with different UUIDs; the backend has no deduplication guard. Third, the clustering service `batch_accept_suggestions` stores collection names and tags exactly as the LLM returns them, bypassing `normalize_collection_name` and `normalize_tag_slug`.
