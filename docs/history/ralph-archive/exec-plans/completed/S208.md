## Schema

`pomodoro_sessions`:
- `id` TEXT PK (uuid str)
- `started_at` DATETIME
- `completed_at` DATETIME nullable
- `focus_minutes` INT default 25
- `break_minutes` INT default 5
- `status` TEXT (active|paused|completed|abandoned)
- `surface` TEXT (read|recall|write|explore|none) default none
- `document_id` TEXT nullable
- `deck_id` TEXT nullable
- `goal_id` TEXT nullable -- no FK constraint until S210
- `paused_at` DATETIME nullable
- `pause_accumulated_seconds` INT default 0
- `created_at` DATETIME

Indexes: `created_at`, `status`, `document_id`, `deck_id`, `goal_id` -- declared via `index=True`.

## Service Surface

- `start_session(focus_minutes, break_minutes, surface, document_id, deck_id, goal_id)`
  -- raises `ActiveSessionExists(existing_id)` if any active or paused row exists.
- `pause_session(id)` -- sets status=paused, paused_at=now.
- `resume_session(id)` -- adds (now - paused_at) to pause_accumulated_seconds, clears paused_at, status=active.
- `complete_session(id)` -- raises `InvalidTransition` if status already completed/abandoned; sets completed_at.
- `abandon_session(id)` -- sets status=abandoned.
- `get_active_session()` -- returns single row with status in (active, paused) or None.
- `get_stats()` -- returns dict {today_count, streak_days, total_completed}.

## Stats math

- today_count: `count(*) where status='completed' and created_at >= local midnight today`.
  Use UTC midnight in the query for portability; store/query consistently.
- total_completed: `count(*) where status='completed'`.
- streak_days: collect distinct UTC-date values of created_at for completed rows;
  walk back from today, counting consecutive days with at least one completion.

## Router

Mount via `app.include_router(pomodoro_router)` in main.py.

- `POST /pomodoro/start` body `{focus_minutes?, break_minutes?, surface?, document_id?, deck_id?, goal_id?}` -> 200 with session payload, or 409 with `{detail, existing_session_id}`.
- `POST /pomodoro/{id}/pause` -> 200.
- `POST /pomodoro/{id}/resume` -> 200.
- `POST /pomodoro/{id}/complete` -> 200; second call returns 409.
- `POST /pomodoro/{id}/abandon` -> 200.
- `GET /pomodoro/active` -> 200 with payload or HTTP 204 if none.
- `GET /pomodoro/stats` -> 200 with `{today_count, streak_days, total_completed}`.
