# S188 - Flashcard generation: context-rich questions with source grounding

## Summary
Enrich the flashcard generation prompt with section headings, entity context (books), and code blocks (technical docs). Add `section_heading` field to FlashcardModel and response. Update frontend to display section_heading.

## Backend Changes

### 1. Model: Add `section_heading` to FlashcardModel
- New nullable column `section_heading: Mapped[str | None] = mapped_column(String(300), nullable=True)`
- Migration in db_init.py: `ALTER TABLE flashcards ADD COLUMN section_heading TEXT`

### 2. Response Schema: Add `section_heading` to FlashcardResponse
- In `flashcards.py` router: add `section_heading: str | None = None` to FlashcardResponse

### 3. Prompt Enrichment in `generate()`
- After fetching chunks, query SectionModel to get section heading + parent section heading
- For books: query Kuzu `get_entities_by_type_for_document()` for PERSON and PLACE types, take top 5, inject into system prompt
- For technical docs: detect `has_code=True` on chunks and include code block excerpts
- Update LLM system prompt to instruct Bloom L3+ and section/chapter citation in answers
- Prepend section heading context to the text passed in the user prompt

### 4. Populate `section_heading` on card creation
- After LLM response, look up chunk's section_id -> SectionModel.heading
- Also look up parent_section_id -> parent heading for chapter context
- Set `card.section_heading = f"{parent_heading} - {section_heading}"` or just section heading

### 5. Unit Tests
- Test generate() for book document produces cards with bloom_level >= 3 for >= 50%
- Test FlashcardResponse includes section_heading populated from section data

## Frontend Changes

### 6. Add `section_heading` to Flashcard interface
- In Study.tsx: add `section_heading: string | null` to Flashcard interface

### 7. Display in FlashcardCard
- Show `card.section_heading` as muted text above the question when present

- ruff check, pytest, tsc --noEmit
- Smoke test: scripts/smoke/S188.sh
