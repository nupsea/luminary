# Exec Plan: S164 -- Collections UI: sidebar tree, CRUD dialog, multi-collection assignment in note editor

**Phase:** V3 Phase 1 - Note Organization and Knowledge Graph
**Priority:** 5
**Status:** in-progress

## Goal

Build the React frontend for the collections system introduced by S161. This replaces the flat `group_name` filter list in the Notes sidebar with a collapsible CollectionTree component (modelled after Obsidian's folder panel), adds a CreateCollectionDialog with an 8-swatch color picker and parent selector, and adds a Collections checkbox section to NoteEditorDialog where collection membership is saved immediately on check/uncheck without requiring a Save click. The backend already has all the required endpoints (POST/GET/PUT/DELETE /collections, /collections/{id}/notes, /collections/{id}/notes/{note_id}); this story is entirely frontend plus one small backend addition: adding `collection_ids: list[str]` to the NoteResponse schema so the editor can pre-check the right boxes.

## Key Implementation Notes

- `AlertDialog` is not in `frontend/src/components/ui/` -- use the existing `Dialog` component for delete confirmation
- The `_to_response` helper in `notes.py` is currently synchronous. To avoid making it async, the callers (`list_notes`, `get_note`) should pre-fetch collection memberships and pass them in. A clean pattern: add `collection_ids: list[str] = []` param to `NoteResponse` and pass it from the router after a separate bulk query
- For `list_notes`, fetch all collection memberships for all returned note IDs in one query: `SELECT note_id, collection_id FROM note_collection_members WHERE note_id IN (:ids)`, then group by note_id into a dict
- For `get_note` (GET /notes/{note_id}), a single query: `SELECT collection_id FROM note_collection_members WHERE note_id = :id`
- The CreateCollectionDialog needs the flat collection list (not tree) for the parent selector. Use GET /collections/tree and flatten client-side (take only top-level items where children is the original tree -- the top-level items are those without a parent). This avoids needing a separate GET /collections flat endpoint
- Drag-and-drop: no DnD library required; use native HTML5 `draggable`, `onDragStart`, `onDragOver`, `onDrop` attributes
