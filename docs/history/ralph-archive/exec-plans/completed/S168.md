# Exec Plan: S168 -- Smart Tag Normalization

**Phase:** V3 Phase 1 - Note Organization and Knowledge Graph
**Priority:** 8
**Status:** in-progress

## Goal

Add a SmartTagNormalizerService that embeds all canonical tag display_names using the
existing EmbeddingService (bge-small-en-v1.5), computes pairwise cosine similarity,
and stores pairs above 0.85 threshold as TagMergeSuggestionModel rows. A new
NormalizationDrawer component in the Notes sidebar lets users review and accept or
reject each pair; accepting reuses the existing POST /tags/merge path.

## Cross-Story Risk

- S162 (TagAliasModel, CanonicalTagModel, _sync_tag_index): S168 reads TagAliasModel to skip already-linked pairs and calls the same merge pattern. If S162 is not fully passing before S168, the alias skip check will behave incorrectly.
- S165 (POST /tags/merge): S168 accept path replicates the merge logic from tags.py rather than calling the HTTP endpoint internally; this avoids an in-process HTTP call. If S165's merge logic changes, S168's accept_suggestion must be updated to match.
- S167 (TagGraphService cache invalidation): accept_suggestion calls invalidate_tag_graph_cache(). The invalidation function signature must remain stable. No functional risk since it is a simple cache clear.

## Implementation Notes

- The existing EmbeddingService (embedder.py) uses bge-small-en-v1.5, not bge-m3 as the story description says. Use the existing service -- the story AC only requires cosine similarity via bge embeddings; the model name discrepancy does not affect correctness of the similarity scan.
- `asyncio.create_task` in the scan endpoint requires a running event loop and its own AsyncSession (not the request session, which will close after the response). Use `get_db()` as an async context manager inside the background task, not a function-scoped Depends session.
- Pairwise cosine similarity over N tags is O(N^2). For the expected tag count (<1000) this is fine. Use numpy dot product on normalized vectors: `sims = embeddings @ embeddings.T` then zero out the diagonal and lower triangle to avoid duplicate pairs.
- suggested_canonical_id defaults to tag_b_id (the tag with higher note_count) but can remain nullable if logic is deferred.
- NormalizationDrawer uses Dialog from `@/components/ui/dialog` (Sheet does not exist in the ui/ directory -- the story's "Sheet" reference is aspirational; Dialog is the correct component to use).
