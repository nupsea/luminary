# Exec Plan: S213 -- Unified eval harness + Pydantic golden schema (evals/lib/)

**Phase:** V3 Phase 8 - Evaluation and Correctness
**Priority:** 56
**Status:** in-progress

## Goal

`evals/run_eval.py` is a 683-line monolith covering retrieval-only evals. Four upcoming stories (S214-S218) each need the same infrastructure: golden loading with schema validation, scoring history, threshold enforcement, and results storage. This story extracts that infrastructure into an `evals/lib/` package, refactors `run_eval.py` to import from it, and adds the `eval_kind` field to the backend `EvalRunModel` and `/monitoring/evals/store` endpoint. The goal is to make future eval runners 30-line thin wrappers rather than copies of `run_eval.py`.

### Evals package (new files)
- `evals/lib/__init__.py` -- public exports: `GoldenEntry`, `RetrievalGoldenEntry`, `SummaryGoldenEntry`, `FlashcardGoldenEntry`, `IntentGoldenEntry`, `load_golden`, `RetrievalEval`, `GenerationEval`, `ClassifierEval`, `compute_hit_rate_5`, `compute_mrr`, `append_history`, `store_results`.
- `evals/lib/schemas.py` -- `GoldenEntry` base Pydantic model + 4 subclasses. `RetrievalGoldenEntry.context_hint` carries over the `str | list[str]` coercion validator from `run_eval.GoldenEntry` (S226 invariant). All subclasses use `model_config = {"extra": "allow"}`.
- `evals/lib/loader.py` -- `load_golden(dataset_name: str, schema: type[BaseModel] = RetrievalGoldenEntry) -> list[BaseModel]`. Raises `ValidationError` with message `"<path>:<lineno>: <error>"` on bad entry; exits 1 via `sys.exit` for CLI use (same contract as current `run_eval.load_golden`).
- `evals/lib/manifest.py` -- move `load_manifest`, `save_manifest`, `is_document_alive`, `lookup_document_by_filename`, `ensure_ingested` from `run_eval.py` verbatim; parameterise `GOLDEN_DIR` and `MANIFEST_PATH` as module-level constants computed from `__file__`.
- `evals/lib/scoring_history.py` -- `append_history(dataset, model, metrics, passed, eval_kind="retrieval")`. Adds `eval_kind` to the persisted JSON object. Backwards-compat: callers that do not pass `eval_kind` get `"retrieval"` written to the file.
- `evals/lib/store.py` -- `store_results(backend_url, dataset, model, eval_kind, metrics)`. Passes `eval_kind` in the POST body to `/monitoring/evals/store`.
- `evals/lib/runners.py` -- abstract base classes `RetrievalEval`, `GenerationEval`, `ClassifierEval`. Each has `.run(samples) -> dict` (NotImplemented) and `.assert_thresholds(metrics, thresholds) -> None` (raises `AssertionError` with a violation message). No heavy imports at module level; RAGAS imports stay lazy inside `GenerationEval.run`.
- `evals/lib/retrieval_metrics.py` -- `compute_hit_rate_5`, `compute_mrr`, `_norm`, `_extract_hint_norms` moved from `run_eval.py`. These functions take `list[dict]` and are pure (no I/O). Both honour `list[str]` `context_hint` per S226.
- `evals/lib/audit.py` -- `python -m evals.lib.audit` entry point. Iterates over the 7 golden files in `evals/golden/` (the same set as `VALID_DATASETS` in `run_eval.py`), calls `load_golden(dataset, RetrievalGoldenEntry)` for each, prints `PASS <dataset>` or `FAIL <dataset>: <error>`. Exit 0 if all pass, exit 1 if any fail. The `if __name__ == "__main__"` block delegates to a `main()` function so it is importable for testing.

## Test Strategy

### Unit tests (`backend/tests/test_evals_lib.py`)

All tests import directly from `evals.lib` via the `sys.path.insert` pattern used by `test_eval_metrics.py`.

- `test_retrieval_golden_entry_accepts_str_hint` -- `RetrievalGoldenEntry(question="q", ground_truth_answer="a", context_hint="text")` coerces to `["text"]`.
- `test_retrieval_golden_entry_accepts_list_hint` -- list form passes through unchanged.
- `test_retrieval_golden_entry_rejects_empty_list` -- `context_hint=[]` raises `ValidationError`.
- `test_retrieval_golden_entry_rejects_non_str_element` -- `context_hint=[1]` raises `ValidationError`.
- `test_load_golden_raises_on_bad_entry` -- write a tmp JSONL with a missing-question entry; assert `SystemExit` with code 1 (since loader calls `sys.exit(1)` on validation error).
- `test_load_golden_error_includes_line_number` -- the stderr output from the loader includes the file path and a line number (use `capsys`).
- `test_append_history_writes_eval_kind` -- call `append_history("book_alice", "no-llm", {"hit_rate_5": 0.6, "mrr": 0.4}, True, eval_kind="retrieval")` against a tmp file; read it back and assert `eval_kind == "retrieval"`.
- `test_append_history_defaults_eval_kind_to_retrieval` -- call without `eval_kind` kwarg; assert written entry has `eval_kind == "retrieval"`.
- `test_summary_golden_entry_requires_expected_themes` -- `SummaryGoldenEntry` with missing `expected_themes` raises.
- `test_intent_golden_entry_requires_expected_route` -- `IntentGoldenEntry` with missing `expected_route` raises.

### Integration tests (`backend/tests/test_evals_router.py`, additions)

- `test_store_eval_run_with_eval_kind` -- POST `/monitoring/evals/store` with `eval_kind="retrieval"`; assert response includes `eval_kind == "retrieval"`.
- `test_store_eval_run_without_eval_kind` -- POST without `eval_kind`; assert response does not 422 and `eval_kind` is `"retrieval"` (default) or `None` (acceptable per nullable column).

### Backwards-compat checks (verified by keeping existing tests green)

- `test_eval_metrics.py`: imports `GoldenEntry`, `compute_hit_rate_5`, `compute_mrr` from `run_eval` -- must keep passing.
- `test_evals_router.py` existing tests -- must keep passing.
- `test_corpus_qa.py` -- no eval-lib dependency, but must keep passing to confirm no inadvertent breakage.

## Cross-Story Risk

- **S226 (multi-hint schema)**: `RetrievalGoldenEntry.context_hint` MUST preserve the `str | list[str]` coercion validator from `run_eval.GoldenEntry`. If the copy to `evals/lib/schemas.py` drops the validator, S226's goldens will silently lose their alternate hints and HR@5 will regress. Invariant I-3 does not apply here but the schema coercion is the exact analogue -- guard it with `test_retrieval_golden_entry_accepts_list_hint` and `test_retrieval_golden_entry_rejects_empty_list`.
- **S212 (retrieval baseline)**: `audit_golden.py` imports `load_golden`, `_norm`, `GOLDEN_DIR`, `VALID_DATASETS`, `load_manifest` from `run_eval`. After refactoring `run_eval.py`, all five symbols must remain importable from `run_eval` (either kept inline or re-exported from lib). A missing re-export breaks `audit_golden.py` silently.
- **S214-S218 (downstream eval stories)**: they depend on the `evals.lib` public API established here. `runners.py` base classes and `scoring_history.append_history` signature must be stable. Adding `eval_kind` as a kwarg-with-default is backwards-compatible.
- **Relevant invariants**: I-6 (settings module-level import) applies to `evals/lib/store.py` if it ever calls `get_settings()` -- currently it does not, but note the rule. I-13 (quality gates run in order: ruff -> pytest -> tsc) applies. I-14 (smoke must exit 0 before `passes=true`). I-15 (use `uv`, never `pip`).
