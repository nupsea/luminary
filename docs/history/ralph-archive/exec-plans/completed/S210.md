# Exec Plan: S210 -- Learning goals backend: typed goals, progress aggregation, session linking

**Phase:** V3 Phase 7 - Focused Learning
**Priority:** 50
**Status:** in-progress

## Goal

Replace the existing document-centric `LearningGoalModel` (used only for FSRS readiness
projection) with a full typed-goals model that supports four goal types
(read|recall|write|explore), optional links to documents/decks/collections, a status
lifecycle (active|paused|completed|archived), and type-dispatched progress computation
aggregated from Pomodoro sessions, review events, notes, and chat turns. Also adds the
real FK from `pomodoro_sessions.goal_id` to `learning_goals.id` (ON DELETE SET NULL
equivalent enforced at service level since SQLite cannot alter FK constraints).

5. **`compute_progress` dispatch logic:**

   **recall** (`goal.deck_id` scoped):
   - Fetch all `PomodoroSessionModel` rows where `goal_id=goal_id AND status='completed' AND surface='recall'`.
   - Collect the session window tuples `(started_at, completed_at)`.
   - Fetch `ReviewEventModel` rows where `reviewed_at` falls within any session window:
     `SELECT * FROM review_events WHERE reviewed_at >= :s_start AND reviewed_at <= :s_end`
     (one query per session, or a single OR-condition query).
   - If `goal.deck_id` set: further filter by `flashcard_id IN (SELECT id FROM flashcards WHERE deck=:deck_id)`.
   - `cards_reviewed = count(distinct flashcard_id)`.
   - `avg_retention = mean(is_correct)` across all review events (float 0.0-1.0).
   - `completed_pct = (cards_reviewed / goal.target_value * 100)` if `target_value` else None.
   - Returns `{cards_reviewed, avg_retention, completed_pct}`.

   **read** (session-based, surface='read'):
   - Fetch completed sessions where `goal_id=goal_id AND status='completed' AND surface='read'`.
   - `minutes_focused = sum(focus_minutes)`.
   - `sessions_completed = count(*)`.
   - `completed_pct = (minutes_focused / target_value * 100)` if `target_value` and `target_unit='minutes'`.
   - Returns `{minutes_focused, sessions_completed, completed_pct}`.

   **write** (notes created in session windows, surface='write'):
   - Fetch completed sessions where `goal_id=goal_id AND status='completed' AND surface='write'`.
   - For each session window `(started_at, completed_at)`: count `NoteModel` rows where
     `created_at >= started_at AND created_at <= completed_at`.
   - Sum across all windows.
   - `notes_created = total count`.
   - `sessions_completed = count(sessions)`.
   - `completed_pct = (notes_created / target_value * 100)` if `target_value`.
   - Returns `{notes_created, sessions_completed, completed_pct}`.
   - NOTE: Use `_as_utc()` on all datetime comparisons (SQLite returns tz-naive).

   **explore** (chat turns in session windows, surface='explore'):
   - Fetch completed sessions where `goal_id=goal_id AND status='completed' AND surface='explore'`.
   - For each session window: count `QAHistoryModel` rows where
     `created_at >= started_at AND created_at <= completed_at`.
   - `turns = total count`.
   - `sessions_completed = count(sessions)`.
   - `completed_pct = (turns / target_value * 100)` if `target_value`.
   - Returns `{turns, sessions_completed, completed_pct}`.
   - NOTE: Use `_as_utc()` on datetime comparisons. `QAHistoryModel` is the correct
     table -- `qa_history` stores all chat Q&A turns (confirmed at models.py line 238).

## FK Strategy on Existing Table (SQLite Limitation)

SQLite does not support `ALTER TABLE ADD FOREIGN KEY`. The goal_id -> learning_goals.id
relationship is enforced at the **service layer**:

1. `link_session(goal_id, session_id)`: verifies the goal exists before updating
   `pomodoro_sessions.goal_id` (raises `GoalNotFound` if goal row is missing).
2. `delete_goal(goal_id)`: runs `UPDATE pomodoro_sessions SET goal_id=NULL WHERE goal_id=:gid`
   before deleting the goal row -- equivalent to ON DELETE SET NULL.
3. No DDL `FOREIGN KEY` clause is added to the `pomodoro_sessions` table; the column
   remains a plain nullable TEXT index.

This matches the pattern used throughout the codebase (SQLite FK pragmas are off by
default in production; db_init.py enables them only within the `create_all_tables`
connection, not globally).
