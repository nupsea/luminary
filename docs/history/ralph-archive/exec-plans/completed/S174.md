# Exec Plan: S174 -- Export: collections as Obsidian-compatible Markdown vault and as Anki deck

**Phase:** V3 Phase 2 - Note Intelligence and Synthesis
**Priority:** 14
**Status:** in-progress

## Goal

Notes and flashcards are currently locked inside Luminary's SQLite database with no path to mobile or external review tools. This story adds two export formats reachable from the CollectionTree actions menu: (1) an Obsidian-compatible Markdown vault -- one .md file per note with YAML frontmatter and wikilinks, zipped in memory; (2) an Anki deck -- a .apkg file built with `genanki` from flashcards whose deck name matches the collection. A new `ExportService` in the service layer handles all assembly logic; the existing `/collections` router gains a single `GET /{id}/export?format=markdown|anki` endpoint that streams the binary result as a file attachment.

3. Implement `backend/app/services/export_service.py`:
   a. `export_collection_markdown(collection_id, session) -> bytes`
      - Fetch `NoteCollectionModel` by id; raise `ValueError` if not found
      - Fetch all member `note_id`s via `NoteCollectionMemberModel`; return minimal zip on empty
      - Batch-fetch `NoteModel` rows for all member ids
      - Resolve parent collection name for folder path (single JOIN on `parent_collection_id`)
      - For each note: build YAML frontmatter (tags, collection path as list, created_at ISO string, source document title via LEFT JOIN `DocumentModel` on `note.document_id`)
      - Resolve `[[note_id|text]]` markers: batch-fetch `NoteModel.content` for all target ids found via regex; replace with `[[{target_first_line[:40]}]]`; unresolvable ids fall back to plain `{text}`
      - Slugify first non-empty line for filename: `{note_id[:8]}-{slug}.md`
      - Write all files into an in-memory `zipfile.ZipFile`; return `bytes`
   b. `export_collection_anki(collection_id, session) -> bytes`
      - Fetch collection name; raise `ValueError` if not found
      - Query `FlashcardModel` where `deck == collection.name`
      - Build `genanki.Deck(id=abs(hash(collection_name)) % (10**10), name=collection_name)`
      - For each flashcard: `genanki.Note(model=BASIC_MODEL, fields=[card.question, card.answer])`
      - Write to temp file via `genanki.Package`, read bytes, clean up; return bytes (empty deck if no cards)
   c. Singleton accessor `get_export_service() -> ExportService`
4. Add `GET /{collection_id}/export` to `backend/app/routers/collections.py`:

## Test Strategy

- **Unit tests (in-memory SQLite, no I/O):**
  - `export_collection_markdown`: file count in zip, frontmatter field presence, wikilink resolution, unresolvable-id fallback, empty collection returns valid zip
  - `export_collection_anki`: return type is bytes, result is a valid zip archive (zipfile.is_zipfile), no-flashcard case returns bytes (not error)
  - `GET /collections/{id}/export?format=markdown`: HTTP 200, `application/zip` content type
  - `GET /collections/{id}/export?format=anki`: HTTP 200
  - `GET /collections/nonexistent/export`: HTTP 404
- **Integration tests:** not needed -- no ML models or LanceDB involved in export
- **Smoke:** `S174.sh` verifies the live endpoint returns a binary attachment with non-zero length for both formats, and returns 404 for unknown collection id

## Cross-Story Risk

- **S161 (NoteCollectionModel/NoteCollectionMemberModel):** export service reads these tables read-only; no write path that could affect collection membership
