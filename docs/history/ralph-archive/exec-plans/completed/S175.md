# Exec Plan: S175 -- Multi-document notes: cross-book annotations with dual entity edge support

**Phase:** V3 Phase 2 - Note Intelligence and Synthesis
**Priority:** 15
**Status:** in-progress

## Goal

NoteModel.document_id is a single nullable FK that prevents users from formally linking a note to multiple source documents. This story introduces a NoteSourceModel pivot table (composite PK: note_id, document_id) so notes can reference any number of source books. The Kuzu graph layer is extended to emit a DERIVED_FROM edge for every NoteSourceModel row. The REST API is updated for create, update, list, and get-by-id. The NoteEditorDialog source picker is upgraded from a single-select dropdown to a multi-select chip UI. NoteModel.document_id is kept nullable for backward compatibility (Reading Journal "Convert to Note" path continues to work via legacy field).
