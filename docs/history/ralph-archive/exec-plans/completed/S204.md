# S204 Execution Plan: Notes from highlighted text sync state across all surfaces

## Problem
Notes created from highlighted text in DocumentReader (via NoteCreationDialog) only invalidated `["notes-for-doc", documentId]` TanStack Query cache. NotesReaderPanel uses `["reader-notes", collectionId]` and Notes.tsx uses `["notes", ...]`, so these surfaces showed stale data.

## Solution
Add query invalidation for `["reader-notes"]`, `["notes"]`, `["notes-groups"]`, and `["collections"]` in all three onSaved callbacks in DocumentReader.tsx:
1. Inline NoteEditor (section-level note creation)
2. NoteCreationDialog onSaved (highlight-to-note flow)
3. NoteEditorDialog onSaved (post-creation full editing)

## Backend Verification
- POST /notes returns full NoteResponse with id, tags, collection_ids -- no change needed
- Async background tasks (embed, graph) fire via asyncio.create_task -- unaffected
