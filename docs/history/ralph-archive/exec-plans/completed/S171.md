# Exec Plan: S171 -- Note-to-note bidirectional links: explicit Zettelkasten-style connections

**Phase:** V3 Phase 2 - Note Intelligence and Synthesis
**Priority:** 11
**Status:** in-progress

## Goal

Add explicit note-to-note linking (Zettelkasten-style) so users can connect notes with typed
relationships (elaborates, contradicts, see-also, supports, questions). Each link is persisted
in SQLite as a NoteLinkModel row and mirrored asynchronously into Kuzu as a LINKS_TO edge. The
Notes tab gains inline authoring support: typing `[[` in the editor opens a debounced autocomplete
popover to search for and insert a link target, and the editor's Links section shows all outgoing
chips and incoming backlinks. Deleting a note cascades its link rows; the `[[id|text]]` syntax
renders as a styled chip in the preview pane.
