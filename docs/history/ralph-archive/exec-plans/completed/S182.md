# Exec Plan: S182 -- YouTube transcript viewer: full feature parity with text documents in the Read section

**Phase:** V3 Phase 3 - UX Polish and Learner Focus
**Priority:** 22
**Status:** in-progress

## Goal

YouTube documents are ingested and their transcripts land in the `chunks` table, but the Read tab currently renders `ReadView` (which fetches `/sections/{id}/content`) for all document types -- YouTube documents have no sections, so the reader shows nothing. This story wires up a dedicated `YouTubeTranscriptView` component that fetches `/documents/{id}/chunks`, renders transcript paragraphs with optional timestamps, and exposes the same "Take a Note" and "Ask about this" entry points as the text reader. On the backend it adds two nullable columns (`youtube_url`, `channel_name`) to `DocumentModel`, a new `GET /documents/{id}/chunks` endpoint, and stores `channel_name` from yt-dlp metadata at ingest time.
