# S183 Execution Plan: Learning tab slim stats bar + document list as primary

## Story
Learning tab: slim LibraryOverview into sidebar widget, document list as primary focus

## Acceptance Criteria Summary
1. LibraryOverview -> compact single-row stats bar (books, notes, avg mastery, cards due)
2. Stats bar is one line -- no large card/hero panels
3. Each pill clickable: cards due -> Study, notes -> Notes, avg mastery -> Progress
4. Document list immediately below stats bar + search/filter controls
5. "Continue reading" highlighted row at top of list (most recently viewed)
6. Empty state: "No books yet -- ingest a PDF or YouTube video to get started" + CTA
7. Unit tests: stats bar renders pills, pill click dispatches luminary:navigate
8. Unit test: most recently viewed document at top of list
9. All quality gates pass

## Implementation Plan

### Frontend changes only (no backend changes needed)

#### 1. `frontend/src/lib/learningUtils.ts` (new file)
- Pure utility functions for testability:
  - `buildStatPillNavigateDetail(pill: "study"|"notes"|"progress")` - returns CustomEvent detail
  - `getMostRecentDocument(items: DocumentListItem[])` - returns first item sorted by last accessed
  - Constants: `STAT_PILL_LABELS` = { books, notes, mastery, due }
- No DOM/React imports

#### 2. `frontend/src/lib/learningUtils.test.ts` (new file)
- Unit test: stats pill navigate detail has correct tab
- Unit test: getMostRecentDocument returns first item
- Unit test: pill labels are correct

#### 3. `frontend/src/pages/Learning.tsx` (modify)
- Remove `LibraryOverview` SSE component (the large collapsible card)
- Add `LibraryStatsBar` component:
  - Single flex row of 4 pill buttons
  - Fetches: GET /documents (total), GET /notes (length), GET /study/due-count, GET /study/sessions (for avg mastery)
  - Each pill dispatches luminary:navigate event on click
  - Loading: skeleton pills; Error: show dashes
- "Continue reading" row: use recentItems[0] as a highlighted DocumentRow at top of list (only when page===1, no filters, no select mode)
- Update EmptyState message to: "No books yet -- ingest a PDF or YouTube video to get started"
- Update App.tsx navigate handler to handle "study" and "progress" tabs

#### 4. `frontend/src/App.tsx` (modify)
- Extend luminary:navigate handler to handle "study" -> /study and "progress" -> /progress

- Notes count: GET /notes?page_size=1 but that returns plain list... use a simple fetch of /notes with limit and take length OR add ?page_size=1000 and count... Actually GET /notes returns `list[NoteResponse]` (no total). Better: fetch /notes with no filters and count. Or add a new endpoint. Given no backend changes, use existing documents response for total and fetch notes separately.
- Actually: /notes returns full list. For efficiency, we can just use notes query response .length. If there are many notes this is fine since it's a single GET.
- Cards due: GET /study/due-count -> { due_today: int }
