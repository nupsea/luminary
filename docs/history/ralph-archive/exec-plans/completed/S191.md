# S191 - Library document action menu: quick navigation to document-scoped tabs

## Summary
Add a three-dot context menu to DocumentCard that lets users navigate directly to Chat, Study, Notes, or Viz tabs with the document pre-scoped.

## Approach
Frontend-only story. No backend changes.

### Files to Modify
1. `frontend/src/components/library/DocumentCard.tsx` -- add MoreVertical icon + dropdown menu
2. `frontend/src/lib/docActionUtils.ts` (NEW) -- pure utility for building navigate event details
3. `frontend/src/lib/docActionUtils.test.ts` (NEW) -- Vitest tests for utility
4. `frontend/src/App.tsx` -- extend onLuminaryNavigate handler for "chat", "study", "viz" tabs
5. `frontend/src/store.ts` -- no changes needed (existing fields suffice)

### Navigation Strategy
- **Read**: existing `onClick(doc.id)` -- no change, just add to menu
- **Chat about this**: set `chatSelectedDocId` + `chatScope='single'` in store, dispatch `luminary:navigate` with `{tab:'chat'}`
- **Study flashcards**: set `activeDocumentId` in store, dispatch `{tab:'study'}`
- **View notes**: set `activeDocumentId` in store, dispatch `{tab:'notes'}`
- **View in graph**: set `activeDocumentId` in store, dispatch `{tab:'viz'}`

### UI Pattern
Use a simple div-based dropdown (per patterns.md) with useRef + outside-click-to-close. No shadcn DropdownMenu component exists.

### Testing
- Pure utility function `buildDocActionDetail(action, docId)` tested in Vitest node env
- Smoke script is frontend-only (no endpoints), will verify tsc passes

## Risks
- None significant. All navigation mechanisms already exist.
