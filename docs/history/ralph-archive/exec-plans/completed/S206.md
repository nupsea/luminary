# Exec Plan: S206 -- Fix flashcard search: FTS5 query returns no results for keyword searches

**Phase:** V3 Phase 6 - Polish and Bug Fixes
**Priority:** 46
**Status:** in-progress

## Goal

The flashcard search endpoint (`GET /flashcards/search?query=...`) uses an FTS5 virtual table
(`flashcards_fts`) to match keywords against flashcard question and answer text. Users report
zero results despite matching flashcards existing. This story fixes five root causes: the FTS
MATCH subquery binding pattern may not transfer params correctly, there is no sanitization of
FTS5 special-syntax characters (causing syntax errors on user input like "C++"), there is no
LIKE fallback when FTS returns 0 results, there is no startup backfill for flashcards that
existed before `_sync_flashcard_fts` was introduced in S184, and the frontend empty-state message
is already in place but may not render correctly when `query` is active.

2. **Fix FTS MATCH subquery param binding** in `FlashcardService.search()`:
   The current pattern builds `fts_sub` with `.params(q=query.strip())` chained on the outer
   `stmt`, not on `fts_sub`. SQLAlchemy may not propagate the named param into the subquery's
   compiled SQL. Rewrite using a `text()` correlated subquery or a direct `bindparam` on the
   inner select to guarantee the `:q` param is bound at the subquery level.

3. **Add `_sanitize_fts_query()`** in `flashcard.py` (near the other FTS helpers, ~line 901):
   Strip characters that FTS5 treats as syntax operators: `"`, `*`, `(`, `)`, `^`, `+`, `-`.
   Join remaining tokens with space (AND semantics). Return empty string for all-whitespace input.

4. **Add LIKE fallback** in `FlashcardService.search()`: after executing FTS subquery, if the
   resulting `FlashcardModel.id.in_()` set is empty, fall back to
   `LIKE '%' || :q || '%'` on both `question` and `answer` columns (OR semantics as fallback
   only). Wrap in `or_(...)`. Log a warning when the fallback is triggered.

5. **Add `backfill_flashcard_fts()`** standalone async function in `flashcard.py`:
   - Accept an `AsyncSession` parameter (or create its own via `get_session_factory()`).
   - Select all `FlashcardModel` rows whose `id` does NOT appear in
     `SELECT c2 FROM flashcards_fts_content` (shadow table).
   - For each unindexed card, call `_sync_flashcard_fts(card, session)`.
   - Commit in batches of 100 to avoid holding a long transaction.
   - Log count of backfilled cards at INFO level.

6. **Wire backfill into lifespan** in `main.py`: after `await create_all_tables(engine)`,
   create a session with `async with get_session_factory()()` and call
   `await backfill_flashcard_fts(session)`. Wrap in try/except so backfill failure does not
   block startup.
