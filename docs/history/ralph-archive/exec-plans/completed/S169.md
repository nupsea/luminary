# Exec Plan: S169 -- Collection-based flashcard generation: deck naming, incremental generation, Study tab integration

**Phase:** V3 Phase 1 - Note Organization and Knowledge Graph
**Priority:** 10
**Status:** in-progress

## Goal

This story extends note-sourced flashcard generation so that a collection is the natural unit for a study deck: cards generated from a collection are named after the collection, duplicate work is avoided via a content-hash deduplication column, and the Study tab surfaces collection decks alongside document decks with distinct iconography. The `source_content_hash` column enables incremental generation -- re-running for an unchanged note is a no-op without a force flag.

## Implementation Notes

### `generate_from_collection` algorithm

```
async def generate_from_collection(
    self, collection_id, count_per_note, difficulty, session, force_regenerate=False
) -> dict[str, int | str]:
    # 1. Fetch collection name
    # 2. Fetch NoteCollectionMemberModel WHERE collection_id=cid -> note_ids
    # 3. For each note_id:
    #    a. Fetch NoteModel
    #    b. content_hash = sha256(note.content[:500].encode()).hexdigest()[:16]
    #    c. If not force_regenerate:
    #       SELECT COUNT(*) FROM flashcards WHERE deck=collection.name
    #         AND source='note' AND source_content_hash=:hash
    #       If count > 0: skipped += 1; continue
    #    d. Call LLM (reuse FLASHCARD_USER_TMPL + FLASHCARD_SYSTEM)
    #    e. Persist FlashcardModel rows with deck=collection.name,
    #       source='note', source_content_hash=hash
    # 4. Return {created, skipped, deck}
```

Each note is processed independently (not batched) so the per-note hash check is accurate. Do NOT use `asyncio.gather` across notes -- share the same `AsyncSession` sequentially to respect I-1.

### `GET /flashcards/decks` query

```sql
SELECT deck,
       source,
       COUNT(*) AS card_count,
       MAX(document_id) AS document_id   -- null for note-sourced
FROM flashcards
GROUP BY deck
```

Join the result set against `NoteCollectionModel` by name to determine `source_type`:
- If `deck` matches a `NoteCollectionModel.name`: `source_type = "collection"`, populate `collection_id`
- Else if `source = "document"` (or `document_id` non-null): `source_type = "document"`
- Else: `source_type = "note"` (tag-scoped or note_ids-scoped, no collection match)
