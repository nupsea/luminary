# S211 -- Learning goals UI on Study tab and Attach-to-goal in timer popover

## Goal

Build the user-facing complement to S210's typed-goals backend:
- A "Goals" sub-section at the top of the Study tab (CRUD list, detail panel,
  type-aware progress).
- An "Attach to goal" select in the FocusTimerPill popover, persisted in the
  Zustand focus store and forwarded to `POST /pomodoro/start`.

Backend was rewritten in S210 (typed goals: read|recall|write|explore). The
existing `GoalsPanel` in `Study.tsx` still calls the old document-centric API
(`/goals` with `document_id` + `target_date` + `/readiness`) -- those endpoints
were removed. The frontend currently appears compiled but is dead at runtime.
This story replaces that surface.

## Backend touches
None expected. S210 backend already implements every endpoint we need:
- `GET /goals?status=active` -- list (server-side filter by status)
- `POST /goals` -- create
- `GET /goals/{id}` -- detail
- `PATCH /goals/{id}` -- edit
- `POST /goals/{id}/archive`
- `POST /goals/{id}/complete`
- `DELETE /goals/{id}` -- delete
- `GET /goals/{id}/progress` -- type-dispatched metrics
- `GET /goals/{id}/sessions?limit=20` -- linked session list

If smoke flushes anything missing, add it under a follow-up. Story scope is UI.

## Smoke outline (S211.sh)

```bash
set -euo pipefail
BASE="${API_BASE:-http://localhost:8000}"

# 1. Create a recall goal with a target value.
GOAL=$(curl -fsS -X POST "$BASE/goals" -H 'Content-Type: application/json' \
  -d '{"title":"S211 smoke","goal_type":"recall","target_value":3,"target_unit":"cards"}')
GID=$(echo "$GOAL" | python -c 'import json,sys;print(json.load(sys.stdin)["id"])')

# 2. Confirm it appears in /goals?status=active.
curl -fsS "$BASE/goals?status=active" | python -c \
  'import json,sys;ids=[g["id"] for g in json.load(sys.stdin)];sys.exit(0 if "'"$GID"'" in ids else 1)'

# 3. Start a pomodoro with this goal.
SESSION=$(curl -fsS -X POST "$BASE/pomodoro/start" -H 'Content-Type: application/json' \
  -d "{\"focus_minutes\":1,\"break_minutes\":1,\"surface\":\"recall\",\"goal_id\":\"$GID\"}")
SID=$(echo "$SESSION" | python -c 'import json,sys;print(json.load(sys.stdin)["id"])')

# 4. Complete it.
curl -fsS -X POST "$BASE/pomodoro/$SID/complete" >/dev/null

# 5. Progress endpoint returns JSON with metrics dict.
PROG=$(curl -fsS "$BASE/goals/$GID/progress")
echo "$PROG" | python -c 'import json,sys;p=json.load(sys.stdin);assert "metrics" in p;assert p["goal_type"]=="recall"'

# 6. Linked sessions list shows the session.
curl -fsS "$BASE/goals/$GID/sessions?limit=20" | python -c \
  'import json,sys;ids=[s["id"] for s in json.load(sys.stdin)];sys.exit(0 if "'"$SID"'" in ids else 1)'

# 7. Stats still tally a goalless completion (run 1-min goalless session, complete, count++).
BEFORE=$(curl -fsS "$BASE/pomodoro/stats" | python -c 'import json,sys;print(json.load(sys.stdin)["total_completed"])')
GS=$(curl -fsS -X POST "$BASE/pomodoro/start" -H 'Content-Type: application/json' \
  -d '{"focus_minutes":1,"break_minutes":1,"surface":"none"}')
GSID=$(echo "$GS" | python -c 'import json,sys;print(json.load(sys.stdin)["id"])')
curl -fsS -X POST "$BASE/pomodoro/$GSID/complete" >/dev/null
AFTER=$(curl -fsS "$BASE/pomodoro/stats" | python -c 'import json,sys;print(json.load(sys.stdin)["total_completed"])')
[ "$AFTER" -gt "$BEFORE" ]

# 8. Archive then delete.
curl -fsS -X POST "$BASE/goals/$GID/archive" >/dev/null
curl -fsS -X DELETE "$BASE/goals/$GID" -o /dev/null -w '%{http_code}\n' | grep -q 204
echo "S211 smoke OK"
```
