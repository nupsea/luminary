# Exec Plan: S186 -- Chat: inline document scope selector replaces settings drawer scope

**Phase:** V3 Phase 4 - Learner Experience Refinement
**Priority:** 26
**Status:** in-progress

## Goal

Move the document scope selector from the two-click ChatSettingsDrawer back into the Chat
header as a compact searchable combobox. The combobox defaults to "All documents" (globe
icon), lets users pick a document (scope=single) in one click, and offers an 'x' clear
button. Changing document within single scope inserts a subtle divider rather than clearing
the conversation; switching between single and all scopes clears it. The ChatSettingsDrawer
retains only model selector and web search toggle. The S147 chatPreload pre-selection
continues to work via the new combobox.

## Files to Touch

### Backend
None -- GET /documents?sort=newest already exists; no new endpoints required.

### Frontend
- `frontend/src/pages/Chat.tsx` -- replace the `scope === "single"` guard + native `<select>` header block with a new `DocumentScopeCombobox` component; wire scope-change logic for divider-vs-clear behavior; remove scope-related props from `<ChatSettingsDrawer>` call site
- `frontend/src/components/ChatSettingsDrawer.tsx` -- remove `scope` and `onScopeChange` props; remove the Scope section from the drawer body; update `ChatSettingsDrawerProps` interface
- `frontend/src/lib/chatSettingsUtils.ts` -- update `DRAWER_SECTIONS` constant to remove `"scope"` (now `["model", "web_search"]`); add `buildScopeComboboxLabel` pure helper for unit testing
- `frontend/src/lib/chatSettingsUtils.test.ts` -- update `DRAWER_SECTIONS` test (scope no longer in drawer); add tests for `buildScopeComboboxLabel`; add tests for chatPreload combobox pre-selection logic

### Other
- `scripts/smoke/S186.sh` -- smoke test verifying GET /documents?sort=newest returns document list and Chat tab backend contract is intact
