# Exec Plan: S162 -- Hierarchical tags: shadow index table, canonical tag registry, prefix-search API

**Phase:** V3 Phase 1 - Note Organization and Knowledge Graph
**Priority:** 2
**Status:** in-progress

## Goal

Tags are currently stored as flat JSON arrays in `notes.tags` with no SQL index, meaning
`GET /notes?tag=` uses a full-table `json_each()` scan that degrades at scale. This story
adds three backend primitives: (1) `NoteTagIndexModel` -- a shadow/denorm table that
explodes each note's tags into indexed rows for O(1) prefix lookup; (2) `CanonicalTagModel`
-- a slug-keyed registry that supports slash-convention parent-child hierarchy
(e.g. `science/biology/genetics`) along with a `TagAliasModel` for synonyms; (3) a new
`/tags` router with flat list, tree, autocomplete, and per-tag note listing endpoints.
The existing `GET /notes?tag=` filter is rewritten to use a prefix subquery against the
shadow index so that querying `science` automatically returns notes tagged
`science/biology` and `science/physics/quantum`.

## Step Sequence

1. Read all files listed above before writing any code (done during planning)
2. Add `NoteTagIndexModel`, `CanonicalTagModel`, `TagAliasModel` to `backend/app/models.py`
3. Add DDL and backfill SQL to `backend/app/db_init.py`:
   - `CREATE TABLE note_tag_index` / `canonical_tags` / `tag_aliases` (via `Base.metadata.create_all` -- no raw DDL needed since models are ORM-mapped)
   - `CREATE INDEX IF NOT EXISTS idx_note_tag_index_tag_full ON note_tag_index(tag_full)`
   - `CREATE INDEX IF NOT EXISTS idx_note_tag_index_note_id ON note_tag_index(note_id)`
   - Backfill: SELECT id, tags FROM notes; for each row and each tag insert into note_tag_index and upsert CanonicalTagModel (use INSERT OR IGNORE for idempotency)
   - Import new models in the import block (the `noqa: F401` import list at top of db_init.py)
4. Implement `_sync_tag_index(note_id, tags, db)` in `backend/app/routers/notes.py`:
   - On create/update: DELETE existing rows for note_id, INSERT one row per tag, UPDATE `CanonicalTagModel.note_count` (+1 for added tags, -1 for removed)
   - On delete: DELETE all rows for note_id, decrement counts
   - Called synchronously (same DB write transaction) -- no `asyncio.create_task`
   - Upsert `CanonicalTagModel` row for any new tag encountered (INSERT OR IGNORE, then increment count)
5. Rewrite the `tag` filter branch in `list_notes` in `backend/app/routers/notes.py`:
   - Replace `json_each` EXISTS clause with:
     `NoteModel.id.in_(select(NoteTagIndexModel.note_id).where(or_(NoteTagIndexModel.tag_full == tag, NoteTagIndexModel.tag_full.like(tag + "/%"))))`
6. Create `backend/app/routers/tags.py`:
   - `GET /tags` -- flat list ordered by `note_count DESC`; fields: id, display_name, note_count, parent_tag
   - `GET /tags/tree` -- nested children list; node note_count includes descendant counts (recursive query or Python aggregation)
   - `GET /tags/{tag_id}/notes` -- notes whose tag_full equals tag_id or starts with `tag_id/`; returns list of NoteResponse
   - `GET /tags/autocomplete?q=` -- `LIKE :q || '%'` on `NoteTagIndexModel.tag_full`, DISTINCT, up to 10 results; returns list of `{tag_full, display_name, parent_tag}`
   - `POST /tags` -- create canonical tag; slug = lowercased, spaces-to-hyphens display_name; 409 if slug already exists
   - `PUT /tags/{tag_id}` -- update display_name or parent_tag; 404 if not found
   - `DELETE /tags/{tag_id}` -- 409 if `note_count > 0`; otherwise delete and cascade alias rows

## Test Strategy

- **Unit tests** (in `test_tags.py`, using in-memory SQLite via `make_engine + create_all_tables + monkeypatch`):
  - Note with tags `['programming/python', 'programming/go']` produces exactly 2 `NoteTagIndexModel` rows both with `tag_root='programming'`
  - `GET /notes?tag=programming` returns notes tagged `programming`, `programming/python`, and `programming/go` (prefix match)
  - Deleting a note removes its `NoteTagIndexModel` rows and decrements `CanonicalTagModel.note_count` to 0
  - `GET /tags/autocomplete?q=prog` returns `programming` and child tags sorted by note_count
  - `DELETE /tags/{id}` returns 409 when `note_count > 0`
  - Concurrent `POST /notes` (5 simultaneous via `asyncio.gather`) does not corrupt `CanonicalTagModel.note_count` -- use `Semaphore(1)` inside `_sync_tag_index` if concurrent access is possible, or verify the serialised session approach prevents corruption
- **Integration tests**: none required beyond the unit tests above (no graph or LanceDB involvement)
- **Smoke** (`scripts/smoke/S162.sh`):
  - POST a note with tags `['science/biology', 'science']`
  - `GET /notes?tag=science` returns the note (prefix match hits `science/biology`)
  - `GET /tags/autocomplete?q=sci` returns at least `science` and `science/biology`
  - `GET /tags/tree` returns HTTP 200 with `children` key present
  - `DELETE /tags/{id}` on a tag with `note_count > 0` returns HTTP 409
  - `DELETE /tags/{id}` on a zero-count tag returns HTTP 200 or 204
