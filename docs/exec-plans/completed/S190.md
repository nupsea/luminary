# S190 - Tag search: type-ahead search in Notes sidebar

## Summary
Add a search input at the top of TagTree that filters the tag tree by substring match with 150ms debounce, highlights matching substrings, shows parent chain for context, and supports Enter/click to apply tag filter.

## Scope
Frontend only. No backend changes needed -- filtering is client-side on the already-fetched tag tree data.

## Implementation Plan

### 1. Add pure utility functions to `tagUtils.ts`
- `filterTagTree(tree, query)`: recursively filter tree, keeping tags where display_name or id matches query (case-insensitive substring). Include parent nodes that have matching descendants (dimmed). Return filtered tree with a `matched` flag per node.
- `highlightMatch(text, query)`: return segments array for rendering highlighted text.

### 2. Update `TagTree.tsx`
- Add search input with placeholder "Search tags..." at the top, above the tag rows.
- Add state: `searchQuery` (raw input), `debouncedQuery` (after 150ms debounce).
- When `debouncedQuery` is non-empty, filter the tree using `filterTagTree`.
- Pass match info to `TagTreeItemRow` for highlighting and dimming parent-only nodes.
- Auto-expand matching branches when search is active.
- Show "No tags matching [query]" when filter returns empty.
- Clear button (x) on input to reset search.
- Enter key applies first matching tag as active filter.

### 3. Update `TagTree.test.ts`
- Test `filterTagTree` with substring match cases.
- Test parent chain shown for deeply nested matching tags.
- Test `highlightMatch` segments.

### 4. Smoke test
- `scripts/smoke/S190.sh`: curl GET /tags/tree, assert 200 + array response.
