# Exec Plan: S176 -- Notes reader-first layout: widen content area, strategic actions, clickable tag navigation

**Phase:** V3 Phase 3 - UX Polish and Learner Focus
**Priority:** 16
**Status:** in-progress

## Goal

Replace the current NoteEditorDialog (Dialog/modal) note-viewing experience with a full-width Sheet/Drawer
that opens from the right and devotes 70%+ of available width to the note content itself. Action buttons
move to a slim floating bar at bottom-right. Tag chips become clickable anchors that dispatch
`luminary:navigate` to the Notes tab with that tag pre-applied as a filter. Edit mode transforms the
reader in-place (textarea replaces content, Save/Cancel replaces floating bar). App.tsx already handles
`luminary:navigate` for Notes but currently reads `detail.tagFilter`; this story standardises the
payload shape to `{ tab: 'notes', filter: { tag: tagPath } }` and updates both the emitter and the
receiver. No backend changes are required -- this is purely a frontend refactor.

## Cross-Story Risk

- **S165 (Tag browser UI):** TagTree dispatches `luminary:navigate` with `{ tab: 'notes', tagFilter: tagPath }` (old shape). After this story, App.tsx reads both shapes so S165 behaviour is preserved. When S165 is re-implemented or refactored, it should switch to the new `filter.tag` shape.
- **S167 (Tag co-occurrence Viz tab):** Also dispatches `luminary:navigate` for notes. Same backward-compat treatment applies.
- **NoteEditorDialog.tsx:** The existing dialog component is no longer the primary note viewer. It remains in the codebase as-is; do not delete it (it may still be referenced by other entry points). The Sheet replaces it only at the Notes page level.
