# Exec Plan: S170 -- Notes performance refactor: fix vector dimension mismatch (384->1024), add tag indexes, batch reindex

**Phase:** V3 Phase 1 - Note Organization and Knowledge Graph
**Priority:** 4
**Status:** in-progress

## Goal

Three technical debts in the notes subsystem must be resolved before Phase 1 scales to large
corpora. First, `vector_store.py` declares the `note_vectors_v2` LanceDB table with a 384-dim
vector field but bge-m3 produces 1024-dim embeddings -- a silent mismatch causing truncated or
failed upserts. The fix adds a startup check that detects and drops+recreates the table when the
stored schema is wrong. Second, a new `ReindexService` identifies notes with missing embeddings
and backfills them, exposed via a guarded `POST /admin/notes/reindex` endpoint. Third, a
`@pytest.mark.slow` performance regression guard verifies that the `NoteTagIndexModel` indexes
introduced in S162 hold at 1,000-note scale (p95 tag query < 100 ms).

## Test Strategy

- Unit tests:
  - Dim mismatch detection: mock `_db.open_table` to return a table whose `schema.field("vector").type` has `list_size=384`; verify `drop_table` is called and warning is logged
  - Reindex count: mock LanceDB search to return empty DataFrame for a subset of notes; verify `ReindexReport.reindexed` matches missing count
  - Admin key auth: verify 403 on wrong key when `ADMIN_KEY` non-empty; verify 200 on correct key; verify 200 when `ADMIN_KEY` is empty string regardless of header
- Performance test (`@pytest.mark.slow`, guarded by `LUMINARY_PERF_TESTS`):
  - Bulk-insert 1,000 `NoteModel` rows and corresponding `NoteTagIndexModel` rows (tag `science`) into an in-memory SQLite DB
  - Execute `GET /notes?tag=science` via `TestClient` 10 times; record wall time with `time.perf_counter`
  - Assert p95 latency < 100 ms; this verifies S162 indexes are effective under load
- Smoke:
  - `POST /admin/notes/reindex` without auth header returns 200 (empty `ADMIN_KEY` in local dev)
  - Response contains `queued=true` and `total_notes` is an integer >= 0

## Cross-Story Risk

- S162 (`NoteTagIndexModel`): this story's performance test directly exercises the S162 indexes. If S162 indexes are missing or wrong, the p95 assertion will fail here.
- S163 (`vector_store.py` note table): this story renames `_get_note_table` and changes the schema. Any code in S163 that calls `_get_note_table` directly (e.g. note search in `notes.py`) must be updated to the new method name `_get_or_create_note_table`.
- S89 (original note vector upsert): `upsert_note_vector` and `delete_note_vector` both call `_get_note_table`; both call sites must be updated to `_get_or_create_note_table` or the rename will cause `AttributeError` at runtime.
