# Exec Plan: S194 -- References: URL validation and pruning of dead links

**Phase:** V3 Phase 5 - Reading Experience and Chat Intelligence
**Priority:** 34
**Status:** in-progress

## Goal

Extend the web references feature (S138) so that LLM-generated URLs are validated
via async HEAD requests, dead links are hidden by default from the frontend, and a
three-tier display (verified / unchecked / unavailable) gives users clear signal
about reference quality. The backend model, migration, service, router, and frontend
ReferencesPanel all need targeted additions; the existing extraction pipeline is
enhanced to set `is_valid` on newly created rows.

## Test Strategy

- **Unit tests (AC9, AC10):** Mock `httpx.AsyncClient` using `unittest.mock.patch` or
  `respx`. Create an in-memory SQLite DB with a `WebReferenceModel` row. Call
  `ReferenceValidatorService().validate_references(doc_id)`. Assert `is_valid` and
  `last_checked_at` are set correctly on the row.
- **Unit tests (AC11):** Use `AsyncClient` + `ASGITransport`. Insert two rows --
  one `is_valid=True`, one `is_valid=False`. Assert default GET excludes the False row
  and `include_invalid=true` returns both.
- **Unit test (AC12):** Mock `_extract_references` to return a list of dicts and mock
  `httpx.AsyncClient` to return status 200 for one URL and status 404 for another.
  Call `ReferenceEnricherService().enrich(doc_id)`. Assert both rows exist in DB with
  correct `is_valid` values.
- **No real HTTP** during any test (AC8): all httpx calls mocked.
- **Smoke:** see below.

## Smoke Test Design

`scripts/smoke/S194.sh`:
1. Ingest a test document via POST /documents/url or reuse an existing one if the
   smoke environment has a seeded DB.
2. POST `/references/documents/{doc_id}/validate` -- assert HTTP 200, `valid + invalid >= 0`.
3. GET `/references/documents/{doc_id}` (default) -- assert HTTP 200, no `is_valid=false`
   items in response body.
4. GET `/references/documents/{doc_id}?include_invalid=true` -- assert HTTP 200.
5. POST `/references/documents/{doc_id}/refresh` -- assert HTTP 200 or 202.

Use `curl -o $TMPFILE -w "%{http_code}"` pattern (macOS BSD-safe, per patterns.md).
Use `mktemp /tmp/S194_XXXXXX.json` for temp files.
