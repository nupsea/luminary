# Exec Plan: S181 -- Viz tab overhaul: collapsible View Options panel, select-all entity types, hide Call Graph

**Phase:** V3 Phase 3 - UX Polish and Learner Focus
**Priority:** 21
**Status:** in-progress

## Goal

Redesign the Viz tab sidebar controls to reduce clutter and improve learner focus. The four main changes are: (1) remove Call Graph from the top-level mode selector and show it only when the selected document is a code document; (2) move all layer toggles (Diagrams, Prerequisites, Cross-book, Notes) out of the main sidebar into a collapsible "View Options" panel; (3) add Select All / Deselect All buttons to the Entity Types section and persist active types in a new `vizStore` (Zustand); (4) add a Cluster view toggle in View Options that groups nodes by type/document when count > 200.

## Files to Touch

### Backend
None -- this story is purely frontend.

### Frontend
- `frontend/src/store.ts` -- add `vizStore` (separate Zustand store) with `activeEntityTypes`, `setActiveEntityTypes`, `selectAllEntityTypes`, `deselectAllEntityTypes`; OR add a `vizStore.ts` file at `frontend/src/` alongside `store.ts`
- `frontend/src/pages/Viz.tsx` -- main changes: (a) remove `call_graph` from the always-shown mode list, replace with conditional display; (b) replace inline layer toggle section + entity type section with a collapsible `ViewOptionsPanel` component or inline collapsible; (c) wire `activeTypes` to vizStore; (d) add cluster view toggle and cluster overlay logic
- `frontend/src/lib/vizUtils.ts` -- new file: pure functions for `isCodeDocument(format, contentType)`, `buildClusterGraph(nodes)`, `getSelectAllState(activeTypes, allTypes)`. Extracted to make them testable in Vitest node env.
- `frontend/src/lib/vizUtils.test.ts` -- unit tests: `isCodeDocument` returns true for code format/content_type; `selectAll` sets all types; `deselectAll` clears all types; cluster trigger when node count > 200

### Other
- `scripts/smoke/S181.sh` -- smoke test verifying the Viz tab backend APIs still respond (graph endpoint, documents list)

## Test Strategy

- Unit tests (`vizUtils.test.ts`, Vitest node env):
  - `isCodeDocument("code", "code")` returns true
  - `isCodeDocument("pdf", "book")` returns false
  - `isCodeDocument("py", "code")` returns true (extension-based)
  - `selectAll(ALL_ENTITY_TYPES)` returns a set of all types
  - `deselectAll()` returns empty set
  - `shouldShowClusterView(nodes, true)` returns true when non-note count > 200, false when <= 200
  - `shouldShowClusterView(nodes, false)` always returns false (toggle off)
- Integration / UI tests: not required for this story; the pure utility functions cover the ACs that need unit testing (AC9)
- Smoke: curl `GET /documents?sort=newest&page=1&page_size=1` and `GET /graph?doc_ids=` -- both must return 200
