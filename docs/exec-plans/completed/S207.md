# Exec Plan: S207 -- Auto-organize: retroactive naming normalization and actionable suggestions

**Phase:** V3 Phase 6 - Polish and Bug Fixes
**Priority:** 47
**Status:** in-progress

## Goal

Existing tags and collections created before S199 were never migrated to the adopted naming
conventions (tags: lower-case-hyphenated, collections: UPPER-CASE-HYPHENATED). This story adds
(A) a startup migration that normalizes all existing names idempotently, (B) a service method
and two new endpoints to detect naming violations and apply renames transactionally, and (C) a
"Naming Fixes" section in OrganizationPlanDialog so users can review and accept/reject individual
renames even when no HDBSCAN cluster suggestions exist.

2. **Service: add `detect_naming_violations()`** to `ClusteringService` in clustering_service.py.
   - Query all `CanonicalTagModel` rows; for each, compute `normalize_tag_slug(row.id)`. If
     different, emit a `{"type": "tag", "current": row.id, "normalized": slug}` entry.
   - Detect duplicate-after-normalize: group by normalized slug; if two tags map to the same
     normalized slug, emit a `{"type": "tag_merge", "current": [slug_a, slug_b], "normalized": slug}`.
   - Query all `NoteCollectionModel` rows where `auto_document_id IS NULL` (skip auto-collections
     since they mirror document titles); for each, compute `normalize_collection_name(row.name)`.
     If different, emit a `{"type": "collection", "current": row.name, "normalized": name, "id": row.id}`.
   - Return list of dicts. Method signature:
     `async def detect_naming_violations(self, db: AsyncSession) -> list[dict]`

3. **Service: add `apply_naming_fixes()`** to `ClusteringService`.
   - Accept a list of fix dicts (same shape as violations but filtered to accepted ones).
   - For tag renames: `UPDATE canonical_tags SET id = :new, display_name = :new WHERE id = :old`;
     `UPDATE note_tag_index SET tag_full = :new, tag_root = split(:new, '/')[0] WHERE tag_full = :old`;
     handle `tag_parent` update for child tags whose parent path changes.
   - For tag merges: reassign all NoteTagIndexModel rows from losing slug to winning slug; delete
     losing CanonicalTagModel row; insert TagAliasModel row (alias -> canonical).
   - For collection renames: `UPDATE note_collections SET name = :new WHERE id = :id`.
   - Wrap entire operation in a single transaction (single `await db.commit()` at the end).
   - Return `{"tags_renamed": int, "tags_merged": int, "collections_renamed": int}`.
   - Method signature:
     `async def apply_naming_fixes(self, fixes: list[dict], db: AsyncSession) -> dict`

## Cross-Story Risk

**I-1 (AsyncSession not shared across gather):** `apply_naming_fixes` must not use
`asyncio.gather` with a shared session. All DB updates should run sequentially within a single
session passed in.

**I-3 (Kuzu get_next guard):** Not relevant -- this story does not touch the Kuzu graph. If tag
renames are later propagated to the Kuzu entity graph, I-3 would apply. For this story, only
SQLite tables are updated.

**I-10 (loading/error/empty states):** The "Naming Fixes" section in OrganizationPlanDialog must
show a loading state while `normalize-check` is in flight, an inline error message on failure,
and an "No naming issues found" message when the list is empty.

**I-13 (quality gate order):** ruff must pass before pytest is run.

  to S201 since we are adding new endpoints and a migration, not modifying S201's on-create path.
