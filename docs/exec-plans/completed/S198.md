# Exec Plan: S198 -- Highlight reliability: text viewer, PDF edge cases, and large selection menu

**Phase:** V3 Phase 5 - Reading Experience and Chat Intelligence
**Priority:** 38
**Status:** in-progress

## Goal

This story fixes three concrete bugs in the S147 highlight feature that block daily reading workflow: (1) ReadView's Universal Text Viewer never shows highlight swatches because `resolveSourceRef` DOM-walks up through MarkdownRenderer output and never finds a `data-section-id` attribute (MarkdownRenderer renders plain DOM without that attribute -- the attribute lives on the parent wrapper div, but text nodes inside rendered markdown are not direct children of that wrapper); (2) PDF Viewer highlighting is intermittent because the page-to-section fallback in `resolveSourceRef` checks `start > 0` before matching, silently discarding all sections when the PDF parser produced `page_start=0`; (3) selecting 3+ paragraphs positions `SelectionActionBar` above the viewport because the bar is placed at `rect.top - 8` which can go negative for tall multi-paragraph selections. The story also adds a 10000-char selection guard (highlight swatches disabled but other actions remain), and cross-section selection anchoring to the start container's section.
