# Exec Plan: S225 -- Graph-augmented deterministic query expansion -- replace HyDE default

**Phase:** V3 Phase 8 - Evaluation and Correctness
**Priority:** 53
**Status:** in-progress

## Goal

Replace HyDE (which introduced ~7pp run-to-run variance due to LLM hallucination) as the
default query expansion strategy with a fully deterministic, local-first alternative.
`_graph_expand()` detects entities in the user query via GLiNER, resolves them to canonical
labels via `EntityDisambiguator`, then fetches up to 5 alias surface forms per entity from
the Kuzu `aliases` column (pipe-delimited). The expanded query feeds both vector and BM25
retrieval so chunks that mention "Holmes" are surfaced when the query says "Mr. Holmes".
HyDE is retained as an opt-in flag. No LLM is called in the default path; no external API
is used (I-16 clean).

## Cross-Story Risk

- **I-2 (asyncio.to_thread):** GLiNER model inference and Kuzu connection calls are both
  synchronous. Both must be wrapped in `asyncio.to_thread` inside the async `_graph_expand`.
  Failure to do so blocks the event loop on every search request.
- **I-3 (has_next guard):** Every Kuzu `get_next()` call inside `_graph_expand` must be
  preceded by `has_next()`. Graph service pattern (e.g. `upsert_entity` line 159) is the
  authoritative model.
- **I-5 (lazy imports):** `get_entity_extractor` and `get_graph_service` must be imported
  inside the `_graph_expand` function body to avoid a circular import chain
  `retriever -> ner -> (indirectly) retriever`. Match `_hyde_expand` and `vector_search`
  patterns in the same file.
- **S212 regression risk:** The existing `retrieve()` HyDE path must remain entirely
  unchanged when `hyde=True, graph_expand=False`. The flag ordering (graph_expand applied
  first, HyDE second) means both can stack -- but the test `test_retrieve_graph_expand_false`
  must confirm no cross-contamination.
- **S226 dependency:** S226 (planned) will iterate on eval thresholds and may depend on the
  baseline produced by S225. The eval results table above must be filled before S225 is
  closed.
- **Kuzu connection thread safety:** Kuzu `_conn` is not thread-safe. Wrapping Kuzu calls
  in `asyncio.to_thread` serialises them via the thread pool; do NOT share `_conn` across
  concurrent `asyncio.gather` tasks (existing invariant, not introduced here).
