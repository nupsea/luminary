# S137 — Technical Flashcard Types with Bloom's Taxonomy Levels

## Goal

Add `flashcard_type` and `bloom_level` columns to the flashcards table, define a `TECH_FLASHCARD_SYSTEM` prompt and a `generate_technical()` method in `FlashcardService`, then extend the Study tab so users can filter by type/level and code answers render with syntax highlighting.

---

- **Types** — no new type file needed; enum values are expressed as `Literal` strings inline (consistent with existing patterns in the file). `FlashcardResponse` Pydantic schema gains two new optional fields.
- **Repo** — `FlashcardModel` in `app/models.py` gains two nullable columns; `app/db_init.py` gains two `ALTER TABLE` migration statements.
- **Service** — `app/services/flashcard.py` gains `TECH_FLASHCARD_SYSTEM`, `TECH_FLASHCARD_USER_TMPL`, and `FlashcardService.generate_technical()`.
- **API** — `app/routers/flashcards.py` gains a new `POST /flashcards/generate-technical` endpoint and `FlashcardResponse` is extended.
- **Frontend** — `frontend/src/pages/Study.tsx` gains filter controls (flashcard_type, bloom_level range) and `frontend/src/components/StudySession.tsx` gains code-block rendering using the existing `MarkdownRenderer` (which already has `rehype-highlight`).

### 4. [Service] Add generate_technical() method to FlashcardService

In `backend/app/services/flashcard.py`, add a new method to `FlashcardService`:

```python
async def generate_technical(
    self,
    document_id: str,
    scope: Literal["full", "section"],
    section_heading: str | None,
    count: int,
    session: AsyncSession,
) -> list[FlashcardModel]:
    """Generate Bloom's-taxonomy-typed flashcards for tech_book/tech_article documents.

    Uses TECH_FLASHCARD_SYSTEM exclusively. Stores flashcard_type and bloom_level
    on every generated card. Only call this for content_type in
    ('tech_book', 'tech_article', 'code').
    """
    llm = get_llm_service()

    doc_result = await session.execute(
        select(DocumentModel).where(DocumentModel.id == document_id)
    )
    doc = doc_result.scalar_one_or_none()
    content_type = doc.content_type if doc else "unknown"

    chunks = await _fetch_chunks(document_id, scope, section_heading, session, content_type)
    if not chunks:
        return []

    # Determine context signals for the prompt
    has_code = any(c.has_code for c in chunks)
    # admonition_type: look up the section's admonition field if scoped to a section
    admonition_type: str | None = None
    if scope == "section" and section_heading:
        sec_result = await session.execute(
            select(SectionModel)
            .where(SectionModel.document_id == document_id)
            .where(SectionModel.heading == section_heading)
            .limit(1)
        )
        sec = sec_result.scalar_one_or_none()
        if sec:
            admonition_type = sec.admonition_type

    combined_text, first_chunk_id = _build_text(chunks)
    if not combined_text:
        return []

    prompt = TECH_FLASHCARD_USER_TMPL.format(
        count=count,
        section_heading=section_heading or "(none)",
        has_code=str(has_code),
        admonition_type=admonition_type or "(none)",
        text=combined_text,
    )

    with trace_chain(
        "flashcard.generate_technical",
        input_value=f"doc={document_id} scope={scope} count={count}",
    ) as span:
        span.set_attribute("flashcard.document_id", document_id)
        span.set_attribute("flashcard.scope", scope)
        span.set_attribute("flashcard.requested_count", count)
        span.set_attribute("flashcard.mode", "technical")

        raw = await llm.generate(prompt, system=TECH_FLASHCARD_SYSTEM, stream=False)
        cards_data = _parse_llm_response(raw, document_id)
        span.set_attribute("flashcard.generated_count", len(cards_data))

    now = datetime.now(UTC)
    flashcards: list[FlashcardModel] = []
    for item in cards_data:
        if not isinstance(item, dict):
            continue
        question = str(item.get("question", "")).strip()
        answer = str(item.get("answer", "")).strip()
        source_excerpt = str(item.get("source_excerpt", "")).strip()
        flashcard_type = str(item.get("flashcard_type", "definition")).strip()
        raw_bloom = item.get("bloom_level")
        bloom_level: int | None = int(raw_bloom) if isinstance(raw_bloom, (int, float)) else None
        if not question or not answer:
            continue
        card = FlashcardModel(
            id=str(uuid.uuid4()),
            document_id=document_id,
            chunk_id=first_chunk_id,
            question=question,
            answer=answer,
            source_excerpt=source_excerpt,
            difficulty="medium",
            fsrs_state="new",
            fsrs_stability=0.0,
            fsrs_difficulty=0.0,
            due_date=now,
            reps=0,
            lapses=0,
            created_at=now,
            flashcard_type=flashcard_type,
            bloom_level=bloom_level,
        )
        session.add(card)
        flashcards.append(card)

    if flashcards:
        await session.commit()
        for card in flashcards:
            await session.refresh(card)

    return flashcards
```

```python
class GenerateTechnicalRequest(BaseModel):
    document_id: str
    scope: Literal["full", "section"] = "full"
    section_heading: str | None = None
    count: int = 10


@router.post("/generate-technical", response_model=list[FlashcardResponse], status_code=201)
async def generate_technical_flashcards(
    req: GenerateTechnicalRequest,
    session: AsyncSession = Depends(get_db),
    service: FlashcardService = Depends(get_flashcard_service),
) -> list[FlashcardResponse]:
    """Generate Bloom's-taxonomy-typed flashcards for technical documents."""
    try:
        cards = await service.generate_technical(
            document_id=req.document_id,
            scope=req.scope,
            section_heading=req.section_heading,
            count=req.count,
            session=session,
        )
    except (
        litellm.exceptions.ServiceUnavailableError,
        litellm.exceptions.APIConnectionError,
        ConnectionRefusedError,
    ) as exc:
        raise HTTPException(
            status_code=503,
            detail="Ollama is not running. Start it with: ollama serve",
        ) from exc
    logger.info(
        "Generated technical flashcards",
        extra={"document_id": req.document_id, "count": len(cards)},
    )
    return [_to_response(c) for c in cards]
```

The new route `/generate-technical` must be registered BEFORE the `/{card_id}` catch-all routes to avoid routing conflicts. In the existing router all named `/generate*` routes are already before `/{document_id}` routes, so follow that pattern.

## Risk Flags

- **Risk: AC 1 migration wording vs. SQL DEFAULT.** The story says "migration sets existing rows to `flashcard_type='definition'`" but a SQL `ALTER TABLE ADD COLUMN ... DEFAULT 'definition'` would incorrectly label all legacy book/note cards as technical definitions. The correct implementation is nullable columns with no SQL DEFAULT; `generate_technical()` always sets the field in Python. The prompt to the LLM enforces it. Document this in a comment in `db_init.py`. Mitigation: implement as nullable with no default; add a test that confirms a legacy card (created without `generate_technical()`) has NULL for both fields.

- **Risk: Route ordering conflict.** The new `POST /flashcards/generate-technical` must be registered before FastAPI's `/{card_id}` catch-all `POST /{card_id}/review`. In the current `flashcards.py`, all fixed-path routes (`/generate`, `/from-gaps`, `/entity-pairs`, `/generate-from-graph`) precede `/{document_id}` routes. Place the new route immediately after `/generate-from-graph`. Mitigation: verify ordering by running the existing test suite after adding the route.

- **Risk: FlashCard height constraint (h-64) clips MarkdownRenderer output.** The current `FlashCard` component uses a fixed `h-64`. Long code blocks in `trace` or `code_completion` cards may overflow. Mitigation: add `overflow-auto` to the card face containers; consider changing `h-64` to `min-h-64` (min-height: 16rem) so the card grows to content.

- **Risk: Tests depend on LLM returning correct flashcard_type.** ACs 3-5 say the unit tests must produce specific types. Since the LLM is mocked, the mock must return the expected JSON. Use deterministic stubs that return preset JSON strings matching the expected flashcard_type/bloom_level. The real LLM behavior is verified only in the integration test (AC 6) and smoke test. Mitigation: unit tests use fixed mock responses; integration test uses a live-ish LLM call (mocked at the boundary as per existing flashcard test patterns using `_MockLLMService`).

- **Risk: MarkdownRenderer inside fixed-height card breaks Framer Motion animation.** The flip animation uses `backfaceVisibility: "hidden"` and absolute positioning. Adding `MarkdownRenderer` (which wraps in a `div.prose`) inside the `motion.div` should be fine as long as layout constraints are maintained. Test visually after implementation.

- **Risk: `bloom_level` from LLM may be a string "4" rather than integer 4.** The `_parse_llm_response` function already returns raw dicts; `generate_technical()` must coerce `bloom_level` safely. The step 4 implementation uses `int(raw_bloom) if isinstance(raw_bloom, (int, float)) else None` — also handle the string case: `int(raw_bloom) if isinstance(raw_bloom, (int, float, str)) and str(raw_bloom).isdigit() else None`.

- **Risk: No stubs module reference for test import.** `test_flashcards.py` imports `from stubs import MockLLMService`. The new test file `test_flashcards_technical.py` must use the same import. Verify `stubs.py` exists in the test directory.
