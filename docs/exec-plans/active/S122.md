### Step 3 — [Service] Create YouTubeDownloader service

Create `backend/app/services/youtube_downloader.py`:

```python
"""YouTube audio download service using yt-dlp subprocess.

Uses subprocess (not the yt-dlp Python API) to match the project pattern
for system tools (see: ffmpeg use in ingestion.py).
"""
import asyncio
import json
import logging
import shutil
from pathlib import Path

logger = logging.getLogger(__name__)

_YOUTUBE_URL_PREFIXES = (
    "https://www.youtube.com/watch",
    "https://youtu.be/",
    "https://youtube.com/watch",
    "http://www.youtube.com/watch",
    "http://youtu.be/",
)


def is_youtube_url(url: str) -> bool:
    """Return True if url looks like a YouTube watch URL."""
    return any(url.startswith(p) for p in _YOUTUBE_URL_PREFIXES)


def check_ytdlp_available() -> bool:
    return shutil.which("yt-dlp") is not None


async def fetch_metadata(url: str) -> dict:
    """Run yt-dlp --dump-json --no-download and return parsed JSON.

    Raises RuntimeError on non-zero exit.
    """
    proc = await asyncio.create_subprocess_exec(
        "yt-dlp", "--dump-json", "--no-download", url,
        stdout=asyncio.subprocess.PIPE,
        stderr=asyncio.subprocess.DEVNULL,
    )
    stdout, _ = await proc.communicate()
    if proc.returncode != 0:
        raise RuntimeError(f"yt-dlp metadata fetch failed (exit {proc.returncode})")
    return json.loads(stdout.decode())


async def download_audio(url: str, dest_path: Path) -> None:
    """Download audio-only WAV to dest_path using yt-dlp.

    dest_path should NOT include an extension -- yt-dlp appends one.
    The actual file written will be dest_path.with_suffix('.wav').

    Raises RuntimeError on non-zero exit.
    """
    proc = await asyncio.create_subprocess_exec(
        "yt-dlp",
        "-x", "--audio-format", "wav", "--audio-quality", "0",
        "-o", str(dest_path) + ".%(ext)s",
        url,
        stdout=asyncio.subprocess.DEVNULL,
        stderr=asyncio.subprocess.DEVNULL,
    )
    await proc.wait()
    if proc.returncode != 0:
        raise RuntimeError(f"yt-dlp download failed (exit {proc.returncode})")
    logger.info("yt-dlp downloaded audio to %s.wav", dest_path)
```

Key design decisions:
- Pure subprocess calls, no yt-dlp Python API import (avoids coupling to internal API)
- `fetch_metadata` and `download_audio` are separate async functions; both awaitable
- `is_youtube_url` is a pure function for easy unit testing
- No `get_youtube_downloader()` singleton needed (stateless utility functions)

### Step 6 — [API] New POST /documents/ingest-url endpoint in documents.py

Place this endpoint after the `POST /documents/ingest` endpoint:

```python
@router.post("/ingest-url")
async def ingest_url(
    body: YouTubeIngestRequest,
    settings: Settings = Depends(get_settings),
):
    """Ingest a YouTube URL by downloading audio via yt-dlp and transcribing with Whisper."""
    from app.services.youtube_downloader import (  # noqa: PLC0415
        check_ytdlp_available,
        download_audio,
        fetch_metadata,
        is_youtube_url,
    )

    if not is_youtube_url(body.url):
        raise HTTPException(status_code=400, detail="URL must be a YouTube watch URL")

    if not check_ytdlp_available():
        raise HTTPException(
            status_code=503,
            detail=(
                "yt-dlp is not installed. Install it with: "
                "pip install yt-dlp  or  brew install yt-dlp"
            ),
        )

    # Fetch metadata first to get the video title (fast, no download)
    try:
        meta = await fetch_metadata(body.url)
    except RuntimeError as exc:
        logger.warning("yt-dlp metadata fetch failed: %s", exc)
        raise HTTPException(status_code=400, detail=str(exc)) from exc

    video_title = meta.get("title") or "YouTube Video"
    doc_id = str(uuid.uuid4())

    # Prepare destination path (yt-dlp appends .wav)
    data_dir = Path(settings.DATA_DIR).expanduser()
    raw_dir = data_dir / "raw"
    raw_dir.mkdir(parents=True, exist_ok=True)
    dest_stem = raw_dir / doc_id  # yt-dlp will write doc_id.wav

    try:
        await download_audio(body.url, dest_stem)
    except RuntimeError as exc:
        logger.error("yt-dlp download failed: %s", exc)
        raise HTTPException(
            status_code=500,
            detail=f"Audio download failed: {exc}",
        ) from exc

    dest = dest_stem.with_suffix(".wav")
    if not dest.exists():
        raise HTTPException(status_code=500, detail="Downloaded audio file not found")

    async with get_session_factory()() as session:
        doc = DocumentModel(
            id=doc_id,
            title=video_title,
            format="wav",
            content_type="audio",
            word_count=0,
            page_count=0,
            file_path=str(dest),
            file_hash=None,   # YouTube URLs are not hashed; no dedup by content
            stage="parsing",
            source_url=body.url,
            video_title=video_title,
        )
        session.add(doc)
        await session.commit()

    asyncio.create_task(run_ingestion(doc_id, str(dest), "wav", "audio"))
    logger.info("YouTube ingestion started", extra={"doc_id": doc_id, "url": body.url})
    return {"document_id": doc_id, "status": "processing"}
```

Important notes:
- `file_hash=None` is intentional: YouTube URLs are not deduplicated by content hash. Re-uploading the same URL creates a new document (acceptable for now; dedup by URL is TD scope).
- `content_type="audio"` keeps the ingestion pipeline unchanged; `source_url` is what distinguishes a YouTube document from a local audio file.
- `format="wav"` matches the downloaded file extension.
- The endpoint is synchronous up through yt-dlp download (which may take 30-60s for long videos). The download happens in the request-response cycle. This is intentional: the client must wait for the download before polling starts, which matches the "processing" contract. The download is fast for audio-only extraction (usually <10s for a 1h video). The endpoint does NOT use `asyncio.create_task` for the download because the file must exist before `run_ingestion` is called.
- If yt-dlp download is very slow for long videos, a future story could move the download into the ingestion graph as a `download_node`. For S122 keep it simple.

## Database Migration Approach

SQLite `create_all` adds new tables but does NOT add new columns to existing tables. The existing pattern in `db_init.py` is to add additive migrations to the `for ddl in [...]` list, each wrapped in a `try/except` that ignores "duplicate column" errors.

S122 adds two statements to that list:
```python
"ALTER TABLE documents ADD COLUMN source_url TEXT",
"ALTER TABLE documents ADD COLUMN video_title TEXT",
```

These run on every app startup. Existing rows will have `NULL` for both columns, which is correct (no backfill needed).

## yt-dlp Metadata Extraction Flow

```
POST /documents/ingest-url { url }
  -> is_youtube_url(url) check (400 if fails)
  -> check_ytdlp_available() check (503 if fails)
  -> fetch_metadata(url)
       yt-dlp --dump-json --no-download {url}
       returns JSON: { "title": "Video Title", "id": "...", ... }
  -> video_title = meta["title"]
  -> download_audio(url, /path/to/{doc_id})
       yt-dlp -x --audio-format wav --audio-quality 0 -o /path/to/{doc_id}.%(ext)s {url}
       produces: /path/to/{doc_id}.wav
  -> create DocumentModel(source_url=url, video_title=video_title, content_type="audio")
  -> asyncio.create_task(run_ingestion(doc_id, wav_path, "wav", "audio"))
  -> return { document_id, status: "processing" }
```

The ingestion graph then runs:
`parse -> classify (skipped, content_type="audio") -> transcribe (Whisper) -> chunk -> embed -> keyword_index -> entity_extract -> section_summarize -> summarize`

`source_url` is set at document creation time, not inside the ingestion graph. This is correct because the graph does not know about the URL; it only receives `file_path`. After ingestion completes, `source_url` and `video_title` are already in the DB row.

## How source_url Gets Stored

`source_url` is written once, at the moment `DocumentModel` is created in the `ingest-url` endpoint handler. It is never modified by the ingestion graph. This is the simplest approach and avoids any coupling between the graph and the URL concept.

## Risk Flags

- **Risk: yt-dlp download blocks the request thread** (mitigation: `asyncio.create_subprocess_exec` is fully async and non-blocking; the event loop stays free during the download. The HTTP client must wait for the response, but the FastAPI process is not blocked from serving other requests.)

- **Risk: yt-dlp output format varies** (mitigation: only read `meta.get("title") or "YouTube Video"` from the metadata JSON; no other fields are required. If the title is missing, a default is used.)

- **Risk: yt-dlp writes a file extension other than `.wav`** (mitigation: the `--audio-format wav` flag forces WAV output. However, if conversion fails, yt-dlp may fall back. The endpoint checks `dest_stem.with_suffix(".wav").exists()` and raises 500 if the file is absent. Future improvement: scan for any `{doc_id}.*` file in `raw_dir` as a fallback.)

- **Risk: Long YouTube videos exhaust request timeout** (mitigation: for audio-only extraction, yt-dlp typically downloads at 10-50x realtime. A 1-hour video takes ~2-6 minutes. If the frontend's default fetch timeout is shorter, the request will appear to hang. Document in the UI that large videos take several minutes. Future: move download into a background task and add a separate status poll.)

- **Risk: Frontend TypeScript strict mode fails if `source_url` is accessed without null checks** (mitigation: all frontend code accesses `doc.source_url` only after optional chaining or null checks; `isYouTubeDoc` handles null safely.)

- **Risk: `content_type = "audio"` filter in the library will include YouTube documents** (mitigation: this is correct behavior -- YouTube docs ARE audio content. If per-story filtering by "youtube" origin is needed, that is a separate feature.)

- **Risk: Existing audio documents lose the audio mini-player if `isYouTubeDoc` incorrectly returns true** (mitigation: `isYouTubeDoc` only returns true when `source_url` contains "youtube" or "youtu.be"; local audio files have `source_url = null` so the check is safe.)

- **Risk: DB migration adds columns but `TeachbackResultModel` import is missing from db_init.py** (mitigation: `TeachbackResultModel` is in models.py but NOT imported in db_init.py; this is pre-existing. Adding `source_url`/`video_title` columns does not require importing that model. The `ALTER TABLE` migration is independent of the ORM import list.)
