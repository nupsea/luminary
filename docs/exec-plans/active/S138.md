# S138 — Web Reference Grounding

## Goal

### 1. [Types] Add WebReferenceModel to backend/app/models.py

Add after `EnrichmentJobModel`:

```python
class WebReferenceModel(Base):
    """LLM-generated canonical web reference for a technical term in a section.

    source_quality values (ordered best to worst):
      official_docs | spec | wiki | tutorial | blog | unknown

    is_llm_suggested=True means URL was produced by the LLM from training knowledge
    and has not been verified via a live HEAD request.
    is_llm_suggested=False means a HEAD request confirmed the URL is reachable.

    Note: any new delete path in documents.py must also delete these rows
    (no FK CASCADE in SQLite without pragma enforcement).
    """

    __tablename__ = "web_references"

    id: Mapped[str] = mapped_column(String, primary_key=True)
    document_id: Mapped[str] = mapped_column(String, nullable=False, index=True)
    section_id: Mapped[str | None] = mapped_column(String, nullable=True, index=True)
    term: Mapped[str] = mapped_column(String(200), nullable=False)
    url: Mapped[str] = mapped_column(Text, nullable=False)
    title: Mapped[str] = mapped_column(String(300), nullable=False)
    excerpt: Mapped[str] = mapped_column(Text, nullable=False, default="")
    # official_docs | spec | wiki | tutorial | blog | unknown
    source_quality: Mapped[str] = mapped_column(String(30), nullable=False, default="unknown")
    is_llm_suggested: Mapped[bool] = mapped_column(Boolean, nullable=False, default=True)
    created_at: Mapped[datetime] = mapped_column(
        DateTime, nullable=False, default=lambda: datetime.now(UTC)
    )

    __table_args__ = (
        UniqueConstraint("document_id", "section_id", "term", "url",
                         name="uq_web_ref_doc_section_term_url"),
    )
```

### 4. [Service] Create backend/app/services/reference_enricher.py

**Module-level constants:**

```python
_SOURCE_QUALITY_RANK: dict[str, int] = {
    "official_docs": 0,
    "spec": 1,
    "wiki": 2,
    "tutorial": 3,
    "blog": 4,
    "unknown": 5,
}

_MAX_REFS_PER_SECTION = 5

_SYSTEM_PROMPT = (
    "You are a technical documentation expert. "
    "For each technical term provided, output a JSON array of up to 5 canonical reference URLs. "
    "Order them: official language/framework docs first, then specification/RFC, "
    "then recognized book publisher, then trusted community wiki, then tutorial, then blog. "
    "For each reference use this exact JSON shape: "
    '{"term": "...", "url": "...", "title": "...", "excerpt": "...", '
    '"source_quality": "official_docs|spec|wiki|tutorial|blog|unknown"}. '
    "Return only the JSON array. No prose outside the array."
)
```

**`extract_references_from_section(section_summary: str, model: str) -> list[dict]` (pure helper):**

- Called by the service with the section summary content as input.
- Builds user prompt: "Section summary:\n{content}\n\nExtract up to 5 key technical terms and for each provide a canonical reference."
- Calls `await litellm.acompletion(model=model, messages=[{"role": "system", ...}, {"role": "user", ...}], temperature=0.0)`.
- Parses JSON from the response (strip markdown fences if present).
- Returns list of dicts; returns `[]` on parse failure (logged as warning).
- Raises `litellm.ServiceUnavailableError` if Ollama is unreachable (propagates to worker).

**`sort_by_quality(refs: list[dict]) -> list[dict]` (pure function):**

- Sorts by `_SOURCE_QUALITY_RANK.get(ref.get("source_quality", "unknown"), 5)` ascending.
- Returns sorted list (deterministic, testable without I/O).

**`class ReferenceEnricherService`:**

```python
async def enrich(self, document_id: str) -> int:
    """Generate web references for all SectionSummaryModel rows without existing references.

    Returns count of new WebReferenceModel rows created.
    """
```

Implementation steps inside `enrich`:
1. Load all `SectionSummaryModel` rows for the document (via `get_session_factory()`).
2. For each section_summary row, check if any `WebReferenceModel` rows already exist for that `(document_id, section_id)` pair -- skip if yes (idempotent on retry).
3. Call `extract_references_from_section(section_summary.content, settings.LITELLM_DEFAULT_MODEL)`.
4. If `settings.WEB_SEARCH_PROVIDER != "none"`, issue `httpx.AsyncClient().head(url, timeout=5)` per URL; set `is_llm_suggested = False` if status < 400. Catch all exceptions silently (keep `is_llm_suggested=True` on error).
5. Call `sort_by_quality(refs)` and take `[:_MAX_REFS_PER_SECTION]`.
6. Write `WebReferenceModel` rows in a single session commit (use `INSERT OR IGNORE` via SQLAlchemy `on_conflict_do_nothing` or catch `IntegrityError`).
7. Return total rows inserted.

**`async def web_refs_handler(document_id: str, job_id: str) -> None`:**

- Instantiates `ReferenceEnricherService` and calls `enrich(document_id)`.
- Logs start and completion at INFO level.
- `litellm.ServiceUnavailableError` propagates (worker marks job `failed`).

def _is_outdated(created_at: datetime) -> bool:

## Risk Flags

**Risk 1: section_id absent from GET /study/due flashcard response.**
The `Flashcard` type in `StudySession.tsx` currently has no `section_id` field. If the backend's `/study/due` endpoint does not return `section_id`, the Source panel cannot fetch section references. Mitigation: read `backend/app/routers/study.py` before writing frontend code; add `section_id: str | null` to the FlashcardResponse Pydantic model and to the TypeScript interface. This is a cross-layer change but contained to one field.

**Risk 2: Phantom data problem (Core Belief #22).**
The enrichment job is only useful if `SectionSummaryModel` rows exist for the document. Fiction books or short uploads often have no section summaries. Mitigation: enqueue the `web_refs` job only from inside `SectionSummarizerService.generate()` after writing at least one row -- this guarantees the data exists before the job runs. The API endpoint must return `[]` gracefully for documents with no rows (AC #6).

**Risk 3: LLM URL hallucination.**
The LLM may generate plausible-looking but non-existent URLs. Mitigation: `is_llm_suggested=True` is the default badge shown to the user; the UI labels these "Not verified". When `WEB_SEARCH_PROVIDER != "none"`, the HEAD verification clears this flag. Users see the badge and can judge credibility. This is acceptable for V1.

**Risk 4: JSON parse failure from LLM response.**
The LLM may wrap the JSON array in markdown fences or prefix it with prose. Mitigation: strip markdown fences before `json.loads()`; return `[]` and log a warning on `json.JSONDecodeError`. The service remains non-fatal; the enrichment job does not fail.

**Risk 5: Outdated badge 18-month cutoff is hardcoded.**
The 548-day cutoff is a magic number. Mitigation: define `_OUTDATED_DAYS = 548` as a module constant in the service and reference it in the `_is_outdated` helper -- both backend and frontend compute this locally (no setting needed for V1).

**Risk 6: Frontend PanelTab type widening.**
Adding `"references"` to `PanelTab` in `DocumentReader.tsx` must be done carefully to avoid TypeScript exhaustiveness errors in the switch/conditional. Mitigation: use `activeTab === "references"` as an additional `else if` branch rather than restructuring the existing conditional chain. Check that `tsc --noEmit` passes after the change.

**Risk 7: delete cascade missing for WebReferenceModel.**
SQLite FK pragma is not reliably enforced (TD-010). Mitigation: manually add `delete(WebReferenceModel).where(...)` to the document delete endpoint in `documents.py`, following the exact pattern used for `ImageModel`, `LearningObjectiveModel`, and `CodeSnippetModel`.

**Risk 8: httpx not in backend dependencies.**
The backend uses `httpx` already (visible in main.py for Ollama health check). No new dependency needed. Confirm with `uv run python -c "import httpx"` before writing URL verification code.
