**3b. Pure helper: `_parse_cloze_text(cloze_text: str) -> list[str]`**

A pure function that returns the list of blank terms in order of appearance. Returns an empty list if no `{{...}}` markers are found.

```python
import re as _re
_CLOZE_BLANK_RE = re.compile(r"\{\{(.+?)\}\}")

def _parse_cloze_text(cloze_text: str) -> list[str]:
    """Return list of blank terms extracted from {{term}} markers in order."""
    return _CLOZE_BLANK_RE.findall(cloze_text)
```

**3c. Pure helper: `_build_cloze_question(cloze_text: str) -> str`**

Replaces `{{term}}` with `[____]` for use in the `question` field (display in list views).

```python
def _build_cloze_question(cloze_text: str) -> str:
    """Replace {{term}} markers with [____] for list-view display."""
    return _CLOZE_BLANK_RE.sub("[____]", cloze_text)
```

**3d. Pure helper: `_parse_cloze_llm_response(raw: str) -> list[dict]`**

Reuses the JSON array extraction logic from `_parse_llm_response`. Each dict must have `cloze_text` with at least one `{{...}}` marker. Malformed entries (no markers) are filtered out.

```python
def _parse_cloze_llm_response(raw: str) -> list[dict]:
    """Parse the LLM JSON array response for cloze cards.

    Filters out any element whose cloze_text has no {{}} markers (malformed).
    Returns only valid elements.
    """
    items = _parse_llm_response(raw, "cloze")
    valid = []
    for item in items:
        if not isinstance(item, dict):
            continue
        cloze_text = str(item.get("cloze_text", "")).strip()
        if not cloze_text:
            continue
        blanks = _parse_cloze_text(cloze_text)
        if not blanks:
            logger.warning("Cloze item has no {{}} markers, skipping: %r", cloze_text[:100])
            continue
        valid.append(item)
    return valid
```

**3e. Service method: `FlashcardService.generate_cloze`**

```python
async def generate_cloze(
    self,
    section_id: str,
    count: int,
    session: AsyncSession,
) -> list[FlashcardModel]:
    """Generate cloze deletion flashcards for a section.

    Prompts the LLM to produce {{term}} fill-in-the-blank sentences.
    Validates that each card has at least one blank. Retries once if the
    first response contains zero valid cards. Cards whose cloze_text has
    no {{}} markers are skipped (never inserted).

    question field = cloze_text with {{term}} replaced by [____]
    answer field = comma-separated terms from the blanks
    cloze_text = raw {{term}} text for frontend rendering
    """
    from app.services.llm import get_llm_service  # already at module top

    llm = get_llm_service()

    # Fetch chunks for the section
    chunk_result = await session.execute(
        select(ChunkModel)
        .where(ChunkModel.section_id == section_id)
        .order_by(ChunkModel.chunk_index)
    )
    chunks = list(chunk_result.scalars().all())
    if not chunks:
        return []

    # Also look up document_id from the first chunk
    document_id = chunks[0].document_id
    first_chunk_id = chunks[0].id

    combined_text, _ = _build_text(chunks)
    if not combined_text:
        return []

    prompt = CLOZE_USER_TMPL.format(count=count, text=combined_text)

    with trace_chain(
        "flashcard.generate_cloze",
        input_value=f"section={section_id} count={count}",
    ) as span:
        span.set_attribute("flashcard.section_id", section_id)
        span.set_attribute("flashcard.requested_count", count)
        span.set_attribute("flashcard.mode", "cloze")

        raw = await llm.generate(prompt, system=CLOZE_SYSTEM, stream=False)
        items = _parse_cloze_llm_response(raw)

        # Retry once if zero valid items
        if not items:
            logger.warning(
                "generate_cloze: first attempt produced no valid cloze cards for section=%s, retrying",
                section_id,
            )
            raw2 = await llm.generate(prompt, system=CLOZE_SYSTEM, stream=False)
            items = _parse_cloze_llm_response(raw2)

        span.set_attribute("flashcard.generated_count", len(items))

    now = datetime.now(UTC)
    flashcards: list[FlashcardModel] = []
    for item in items:
        cloze_text = str(item.get("cloze_text", "")).strip()
        source_excerpt = str(item.get("source_excerpt", "")).strip()
        blanks = _parse_cloze_text(cloze_text)
        if not blanks:
            continue  # defensive: filtered above but guard here too
        question = _build_cloze_question(cloze_text)
        answer = ", ".join(blanks)
        card = FlashcardModel(
            id=str(uuid.uuid4()),
            document_id=document_id,
            chunk_id=first_chunk_id,
            source="document",
            deck="default",
            question=question,
            answer=answer,
            source_excerpt=source_excerpt,
            difficulty="medium",
            is_user_edited=False,
            fsrs_state="new",
            fsrs_stability=0.0,
            fsrs_difficulty=0.0,
            due_date=now,
            reps=0,
            lapses=0,
            created_at=now,
            flashcard_type="cloze",
            bloom_level=None,
            cloze_text=cloze_text,
        )
        session.add(card)
        flashcards.append(card)

    if flashcards:
        await session.commit()
        for card in flashcards:
            await session.refresh(card)

    return flashcards
```

### Step 5 — [Frontend] Create ClozeCard.tsx

Create `/Users/sethurama/DEV/LM/learning-mate/frontend/src/components/ClozeCard.tsx`.

The component:
- Accepts `card: FlashcardCard` (with `cloze_text: string | null`)
- Parses `cloze_text` on render using a regex split: `cloze_text.split(/(\{\{.+?\}\})/g)` produces an alternating array of plain text segments and `{{term}}` tokens
- Renders each segment inline; each `{{...}}` token renders as an underline box with `?` placeholder initially
- `revealedCount` state starts at 0
- "Reveal next blank" button increments `revealedCount`; when `revealedCount >= totalBlanks`, show FSRS rating buttons
- Progressive reveal animation: use Tailwind `transition-all duration-300` and conditional opacity/opacity-0 on each blank span
- Fallback: if `cloze_text` is null or has no `{{...}}` patterns, delegate rendering to the standard `FlashCard` flip flow (show question, flip to answer)

Exact interface:

```tsx
interface ClozeCardProps {
  card: FlashcardCard  // FlashcardCard defined in StudySession.tsx
  onRate: (rating: Rating) => Promise<void>
  isRating: boolean
}
```

State:
```tsx
const [revealedCount, setRevealedCount] = useState(0)
```

Parsing logic (pure):
```tsx
// Parse cloze_text into alternating text/blank segments
function parseCloze(cloze_text: string): Array<{ type: "text"; content: string } | { type: "blank"; term: string }> {
  const parts = cloze_text.split(/(\{\{.+?\}\})/g)
  return parts.map((part) => {
    const match = /^\{\{(.+?)\}\}$/.exec(part)
    return match ? { type: "blank" as const, term: match[1] } : { type: "text" as const, content: part }
  })
}
```

Render logic:
- Segments of type `"text"` render as `<span>{content}</span>`
- Segments of type `"blank"` at index `blankIndex`:
  - If `blankIndex < revealedCount`: render as `<span className="font-bold text-primary underline">{term}</span>`
  - Else: render as `<span className="inline-block min-w-[4rem] rounded border-b-2 border-foreground text-center text-muted-foreground">?</span>`
- Use a running `blankIndex` counter when iterating segments (only increment for `"blank"` segments)
- Button: "Reveal next blank" — disabled when `revealedCount >= totalBlanks`
- When `revealedCount >= totalBlanks`: render FSRS rating buttons in the same layout as `StudySession.tsx` `RATINGS`

Animation approach:
```tsx
// Each blank uses transition-opacity: hidden before reveal, visible after
<span
  className={`inline-block min-w-[4rem] rounded border-b-2 text-center transition-all duration-300 ${
    isRevealed
      ? "border-primary font-bold text-primary"
      : "border-foreground text-muted-foreground"
  }`}
>
  {isRevealed ? term : "?"}
</span>
```

No Framer Motion flip animation (per story requirement).

**Exported parsing function in ClozeCard.tsx:**

```tsx
/** Parse a cloze_text string into segments for rendering. Exported for testing. */
export function parseClozeSegments(
  clozeText: string
): Array<{ type: "text"; content: string } | { type: "blank"; term: string }> {
  const parts = clozeText.split(/(\{\{.+?\}\})/g)
  return parts.map((part) => {
    const match = /^\{\{(.+?)\}\}$/.exec(part)
    return match
      ? { type: "blank" as const, term: match[1] }
      : { type: "text" as const, content: part }
  })
}
```

**Test file `frontend/src/components/ClozeCard.test.ts`:**

```ts
import { describe, expect, it } from "vitest"
import { parseClozeSegments } from "./ClozeCard"

describe("parseClozeSegments", () => {
  it("parses two blanks correctly", () => {
    const segments = parseClozeSegments("{{X}} and {{Y}} are different")
    expect(segments).toHaveLength(4) // blank, " and ", blank, " are different" -- actually 5
    // More precisely: ["", "{{X}}", " and ", "{{Y}}", " are different"]
    // After split: 5 items. After map: [text(""), blank(X), text(" and "), blank(Y), text(" are different")]
    const blanks = segments.filter((s) => s.type === "blank")
    expect(blanks).toHaveLength(2)
    expect((blanks[0] as { type: "blank"; term: string }).term).toBe("X")
    expect((blanks[1] as { type: "blank"; term: string }).term).toBe("Y")
  })

  it("returns no blanks for plain text", () => {
    const segments = parseClozeSegments("No blanks here")
    expect(segments.every((s) => s.type === "text")).toBe(true)
  })

  it("handles single blank", () => {
    const segments = parseClozeSegments("A {{generator}} uses yield")
    const blanks = segments.filter((s) => s.type === "blank")
    expect(blanks).toHaveLength(1)
    expect((blanks[0] as { type: "blank"; term: string }).term).toBe("generator")
  })
})
```

## Risk Flags

**Risk 1: Route ordering conflict in flashcards.py**
FastAPI matches routes in registration order. The existing comment warns that `/audit/{document_id}` must come before `/{document_id}`. The new `/cloze/{section_id}` route MUST also be registered before the `GET /{document_id}` route. The POST method means there is no GET conflict, but a careful inspection of route order is required. The route prefix is `/flashcards`, so `POST /flashcards/cloze/{section_id}` will not collide with `GET /flashcards/{document_id}` (different HTTP methods). Still, keep the ordering consistent and add a comment.
Mitigation: Insert the new endpoint near the existing `/generate`, `/from-gaps` endpoints (all POST, all before `/{document_id}`).

**Risk 2: Vitest not configured in the frontend**
The story requires a Vitest unit test for `parseClozeSegments`. No Vitest is present; only `tsc --noEmit` and `npm run build` are current CI checks. Adding Vitest requires `npm install --save-dev vitest` and config changes. This will not break existing CI but adds a new `npm run test` script. The implementation plan adds this.
Mitigation: Add `vitest` as a dev dependency, add a minimal `vitest.config.ts`, and add `"test": "vitest run"` to scripts. The existing `npx tsc --noEmit` check is unaffected.

**Risk 3: Study.tsx GeneratePanel Cloze tab requires a section_id**
Cloze generation is section-scoped (the endpoint takes `section_id` not `document_id`). The current "From Text" tab can optionally scope to a section. The Cloze tab MUST have a section selector and the Generate button must be disabled until a section is selected. This is a UX constraint, not a bug, but must be clearly enforced to avoid calling the endpoint without a section_id.
Mitigation: Disable the "Generate Cloze Cards" button when no section is selected. Show helper text "Select a section to generate cloze cards."

**Risk 4: cloze_text None handling in StudySession**
The `Flashcard` interface in `StudySession.tsx` currently has no `cloze_text` or `flashcard_type`. Adding these as nullable means existing cards (which return `null` from the API) will work correctly with the `isClozeCard` guard. The `FlashcardResponse` Pydantic model sets `cloze_text: str | None = None` which serializes to `null` in JSON, so no migration needed on existing rows.
Mitigation: The guard `isClozeCard = flashcard_type === 'cloze' && cloze_text !== null && /\{\{.+?\}\}/.test(cloze_text)` handles all null/undefined cases.

**Risk 5: question field for list views**
The `question` field is used in the Study tab card list and session history. For cloze cards, `question` stores the `[____]` display form, which is readable. The `cloze_text` field carries the `{{term}}` form for the ClozeCard renderer. This dual-field pattern follows the existing `question`/`answer` pattern and does not break list rendering.

**Risk 6: `_parse_cloze_text` regex import collision**
The module already imports `re` at the top. Adding `_CLOZE_BLANK_RE = re.compile(r"\{\{(.+?)\}\}")` as a module-level compiled regex is the correct pattern. No new imports needed.

**Risk 7: Backend test fixture for SectionModel**
`test_flashcards.py` does not have a `_make_section()` helper. The new test file must create `SectionModel` rows directly. Ensure the `preview` field is populated with sufficient length (>= 200 chars as section filtering threshold) to avoid the section being skipped by `_fetch_chunks` heuristics. Since `generate_cloze` directly queries by `section_id` on `ChunkModel`, the `preview` length does not actually gate cloze generation. Still, keep the fixture realistic.

## Notes on Study.tsx GeneratePanel vs. GenerateFlashcardsDialog.tsx

The story says "GenerateFlashcardsDialog.tsx: add 'Cloze' tab alongside 'Q&A'". However, reading both files:

- `GenerateFlashcardsDialog.tsx` is for **note-based** flashcard generation (generates from notes by tag or note IDs). It does not have a "Q&A" tab — it has "By Tag" and "Specific Notes" tabs.
- `Study.tsx` contains `GeneratePanel` which is the flashcard generation UI for documents (from text, from entities).

The story description says cloze cards are generated via `POST /flashcards/cloze/{section_id}` (section-scoped, from document content). The correct location for the Cloze tab is the `GeneratePanel` inside `Study.tsx`, NOT `GenerateFlashcardsDialog.tsx`.

The phrase "alongside 'Q&A'" in the story refers to the conceptual distinction (Q&A vs cloze as card types), not a literal tab named "Q&A" in a dialog. The "From Text" tab in `GeneratePanel` generates Q&A cards; adding a "Cloze" tab alongside it is the correct interpretation.

`GenerateFlashcardsDialog.tsx` does not need to be changed.
