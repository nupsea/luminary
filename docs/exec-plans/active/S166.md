# Exec Plan: S166 -- Semantic note clustering via HDBSCAN: auto-suggest collections from embedding similarity

**Phase:** V3 Phase 1 - Note Organization and Knowledge Graph
**Priority:** 9
**Status:** in-progress

## Goal

Add an automated semantic clustering feature that reads all note embeddings from LanceDB
`note_vectors_v2`, runs HDBSCAN (sklearn >= 1.3) to group notes by topic, generates a
human-readable collection name per cluster via LiteLLM, and persists each cluster as a
`ClusterSuggestionModel` row with status='pending'. The Notes sidebar gains a "Suggested
Collections" section (visible only when pending suggestions exist) with an "Auto-organize"
button that fires the clustering job and suggestion cards that allow the user to Accept
(creates a real NoteCollectionModel + member rows) or Reject (marks row rejected).

Dependencies already implemented: S161 (NoteCollectionModel, NoteCollectionMemberModel in
models.py and db_init.py), S170 (note_vectors_v2 confirmed 1024-dim).

### ClusterSuggestionModel (models.py)

Add after TagMergeSuggestionModel (currently the last model before PredictionEventModel):

```python
class ClusterSuggestionModel(Base):
    """A cluster of semantically similar notes suggested as a collection.

    note_ids: JSON list of note id strings.
    confidence_score: mean pairwise cosine similarity of cluster members (0.0 - 1.0).
    status: 'pending' | 'accepted' | 'rejected'
    """

    __tablename__ = "cluster_suggestions"

    id: Mapped[str] = mapped_column(String, primary_key=True)
    suggested_name: Mapped[str] = mapped_column(String, nullable=False)
    note_ids: Mapped[list] = mapped_column(JSON, nullable=False)
    confidence_score: Mapped[float] = mapped_column(Float, nullable=False)
    status: Mapped[str] = mapped_column(String, nullable=False, default="pending")
    created_at: Mapped[datetime] = mapped_column(DateTime, default=lambda: datetime.now(UTC))
```

### ClusteringService (clustering_service.py)

Key invariants:
- All LanceDB calls via `asyncio.to_thread` (I-2).
- sklearn HDBSCAN is CPU-bound sync work; run inside `asyncio.to_thread` as well.
- LiteLLM import at module level with `# noqa: PLC0415` (AC requirement; unit tests patch
  at `app.services.clustering_service`).
- Rate-limit guard: if any ClusterSuggestionModel with status='pending' and
  `created_at > now - 1 hour` exists, return -1 immediately (no LanceDB fetch, no HDBSCAN).
- Fire-and-forget caller in the router spawns `asyncio.create_task` and opens its own
  session via `get_session_factory()()` (FastAPI fire-and-forget pattern from patterns.md).

Pseudocode for `cluster_notes(db)`:

```
1. Rate-limit check: query ClusterSuggestionModel WHERE status='pending'
   AND created_at > now - 1h. If any rows: return -1.
2. Fetch all rows from note_vectors_v2 via asyncio.to_thread:
   rows = await asyncio.to_thread(lambda: tbl.to_pandas())
   If < 3 rows (HDBSCAN min_cluster_size): return 0.
3. Build numpy matrix from rows["vector"].tolist()
4. Run HDBSCAN in thread:
   labels = await asyncio.to_thread(
       lambda: HDBSCAN(min_cluster_size=3, min_samples=2, metric='cosine')
               .fit_predict(matrix)
   )
5. For each unique label != -1:
   a. Gather member note_ids (rows where label matches).
   b. Compute centroid = mean of member vectors.
   c. Select up to 3 medoid note_ids (closest to centroid by cosine distance).
   d. confidence_score = mean pairwise cosine similarity across all cluster members.
      Use sklearn.metrics.pairwise.cosine_similarity(member_matrix) -- symmetric matrix,
      take upper triangle values, mean them.
   e. Fetch NoteModel rows for medoid_ids to get content excerpts (first 200 chars each).
   f. LLM name generation: call litellm.acompletion with a short prompt including the
      3 excerpts; parse the response for a 2-5 word collection name.
   g. Insert ClusterSuggestionModel row; await session.flush() (not commit -- caller commits).
6. await db.commit()
7. Return number of suggestions inserted.
```

LiteLLM import pattern (module-level per AC):
```python
import litellm  # noqa: PLC0415
```
Unit tests patch `app.services.clustering_service.litellm.acompletion`.

### Router endpoints (notes.py)

Static paths that must be registered BEFORE the existing `/{note_id}` routes
(FastAPI dynamic path routing -- patterns.md):

```
POST /notes/cluster
GET  /notes/cluster/suggestions
POST /notes/cluster/suggestions/{id}/accept
POST /notes/cluster/suggestions/{id}/reject
```

Fire-and-forget pattern for POST /notes/cluster:
- Check rate-limit synchronously in the handler (SELECT from DB, fast).
  If cached: return `{"cached": True, "last_run": <ISO datetime of latest pending row>}`.
- Otherwise: spawn `asyncio.create_task(_run_clustering())` where `_run_clustering` opens
  its own session via `async with get_session_factory()() as session:`.
- Return `{"queued": True, "total_notes": <note count from NoteModel>}` immediately.

GET /notes/cluster/suggestions response schema:
```python
class NotePreview(BaseModel):
    note_id: str
    excerpt: str  # first 200 chars of content

class ClusterSuggestionResponse(BaseModel):
    id: str
    suggested_name: str
    note_count: int
    confidence_score: float
    status: str
    created_at: datetime
    previews: list[NotePreview]  # up to 3 medoid note excerpts
```

The handler fetches ClusterSuggestionModel rows WHERE status='pending', ordered by
confidence_score DESC. For each suggestion, fetches up to 3 NoteModel rows from note_ids
(take first 3 elements) to build previews. Uses the bulk one-to-many loading pattern
(patterns.md) -- single query for all note_ids across all suggestions, then map back.

POST /notes/cluster/suggestions/{id}/accept:
- Load ClusterSuggestionModel by id; 404 if not found.
- Create NoteCollectionModel (name=suggested_name, color='#6366F1').
- For each note_id in suggestion.note_ids: insert NoteCollectionMemberModel row
  (INSERT OR IGNORE via ON CONFLICT, same as existing collection member pattern).
- Set suggestion.status = 'accepted'.
- await db.commit().
- Return `{"collection_id": <new collection id>}`.

POST /notes/cluster/suggestions/{id}/reject:
- Load ClusterSuggestionModel by id; 404 if not found.
- Set suggestion.status = 'rejected'.
- await db.commit().
- Return `{"ok": True}`.
